Componentes con y sin estado


En React, un componente con estado es un componente que tiene estado.

Por otro lado, los componentes sin estado no tienen estado. Tanto los componentes con estado como los sin estado pueden utilizar props.

class Tienda extends React.Component {

    constructor(props) {

        super(props);

        this.state = { vender: "cualquier cosa" };

    }

    render() {

        return <h1>Estoy vendiendo {this.state.vender}.</h1>;

    }

}

class Semana extends React.Component {

    render() {

        return <h1>Hoy es {this.props.dia}!</h1>;

    }

}

Uno de los patrones de programación más comunes en React es utilizar componentes padres con estado para mantener su propio estado y pasarlo a uno o más componentes hijos sin estado como props.

class BabyYoda extends React.Component {

Manejadores de eventos y estado


Las funciones de manejo de eventos en React se utilizan a menudo para actualizar el estado.

Estas funciones de manejo suelen recibir un evento como argumento, que se utiliza para actualizar los valores del estado correctamente.


En el siguiente ejemplo de código, utilizamos event.target.value para obtener el valor del input.

JavaScript Copiar código 
 
                            
class MiComponente extends React.Component {

    constructor(props) {

        super(props);

        this.state = { texto: "" };

        this.cambiarTexto = this.cambiarTexto.bind(this);

    }

    cambiarTexto(event) {

        this.setState({

           texto: event.target.value
 
        });

    }

    render() {

        return (

            <div>
 
                <input onChange={this.cambiarTexto} value={this.state.texto} />

                <p>Escribiste {this.state.texto}</p>

            </div>

        );

    }

}

Cambiando Props y Estado


En React, un componente nunca debe cambiar sus propias props directamente. Un componente padre debe cambiarlas.

El estado, por otro lado, es lo opuesto a las props: un componente realiza un seguimiento de su propio estado y puede cambiarlo en cualquier momento.

JavaScript Copiar código 
 
                            
class Reloj extends React.Component {

    constructor(props) {

        super(props);

        this.state = { ahora: new Date() };

        this.actualizarHora = this.actualizarHora.bind(this);

    }

    actualizarHora() {

        this.setState({ ahora: new Date() });

    }

    render() {

        return (
   
            <div>

                <h1>Ahora son las {this.state.ahora.toLocaleTimeString()}</h1>
 
                <h2>{this.props.subtitulo}</h2>

                <button onClick={this.actualizarHora}>Actualizar el reloj</button>

            </div>

        );

    }

}

Props: Pasando funciones de cambio de estado


Si un componente padre de React define una función que cambia su estado, esa función se puede pasar a un componente hijo y llamarse dentro del componente para actualizar el estado del componente padre.

                <p>{this.state.nombre}</p>

            </div>

        );

    }

}

Componentes sin estado heredados


Los componentes sin estado heredan de los componentes con estado.

Resumen

En esta parte, veremos una versión simple de un patrón de programación. En este patrón, un componente con estado se encarga de manejar los datos y el estado de la aplicación. A su vez, un componente sin estado recibe los datos del componente con estado y los muestra en pantalla.

Nuestro patrón de programación utiliza dos componentes React: un componente con estado y un componente sin estado. "Stateful" describe cualquier componente que tiene una propiedad de estado; "stateless" describe cualquier componente que no la tiene.

Importante

Propiedad State

Los componentes con estado tienen una propiedad llamada “state”, mientras que los componentes sin estado no tienen esta propiedad. Al utilizar este patrón de programación, podemos separar la lógica de la aplicación de su presentación visual. Es decir, un componente con estado pasa su estado a un componente sin estado.

Ahora vamos a construir un componente con estado que pueda pasar su estado a un componente sin estado. Para hacer esto, necesitamos dos clases de componentes: una clase con estado y una clase sin estado.

1

Vamos a construir la clase con estado: En un archivo "Parent.js", que es donde construiremos esta clase, primero importamos la biblioteca React y la biblioteca ReactDOM.

2

Creamos una nueva clase llamada "Parent": Es nuestra clase con estado. Esta clase extiende la clase "Component" de React. Para hacer que esta clase sea "stateful", necesitamos crear un constructor. Dentro del constructor, llamamos a la función "super(props)" y luego declaramos una propiedad "state" con un objeto que tiene una sola propiedad "name" con el valor de "Juan".

3

En la función "render()": Retornamos un div vacío.

JavaScript Copiar código 
 
                            
import React from "react";

import ReactDOM from "react-dom";

class Parent extends React.Component {

    constructor(props) {

        super(props);

        this.state = { name: "Juan" };

    }

    render() {

        return <div></div>;

    }

}

ReactDOM.render(<Parent />, document.getElementById("root"));
4

En el archivo Childjs.: Construimos nuestro componente sin estado llamado "Child", que recibe las props y muestra el nombre recibido en pantalla.

JavaScript Copiar código 
 
                            
import React from "react";

export class Child extends React.Component {

    render() {

        return <h1>Hey, my name is {this.props.name}!</h1>;
  
    }

}
5

Finalmente: Vamos a renderizar nuestro componente "Child" dentro del componente "Parent" y pasarle el estado "name" como una propiedad (prop).

JavaScript Copiar código 
 
                            
import React from "react";

import ReactDOM from "react-dom";

import Child from "./Child";

class Parent extends React.Component {

    constructor(props) {

        super(props);

        this.state = { name: "Juan" };

    }

    render() {

        return <Child name={this.state.name} />;

    }

}

ReactDOM.render(<Parent />, document.getElementById("app"));
6

En el archivo Parentjs.: Importamos nuestro componente "Child" desde el archivo "Child.js".

7

En la función "render()" de nuestro componente "Parent": Renderizamos el componente "Child" y le pasamos el estado "name" como una propiedad (prop).

Al renderizar el componente "Child" dentro de "Parent" y pasarle el estado "name" como una propiedad (prop), logramos que nuestro componente "Child" se vuelva dependiente del estado de "Parent". Cuando el estado de "Parent" cambie, el componente "Child" se volverá a renderizar y se mostrará el nuevo valor de "name" en pantalla.


Componentes sin estado heredados (2)


Ahora, como aprendimos antes, un componente puede cambiar su estado llamando a this.setState(), por lo que quizás te preguntes: ¿cómo cambia un componente hijo sus props? La respuesta es: ¡no lo hace!

Un componente nunca debería actualizar this.props. Observemos este componente para ver un ejemplo de lo que no se debe hacer.

JavaScript Copiar código 
 
                            
import React from "react";

class Bad extends React.Component {

    render() {

        this.props.message = "yo"; // NOOOOOOOOOOOOOO!!!

        return <h1>{this.props.message}</h1>;

    }

}

Esto puede ser confuso. Las props y el state almacenan información dinámica. Por definición, la información dinámica puede cambiar. Si un componente no puede cambiar sus props, entonces, ¿para qué sirven las props?

Un componente React debería usar props para almacenar información que puede ser cambiada, pero que sólo puede ser cambiada por un componente diferente.

Un componente React debería usar state para almacenar información que el propio componente puede cambiar.


Entonces, ¿cómo actualiza un componente hijo sin estado el estado del componente padre?

1

El componente padre define un método que llama a this.setState().

2

El componente padre enlaza el método recién definido a la instancia actual del componente en su constructor. Esto garantiza que cuando pasemos el método al componente hijo, todavía actualizará el componente padre.

3

Una vez que el padre ha definido un método que actualiza su estado y se ha vinculado a él, el padre pasa ese método al hijo.

4

El hijo recibe la función pasada y la usa como un controlador de eventos.


Veamos un ejemplo:

JavaScript Copiar código 
 
                            
import React, { Component } from "react";

class ParentComponent extends Component {

    constructor(props) {

        super(props);

        this.state = { counter: 0 };

        this.handleClick = this.handleClick.bind(this);

    }

    handleClick() {

        this.setState({ counter: this.state.counter + 1 });

    }

    render() {

        return (

            <div>

                <h1>Counter: {this.state.counter}</h1>

                <ChildComponent onClick={this.handleClick} />

            </div>

        );

    }

}

class ChildComponent extends Component {

    render() {

        return <button onClick={this.props.onClick}>Increment counter</button>;

    }

}

export default ParentComponent;

En este ejemplo:

1

El componente ParentComponent es el componente padre y el componente ChildComponent es el componente hijo.
El componente ParentComponent define un estado con una propiedad counter, que es el valor del contador que queremos incrementar. También define un método handleClick que actualiza el estado cada vez que se hace clic en el botón dentro del componente ChildComponent. Este método está vinculado al this de ParentComponent usando .bind( this).

2

El componente ParentComponent pasa el método handleClick como una prop al componente ChildComponent.
Dentro del componente ChildComponent, el método handleClick se usa como un controlador de eventos para el botón. Cada vez que se hace clic en el botón, se llama al método handleClick pasado como prop, lo que hace que se actualice el estado en el componente ParentComponent.


Pasar todo el estado al hijo


Es posible pasar todo el objeto de estado al componente hijo en React. Para hacer esto, simplemente pasamos el objeto de estado como una propiedad (prop) al componente hijo.

Ejemplo

Si tenemos un componente padre con un objeto de estado que contiene las propiedades "name", "age" y "gender", podemos pasar todo el objeto de estado al componente hijo.

JavaScript Copiar código 
 
                            
import React from "react";

import Child from "./Child";

class Parent extends React.Component {

    constructor(props) {

        super(props);

        this.state = {

            name: "Juan",
 
            age: 30,

            gender: "Masculino"

        };

    }

    render() {

        return (

            <div>

                <Child state={this.state} />

            </div>

        );

    }

}

Componente hermano: Mostrar información


A continuación, aprenderemos cómo compartir información entre componentes hermanos utilizando un componente padre.

Resumen

En React, a menudo es necesario que los componentes hermanos compartan información entre sí. Esto se puede lograr utilizando un componente padre para mantener el estado y pasar datos a través de las props a los componentes hermanos.

1

Crearemos un componente padre que mantendrá el estado y pasará la información a sus componentes hijos.

JavaScript Copiar código 
 
                            
class Padre extends React.Component {

    constructor(props) {

        super(props);

        this.state = {

            mensaje: ""

        };

        this.actualizarMensaje = this.actualizarMensaje.bind(this);

    }

    actualizarMensaje(nuevoMensaje) {

        this.setState({

            mensaje: nuevoMensaje

        });

    }

    render() {

        return <div>{/* Aquí renderizaremos los componentes hermanos */}</div>;

    }

}
2

Crearemos dos componentes hermanos, ComponenteA y ComponenteB. Uno de ellos será responsable de actualizar el estado del componente padre, y el otro mostrará la información del estado.

JavaScript Copiar código 
 
                            
class ComponenteA extends React.Component {

    constructor(props) {

        super(props);

        this.handleChange = this.handleChange.bind(this);

    }

    handleChange(event) {

        this.props.actualizarMensaje(event.target.value);

    }

    render() {

        return (

            <div>

                <input onChange={this.handleChange} />

            </div>

        );

    }

}

class ComponenteB extends React.Component {

    render() {

        return (

            <div>

                <h1>Mensaje: {this.props.mensaje}</h1>

            </div>

        );

    }

}
3

Finalmente, renderizamos los componentes hermanos en el componente padre y pasamos el estado y las funciones de actualización a través de las props.

JavaScript Copiar código 
 
                            
class Padre extends React.Component {

// ...

    render() {

        return (

            <div>

                <ComponenteA actualizarMensaje={this.actualizarMensaje} />

                <ComponenteB mensaje={this.state.mensaje} />

            </div>

        );

    }

}

Tip

Creando un componente padre que mantenga el estado y pasando la información a través de las props a sus componentes hijos, podemos lograr una comunicación efectiva entre componentes hermanos en una aplicación React.

De esta forma, podemos compartir información entre componentes hermanos utilizando un componente padre.


Ciclo de vida en React


Hemos visto que los componentes de React pueden ser altamente dinámicos. Se crean, se renderizan, se agregan al DOM, se actualizan y se eliminan. Todos estos pasos forman parte del ciclo de vida de un componente.

El ciclo de vida del componente tiene tres partes de alto nivel:

1

Montaje: Cuando el componente se inicializa y se coloca en el DOM por primera vez.

2

Actualización: Cuando el componente se actualiza como resultado del cambio de estado o de las props modificadas.

3

Desmontaje: Cuando el componente se elimina del DOM.


Todos los componentes de React con los que hemos interactuado realizan el primer paso como mínimo. Si un componente nunca se montara, nunca lo veríamos.

La mayoría de los componentes se actualizan en algún momento

Un componente puramente estático, como por ejemplo un logotipo, podría no actualizarse nunca. Pero si cambia el estado de un componente, se actualiza. O si se pasan diferentes props a un componente, también se actualiza.

Por último, un componente se desmonta cuando se elimina del DOM.

Ejemplo

Si tenemos un botón que oculta un componente, es probable que ese componente se desmonte. Si nuestra aplicación tiene varias pantallas, es probable que cada pantalla (y todos sus componentes secundarios) se desmonten. Si un componente está "vivo" durante toda la vida útil de nuestra aplicación (por ejemplo, un componente de nivel superior <App /> o una barra de navegación persistente), no se desmontará. Pero la mayoría de los componentes pueden desmontarse de alguna manera.


Cabe destacar que cada instancia de componente tiene su propio ciclo de vida. Por ejemplo, si tenemos 3 botones en una página, entonces hay 3 instancias de componentes, cada una con su propio ciclo de vida. Sin embargo, una vez que se desmonta una instancia de componente, eso es todo, nunca se volverá a montar, actualizar o desmontar.


Fases del ciclo de vida


Hay tres categorías de métodos del ciclo de vida: montaje, actualización y desmontaje.

1

Montaje: Un componente se "monta" cuando se renderiza por primera vez. Es en este momento cuando se llaman los métodos del ciclo de vida de montaje.

2

Actualización: La primera vez que se renderiza una instancia de componente, no se actualiza. A partir de la segunda renderización, un componente se actualiza cada vez que se renderiza.

3

Desmontaje: El período de desmontaje de un componente ocurre cuando el componente se elimina del DOM. Esto podría suceder si el DOM se vuelve a renderizar sin el componente, o si el usuario navega a un sitio web diferente o cierra su navegador web.


Montaje: Métodos del ciclo de vida


React admite tres métodos del ciclo de vida de montaje para clases de componentes:

ComponentWillMount()

Render()

ComponentDidMount()

Primero se llamará a componentWillMount(), seguido del método render() y, finalmente, el método componentDidMount().


Montaje de un componente


Un componente de React se monta cuando se renderiza en el DOM por primera vez. Si ya está montado, un componente puede volver a renderizarse si necesita cambiar su apariencia o contenido.

componentDidMount


Este método es ideal para realizar acciones como realizar solicitudes de datos, configurar suscripciones a eventos o iniciar temporizadores, ya que se garantiza que el componente ya esté en el DOM cuando se ejecute.

Definición: ComponentDidMount

Es un método del ciclo de vida que se ejecuta inmediatamente después de que el componente se monta en el DOM por primera vez.

Ejemplo

Cómo usar componentDidMount() para realizar acciones después de que el componente se haya montado en el DOM. En este caso, simplemente cambiamos el mensaje después de 3 segundos:

JavaScript Copiar código 
 
                            
class WelcomeMessage extends React.Component {

    constructor(props) {

        super(props);

        this.state = {

            message: "¡Hola!"

        };

    }

    componentDidMount() {

        setTimeout(() => {

            this.setState({

                 message: "¡Bienvenido a nuestra aplicación!"

            });

        }, 3000);

    }

    render() {

        return <div>{this.state.message}</div>;

    }

}
1

En el constructor: Inicializamos el estado del componente con un mensaje de bienvenida inicial: "¡Hola!".

2

El método componentDidMount(): Se ejecuta automáticamente después de que el componente se haya montado en el DOM. En este método, configuramos un temporizador utilizando setTimeout(), que cambiará el estado del mensaje después de 3 segundos (3000 milisegundos).

3

Cuando el temporizador se dispara después de 3 segundos: Actualizamos el estado del mensaje a "¡Bienvenido a nuestra aplicación!".

4

Cuando el estado del componente se actualiza: El método render() se ejecuta nuevamente y muestra el nuevo mensaje en la pantalla.



Renderizado inicial:

Luego de 3 segundos:


componentWillUnmount


React admite un método de desmontaje del ciclo de vida, componentWillUnmount, que se llamará justo antes de que un componente se elimine del DOM.

Definición: ComponentWillUnmount

Se utiliza para realizar cualquier limpieza necesaria (cancelar temporizadores o intervalos, por ejemplo) antes de que el componente desaparezca.

Ten en cuenta que el método this.setState() no debe llamarse dentro de componentWillUnmount() porque el componente no se volverá a renderizar.

Ejemplo

Supongamos que tenemos un componente que muestra un temporizador en la pantalla. El temporizador se actualiza cada segundo.

JavaScript Copiar código 
 
                            
class Timer extends React.Component {

    constructor(props) {

        super(props);

            this.state = {
   
            seconds: 0

        };

    }

    componentDidMount() {

        this.timerID = setInterval(() => this.tick(), 1000);

    }

    componentWillUnmount() {

        clearInterval(this.timerID);

    }

    tick() {

        this.setState((prevState) => ({

            seconds: prevState.seconds + 1

        }));

    }

    render() {

        return <div>Segundos transcurridos: {this.state.seconds}</div>;

    }

}
1

En el método componentDidMount(): Iniciamos un temporizador utilizando setInterval(). Este temporizador llama a la función tick() cada 1000 milisegundos (1 segundo). También almacenamos el ID del temporizador en this.timerID para que podamos referenciarlo más tarde.

2

La función tick(): Actualiza el estado del componente, incrementando la cantidad de segundos transcurridos en 1.

3

Si el componente Timer se elimina del DOM en algún momento (por ejemplo, al navegar a otra página o al ocultar el componente): Queremos detener el temporizador y evitar que siga ejecutándose en segundo plano. Aquí es donde entra en juego componentWillUnmount().

4

En el método componentWillUnmount(): Llamamos a clearInterval() y le pasamos el ID del temporizador (this.timerID) para cancelar el temporizador y detener la ejecución de la función tick().

5

Finalmente: En el método render(), mostramos la cantidad de segundos transcurridos almacenados en el estado del componente.


Fase de montaje del componente


Un componente se "monta" cuando se renderiza por primera vez. Cuando un componente se monta, automáticamente llama a estos tres métodos, en el orden de:

1

Constructor()

2

Render()

3

ComponentDidUpdate()


Método de actualización del ciclo de vida


Cuando un componente se actualiza, shouldComponentUpdate() se llama después de componentWillReceiveProps(), pero aún antes de que comience la renderización.

Recibe automáticamente dos argumentos: nextProps y nextState.

ShouldComponentUpdate() debe devolver true o false.

La mejor manera de usar este método es hacer que devuelva false solo bajo ciertas condiciones. Si se cumplen esas condiciones, el componente no se actualizará.

JavaScript Copiar código 
 
                            
shouldComponentUpdate(nextProps, nextState) {

    if ((this.props.text == nextProps.text) &&

       (this.state.subtext == nextState.subtext)) {

        return false;

    } else {

        return true;

    }

}

componentDidUpdate


Este método se utiliza principalmente para realizar acciones adicionales en respuesta a cambios en el componente, como manipulación del DOM, llamadas a una API o actualización de datos en función de las nuevas propiedades o estado.

Definición: ComponentDidUpdate

Es un método del ciclo de vida que se ejecuta cuando el componente se actualiza, es decir, cuando se producen cambios en el estado o en las propiedades (props) del componente.

Ejemplo

Supongamos que tenemos un componente que muestra un mensaje de bienvenida a un usuario en función de su nombre, que se pasa a través de las propiedades (props).

JavaScript Copiar código 
 
                            
class WelcomeMessage extends React.Component {

    constructor(props) {

        super(props);

        this.state = {

            message: ""

        };
 
    }

    componentDidMount() {

        this.updateMessage(this.props.name);
 
    }

    componentDidUpdate(prevProps, prevState) {

        if (prevProps.name !== this.props.name) {

            this.updateMessage(this.props.name);

        }

    }

    updateMessage(name) {

        this.setState({ message: `¡Bienvenido, ${name}!` });

    }

    render() {

        return <div>{this.state.message}</div>;

    }

}

En este ejemplo, el componente WelcomeMessage recibe el nombre del usuario a través de las propiedades (props). El mensaje de bienvenida se construye y almacena en el estado del componente.

1

Cuando el componente se monta por primera vez: Se llama al método componentDidMount(), que a su vez llama a updateMessage() para establecer el mensaje de bienvenida en función del nombre proporcionado.

2

Si el nombre del usuario cambia: Queremos actualizar el mensaje de bienvenida. Aquí es donde entra en juego componentDidUpdate(). Este método se ejecuta después de que el componente se actualiza, es decir, después de que se produzcan cambios en las propiedades o en el estado.

3

En componentDidUpdate(): Comparamos las propiedades anteriores (prevProps) con las propiedades actuales (this.props). Si el nombre del usuario ha cambiado, actualizamos el mensaje de bienvenida llamando nuevamente a updateMessage() con el nuevo nombre.

4

Finalmente: En el método render(), mostramos el mensaje de bienvenida almacenado en el estado del componente.

Ahora que hemos creado el método render() para que <Clock /> renderice un elemento <div>, el siguiente paso es establecer el estado del componente.

3

Queremos almacenar la hora actual en el estado de <Clock />: Para hacer esto, necesitamos crear un constructor para el componente y establecer el estado inicial a { date: new Date() }. También es importante llamar al método super(props) dentro del constructor.

El constructor es el primer método que se llama cuando se monta la instancia del componente. Entonces, agregamos lo siguiente al componente:

JavaScript Copiar código 
 
                            
class Clock extends React.Component {

    constructor(props) {

        super(props);

        this.state = {

            date: new Date()

        };

    }

    render() {

        return <div></div>;

    }

}

Ahora, <Clock /> tiene un estado inicial y está listo para ser utilizado para "conectar" el estado con la pantalla.

4

Ahora vamos a actualizar el reloj en tiempo real: Para hacer esto, necesitamos implementar el método componentDidMount() que se ejecuta automáticamente después de que el componente es montado. Aquí es donde podemos establecer nuestro intervalo de actualización para el reloj.

5

En el método componentDidMount(): Creamos un intervalo usando setInterval(). Este intervalo debe llamar a this.tick() cada segundo. La función tick() actualizará el estado de <Clock /> con una nueva fecha. Recordemos usar this.setState() para actualizar el estado de manera adecuada.

6

Necesitamos implementar componentWillUnmount(): Para limpiar el intervalo después de que el componente se desmonte.

JavaScript Copiar código 
 
                            
import React from "react"; // importamos React para poder utilizarlo en el componente

import ReactDOM from "react-dom"; // importamos ReactDOM para poder renderizar el componente en el navegador

// creamos la clase Clock que extiende la clase React.Component

class Clock extends React.Component {

    // definimos el constructor de la clase y le pasamos el parámetro props

    constructor(props) {

        // llamamos al constructor de la clase padre con super(props)

        super(props); 

        // definimos el estado inicial con un objeto que tiene una propiedad date que contiene la fecha actual

        this.state = { date: new Date() };

    }



    // definimos el método render que se encarga de renderizar el componente en la página web

    render() {

        // lo que devuelve la función render

        return (

            <div>

                {/* un div que contiene la hora actual */}

                {this.state.date.toLocaleTimeString()}

            </div>

        );

    }



    // definimos el método componentDidMount, que se ejecuta después de que el componente se haya montado en la página

    componentDidMount() {

        // utilizamos setInterval para actualizar el estado cada segundo

        this.timerID = setInterval(

            () => this.tick(),

            1000

        );

    }



    // definimos el método componentWillUnmount, que se ejecuta justo antes de que el componente se desmonte de la página

    componentWillUnmount() {

        clearInterval(this.timerID); // detenemos el setInterval para evitar errores

    }



    // definimos el método tick que se encarga de actualizar el estado del componente

    tick() {

        this.setState({

            date: new Date()

        });

    }

}



// renderizamos el componente Clock en el elemento HTML con el id "root"

ReactDOM.render(<Clock />, document.getElementById("root"));

Ahora el reloj se actualizará en tiempo real y mostrará la hora actual.

Resumen

El código define una clase llamada Clock que extiende React.Component. El constructor establece el estado inicial con la fecha actual, y el método render() se encarga de mostrar la hora actual en un div.

Además, hay tres métodos adicionales en la clase:

ComponentDidMount(): Inicia un intervalo que actualiza el estado del componente cada segundo.

ComponentWillUnmount(): Detiene el intervalo antes de que el componente se desmonte.

Tick(): Actualiza el estado con la fecha actual.

Nota

ToLocaleTimeString()

Es un método de JavaScript que se utiliza para convertir la hora de un objeto Date en una cadena de texto con formato legible para humanos, de acuerdo con la configuración regional del sistema donde se está ejecutando el código.

Finalmente, el componente se renderiza en un elemento HTML con el id "root" utilizando ReactDOM.render().
