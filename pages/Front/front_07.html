Ideas Iniciales


React es una biblioteca de JavaScript para crear interfaces de usuario dinámicas y reutilizables. Aprenderemos los fundamentos de React y cómo utilizarlo para construir aplicaciones web modernas.

Mientras más crecen nuestras aplicaciones web, se vuelve más complicado nuestro trabajo utilizando solamente HTML, CSS y JavaScript. El valor de React es que nos facilita crear y organizar nuestro código, acelerar el proceso de creación, además de ofrecernos una mejor experiencia como desarrolladores.

Para comenzar en el mundo de React es muy importante tener una base sólida en las principales tecnologías front end: HTML, CSS y JavaScript, por lo que es clave repasar y tener bien claros los conceptos de clases anteriores.

Antes de profundizar en los conceptos de React, es importante tener un conocimiento sólido de los fundamentos de las aplicaciones web. En el mundo del desarrollo web, hay ciertos principios y prácticas que debemos tener en cuenta para asegurarnos de que nuestras aplicaciones sean eficientes, seguras y escalables.

En este sentido, es útil repasar algunos de los conceptos básicos del desarrollo web antes de empezar a programar en React. Para ello, vamos a examinar una aplicación de ejemplo en línea utilizando el navegador web. La aplicación que examinaremos será Airbnb: https://www.airbnb.com.ar/

IMPORTANTE

Para estudiar estos contenidos, utilizaremos el navegador Chrome, aunque es posible utilizar otros navegadores como Mozilla y Safari con algunas diferencias. Además, recomendamos mantener siempre abierta la Consola para desarrolladores en el navegador, ya que esta herramienta nos permitirá ver información importante sobre la comunicación entre el navegador y el servidor.



Presentación 


Comenzaremos inspeccionando una aplicación web para entender cómo funciona y cómo se comunican las diferentes partes a través de HTTP GET. Luego, presentaremos las bases de React y su importancia en el desarrollo web moderno. Abordaremos los módulos de JavaScript, aprendiendo cómo importar y exportar funciones entre diferentes archivos de código.

Veremos cómo utilizar Create React App, una herramienta que simplifica la creación de proyectos de React.

HTTP GET - Network


El servidor y el navegador web se comunican entre sí mediante el protocolo HTTP. La pestaña Network muestra cómo se comunican el navegador y el servidor.

La pestaña Network en el panel de Developer Console muestra información sobre todas las solicitudes HTTP que se realizan al cargar la página de Airbnb.


Esta pestaña proporciona una visión detallada de todas las solicitudes realizadas por la página, incluyendo información sobre la URL solicitada, el método utilizado, los parámetros enviados en la solicitud, los encabezados de la solicitud y de respuesta, y los tiempos de respuesta.

Cuando recargamos la página (presionamos la tecla F5 o el símbolo ↻ en el navegador), la consola muestra una lista de solicitudes HTTP que se realizaron para cargar la página, como solicitudes de recursos como imágenes, archivos CSS y JavaScript, y otras solicitudes de recursos que son necesarios para que la página se cargue correctamente.

Es posible ver el tiempo de carga de cada recurso solicitado, así como el tamaño del recurso en bytes y la velocidad de transferencia. Además, también se puede ver información sobre cualquier error que haya ocurrido durante la solicitud, como errores de tiempo de espera, errores de conexión y errores HTTP.

Una cabecera importante Content-Type nos dice que la respuesta es un archivo de texto en formato utf-8, cuyo contenido se ha formateado con HTML. De esta manera, el navegador sabe que la respuesta es una página HTML normal y la representa en el navegador "como una página web".

Un poco más de info sobre las cabeceras:

Cache-control: (No-store, max-age=0, private, must-revalidate) Indica al cliente que no debe almacenar en caché la respuesta, incluso si normalmente lo haría. También indica que la respuesta no debe almacenarse en caché por ningún intermediario (por ejemplo, un proxy) y que se debe validar la respuesta en cada acceso. En resumen, el servidor no desea que se almacene en caché ninguna parte de la respuesta.

Server: El valor de este header indica el software del servidor web que maneja la solicitud. En este caso, el servidor web es Nginx.

Set-cookie: Este header se utiliza para enviar cookies al cliente. En este caso, se están enviando varias cookies con diferentes nombres y valores. Cada cookie tiene una fecha de caducidad, un dominio y una ruta específicos.

Status: Este header indica el estado de la solicitud HTTP. En este caso, la solicitud se completó correctamente y se envió una respuesta con un código de estado 200 OK.

Strict-transport-security: Este header indica que el sitio web solo debe accederse a través de una conexión segura HTTPS y no a través de HTTP. El valor max-age especifica cuánto tiempo el navegador debe recordar esta política.

Introducción a React



React es una biblioteca de JavaScript desarrollada por Facebook que permite crear interfaces de usuario (UI) dinámicas y eficientes. A diferencia de las páginas web estáticas creadas solo con HTML y CSS, React permite crear componentes reutilizables que pueden cambiar su apariencia y comportamiento en función de los datos y la interacción del usuario. Módulos JavaScript


Un módulo en JavaScript es simplemente un archivo que contiene código.

Los módulos permiten organizar y acceder al código de manera más estructurada.


Un módulo puede ser tan simple como una sola función en un archivo separado.

Por ejemplo, supongamos que tenemos una función addTwo en un archivo llamado addTwo.js:

JavaScript Copiar código 
 
                            
function addTwo(a, b) {

    console.log(a + b);

}

Para convertir este archivo en un módulo JavaScript, debemos usar la sintaxis de exportación.

Hay dos formas principales de exportar módulos en JavaScript:

Exportaciones predeterminadas.

Exportaciones con nombre.


Exportaciones predeterminadas


Cada módulo JavaScript puede tener una exportación predeterminada. Para exportar la función addTwo como predeterminada, podemos agregar las palabras clave export default antes de la declaración de la función:

JavaScript Copiar código 
 
                            
export default function addTwo(a, b) {

    console.log(a + b);

}

O también podemos usar una sintaxis alternativa:

JavaScript Copiar código 
 
                            
function addTwo(a, b) {

    console.log(a + b);

}

export default addTwo;

Exportaciones con nombre


Las exportaciones con nombre permiten exportar solo ciertas partes de un archivo JavaScript. Podemos exportar tantos elementos como queramos.

Por ejemplo, supongamos que tenemos dos funciones, addTwo y addThree:

JavaScript Copiar código 
 
                            
function addTwo(a, b) {

    console.log(a + b);

}

function addThree(a, b, c) {

    console.log(a + b + c);

}

Para exportar ambas funciones como exportaciones con nombre, podemos hacer lo siguiente:

JavaScript Copiar código 
 
                            
export function addTwo(a, b) {

    console.log(a + b);

}

export function addThree(a, b, c) {

    console.log(a + b + c);

}

Otra forma de hacerlo es:

JavaScript Copiar código 
 
                            
function addTwo(a, b) {

    console.log(a + b);

}

function addThree(a, b, c) {

    console.log(a + b + c);

}

export { addTwo, addThree };

Importar módulos


La sintaxis exacta para importar un módulo depende de cómo se exportó. Supongamos que tenemos dos módulos, addTwo.js y mathOperations.js, y queremos importar el módulo addTwo.js en el módulo mathOperations.js.

Importar un módulo exportado como predeterminado

Si la función addTwo se exportó como predeterminada, podemos importarla de la siguiente manera:

JavaScript Copiar código 
 
                            
import addTwo from "./addTwo";

// El resto del código de mathOperations.js va aquí

Importar un módulo exportado con nombre

Si la función addTwo se exportó con nombre, utilizamos una sintaxis diferente para importarla:

JavaScript Copiar código 
 
                            
import { addTwo } from "./addTwo";

// El resto del código de mathOperations.js va aquí

Los ejemplos mostrados anteriormente sirven para que sea más fácil comprender lo que está sucediendo en las aplicaciones React que estaremos construyendo más adelante.

Debemos tener en cuenta que hay muchas más reglas e implementaciones de trabajo con módulos en JavaScript.


La intención es familiarizarnos con la sintaxis más común utilizada, pero esto no puede considerarse una visión completa de cómo funcionan módulos en JavaScript.


Entorno de trabajo con Node y NPM


React es una biblioteca de JavaScript desarrollada por Facebook que permite la creación de interfaces de usuario de una manera eficiente y fácil de mantener. Para trabajar con React y aprovechar al máximo sus características, es necesario instalar Node y NPM en la computadora.

Node.js

Node.js es un entorno de tiempo de ejecución de JavaScript que permite ejecutar código JavaScript fuera del navegador. Aunque React está diseñado principalmente para el desarrollo web en el lado del cliente, la adopción de Node.js ofrece varios beneficios para el desarrollo de aplicaciones React, como:

Entorno de desarrollo unificado.
Node.js proporciona un entorno de desarrollo unificado en el que tanto el código del lado del cliente como del lado del servidor se pueden desarrollar y probar utilizando JavaScript. Esto permite a los desarrolladores de React mantener una base de código coherente y simplificar el proceso de desarrollo.

Herramientas de desarrollo y construcción.
El ecosistema de Node.js incluye una gran cantidad de herramientas y bibliotecas que facilitan el desarrollo y la construcción de aplicaciones React. Herramientas como Webpack, Babel y ESLint son ampliamente utilizadas en el desarrollo de aplicaciones React para la optimización del código, la transpilación de JavaScript moderno a una versión compatible con navegadores más antiguos y la aplicación de buenas prácticas de codificación.


NPM (Node Package Manager)


NPM es el gestor de paquetes predeterminado para Node.js y es una herramienta esencial para trabajar con React. A continuación, se detallan algunas razones por las que es necesario instalar NPM al trabajar con React:

Gestión de dependencias
React tiene varias dependencias, como bibliotecas adicionales y complementos, que son necesarios para desarrollar aplicaciones React de manera efectiva. NPM facilita la instalación y administración de estas dependencias, garantizando que las versiones correctas estén instaladas y que las dependencias estén actualizadas.

Creación y administración de proyectos
NPM proporciona herramientas y utilidades que facilitan la creación y administración de proyectos React. Por ejemplo, create-react-app es una herramienta popular provista por NPM que permite a los desarrolladores configurar rápidamente un entorno de desarrollo React con una estructura de proyecto estándar y una serie de herramientas preconfiguradas.

Scripts personalizados y automatización de tareas


NPM permite a los desarrolladores de React crear scripts personalizados y automatizar tareas comunes, como la construcción, prueba y despliegue de aplicaciones. Estos scripts pueden ser fácilmente configurados en el archivo package.json del proyecto, lo que facilita la automatización del flujo de trabajo de desarrollo.

Resumen

Instalar Node y NPM es fundamental para trabajar con React, ya que proporcionan un entorno de desarrollo unificado y eficiente, facilitan la gestión de dependencias y ofrecen herramientas para automatizar tareas comunes.

Al instalar Node y NPM en la computadora, podremos aprovechar al máximo las características y funcionalidades de React y desarrollar aplicaciones de manera eficiente y fácil de mantener.

React


React.js es una biblioteca de JavaScript desarrollada por ingenieros de Facebook.

¿Por qué React?

React ofrece varias ventajas sobre el desarrollo tradicional con HTML, CSS y JavaScript, como:

Componentización: React promueve la división de la interfaz de usuario en componentes individuales y reutilizables. Estos componentes pueden ser combinados y anidados para crear interfaces de usuario complejas y bien estructuradas.

Rendimiento: React utiliza un algoritmo de reconciliación eficiente y un "DOM virtual" para minimizar las actualizaciones costosas del DOM real en el navegador, lo que resulta en un mejor rendimiento y una experiencia de usuario más fluida.

Ecosistema y comunidad: React cuenta con una amplia comunidad de desarrolladores y un ecosistema de herramientas, bibliotecas y componentes de terceros que facilitan el desarrollo de aplicaciones web robustas y escalables.

Nota

Conceptos Básicos de React


Veamos algunos conceptos esenciales que necesitamos para comenzar a trabajar con React (aunque los veremos en profundidad más adelante):

Componentes

Los componentes son bloques de construcción básicos en React.

Resumen

Cada componente representa una parte de la interfaz de usuario y se encarga de su propia representación y lógica.

Los componentes pueden ser creados como funciones o clases, aunque las funciones son el enfoque preferido en versiones más recientes de React.

Un componente puede manejar su propio estado y podemos combinar componentes como bloques de construcción para formar interfaces de usuario más complejas.

Un componente de React es como un bloque de construcción que podemos combinar con otros componentes para crear la aplicación web que diseñes.

JSX

JSX es una extensión de sintaxis para JavaScript que permite escribir código HTML directamente dentro de nuestro código JavaScript. Aunque no es obligatorio, JSX hace que el código de React sea más legible y fácil de escribir. Los elementos JSX se convierten en llamadas a funciones de React durante la compilación.


Propiedades (props)


En React, la información que controla la apariencia y el comportamiento de un componente se divide en "estado" y "propiedades".

Las propiedades son datos inmutables que se pasan de un componente padre a un componente hijo. Las propiedades permiten la comunicación y el intercambio de datos entre componentes.

Definición: Props

Es una abreviación de "propiedades." En React, si un componente es renderizado (creado) por otro componente, puede recibir valores del componente que lo crea.

Enviar props a un componente es como enviarle un "mensaje" pero este mensaje consiste en distintos valores que puede usar para personalizar cómo se verá el componente y cómo funcionará dentro de la aplicación.

Nota

Props

Solo pueden ser enviados del componente "padre" al componente "hijo", no del componente "hijo" al componente "padre".


Estado y Ciclo de vida (componentes)


Podemos definir dos cualidades que tienen los componentes de React: Estados y ciclos de vida.

Estado

El estado es una representación de la información local y mutable de un componente.

Resumen

El estado se utiliza para almacenar datos que pueden cambiar a lo largo del tiempo o en respuesta a eventos del usuario.

El estado de un componente es una representación de su conjunto de propiedades y sus valores actuales en un momento en particular de la ejecución de la aplicación. Los componentes pueden manejar y actualizar su estado internamente.

Ciclo de vida de los componentes

Los componentes en React tienen un ciclo de vida que se refiere a las diferentes etapas por las que pasa un componente desde su creación hasta su eliminación.

Estas etapas del componente incluyen:

Creación (montaje).

Actualización.

Eliminación (desmontaje).

React proporciona métodos especiales, llamados métodos del ciclo de vida.

JSX


El código JSX se asemeja mucho al HTML, lo que lo hace más fácil de entender para aquellos que tienen experiencia en la creación de páginas web.

Definición: JSX

Es una extensión de sintaxis para JavaScript que fue creada específicamente para ser utilizada con React.

Si un archivo JavaScript contiene código JSX, entonces ese archivo deberá ser compilado. Esto significa que antes de que el archivo llegue a un navegador web, un compilador de JSX traducirá cualquier JSX en JavaScript normal.

¿Qué significa?

Extensión de sintaxis

En este caso, significa que JSX no es JavaScript válido. Los navegadores web no pueden leerlo directamente. Si un archivo JavaScript contiene código JSX, entonces ese archivo deberá ser compilado.

Por lo tanto, es necesario convertir el código JSX en JavaScript "normal" para que los navegadores puedan interpretarlo y mostrar la interfaz de usuario correctamente. Este proceso se conoce como transpilación.

Definición: Transpilador

Es una herramienta que toma un fragmento de código y lo transforma en otro código. En el caso de React, el transpilador más popular es Babel, que convierte el código JSX en código JavaScript que los navegadores pueden entender.

Vamos a explorar un ejemplo de cómo Babel transpila el código JSX en JavaScript.

Código JSX original:

JavaScript Copiar código 
 
                            
function Heading(props) {

    return <h1>{props.title}</h1>;

}

Código JavaScript transpilado:

JavaScript Copiar código 
 
                            
"use strict";

function Heading(props) {

    return /*#__PURE__*/ React.createElement("h1", null, props.title);

}

Como podemos observar, el código JSX se transforma en una llamada a la función React.create()Element(). Esta función crea un elemento de React, que se renderizará en el navegador como un elemento HTML.


Introducción a JSX


El siguiente fragmento de código puede resultar un tanto extraño al principio:

JavaScript Copiar código 
 
                            
const h1 = <h1>Hello world</h1>;

¿Qué tipo de código es ese? ¿Es JavaScript, HTML o alguna otra cosa?

A simple vista, parece ser código JavaScript, ya que comienza con const y termina con ; Sin embargo, si se intentara ejecutar ese código en un archivo HTML, no funcionaría.

Resumen

En JSX, los elementos de la interfaz de usuario se escriben dentro de etiquetas como si fueran HTML, pero en realidad son objetos de JavaScript.


Por otro lado, el código también contiene <h1>Hello world</h1>, que luce exactamente como HTML. Pero si se tratara de ejecutarlo en un archivo JavaScript, esa parte no funcionará.

Entonces, ¿qué sucede? La respuesta es JSX, una extensión de sintaxis que combina JavaScript y HTML en una sola sintaxis.


En el caso de esta línea de código, la variable h1 se está definiendo como un objeto que contiene un elemento <h1> con el texto "Hello world" como su contenido. Esta sintaxis puede parecer extraña al principio, pero una vez que se entiende la lógica detrás de JSX, se vuelve más intuitiva y fácil de usar en el desarrollo de aplicaciones web en React.


Elementos JSX (1)


Un ejemplo de un elemento JSX es:

JavaScript Copiar código 
 
                            
<h1>Hola mundo</h1>;

Este elemento JSX se ve exactamente como HTML. La única diferencia notoria es que encontrarás este código en un archivo JavaScript en lugar de en un archivo HTML.

En JSX, los elementos se escriben de manera similar a como lo haríamos con HTML, pero con algunas diferencias.

Atención

Elementos JSX

Deben estar envueltos en etiquetas de apertura y cierre, y los nombres de las etiquetas deben comenzar con una letra mayúscula.

Además, los atributos en JSX se escriben en camelCase en lugar de kebab-case, que es común en HTML.

Es importante recordar que los elementos JSX no son lo mismo que los elementos DOM.

Importante

Elemento JSX en React

Cuando se renderiza un elemento JSX en React, se crea un nuevo objeto de elemento React que luego se convierte en un árbol de elementos DOM para ser renderizado en el navegador.


Elementos JSX (2)


Los elementos JSX se parecen mucho a HTML, pero en realidad son tratados como expresiones de JavaScript. Es decir, pueden ser utilizados en cualquier lugar donde una expresión de JavaScript pueda ser utilizada.

Un elemento JSX puede ser almacenado en una variable, pasado como argumento a una función, almacenado en un objeto o array, o utilizado de cualquier otra manera que se nos ocurra.


Por ejemplo, podemos guardar un elemento JSX en una variable de esta manera:

JavaScript Copiar código

También podemos almacenar varios elementos JSX en un objeto, como en este caso:

JavaScript Copiar código 
 
                            
const miEquipo = {

    center: <li>Benzo Walli</li>,

    powerForward: <li>Rasha Loa</li>,

    smallForward: <li>Tayshaun Dasmoto</li>,

    shootingGuard: <li>Colmar Cumberbatch</li>,

    pointGuard: <li>Femi Billon</li>

};

Atención

Elementos JSX

Aunque se parecen a HTML, en realidad son expresiones de JavaScript que deben ser compiladas antes de ser ejecutadas en un navegador web.

De esta manera, podemos utilizar los elementos JSX para construir interfaces de usuario dinámicas e interactivas en React.


Tipos de notación


Los tipos de notación son diferentes convenciones o estilos que se utilizan para escribir nombres de variables, funciones, atributos, y otros elementos en el código de programación.

Nota

CamelCase y kebab-case

Son 2 tipos de notación para escribir nombres de variables, atributos y funciones en diferentes lenguajes de programación.

Los tipos de notación son importantes porque ayudan a que el código sea más legible y fácil de entender para otros desarrolladores.

Algunos tipos de notación conocidos son:

camelCase: Es un estilo de escritura en el que se eliminan los espacios y se escriben las palabras juntas, comenzando cada palabra con una letra mayúscula a excepción de la primera palabra. Por ejemplo, "miVariable" o "miFuncion". CamelCase se utiliza comúnmente en JavaScript, Java, C#, y otros lenguajes de programación.

kebab-case: Es un estilo de escritura en el que se escriben las palabras separadas por guiones. Por ejemplo, "mi-variable" o "mi-atributo". Kebab-case se utiliza comúnmente en HTML, CSS, y otros lenguajes de marcado.

snake_case: Es un estilo de escritura en el que se eliminan los espacios y se escriben las palabras juntas, separando cada palabra con un guión bajo. Por ejemplo, "mi_variable" o "mi_funcion". Snake_case se utiliza comúnmente en Python y otros lenguajes de programación.

PascalCase: Es similar a CamelCase, pero la primera letra de cada palabra se escribe en mayúscula. Por ejemplo, "MiVariable" o "MiFuncion". PascalCase se utiliza comúnmente en C# y otros lenguajes de programación.

Es importante conocer y seguir estas convenciones para escribir un código más legible y fácil de entender para otros desarrolladores.


Cada tipo de notación tiene sus propias convenciones y usos comunes en diferentes lenguajes de programación y tecnologías.

Al igual que los elementos HTML, los elementos JSX también pueden tener atributos que les proporcionan información adicional.

Resumen

Los atributos JSX se escriben utilizando una sintaxis similar a la de HTML: un nombre, seguido de un signo igual y un valor, que debe estar entre comillas.

JavaScript Copiar código 
 
                            
nombre-de-mi-atributo="valor-de-mi-atributo"

Aquí hay algunos ejemplos de elementos JSX con atributos:

<a href="http://www.example.com">Bienvenido a la Web</a>;

const titulo = <h1 id="titulo">Introducción a React.js: Parte I</h1>;

Un solo elemento JSX puede tener muchos atributos, al igual que en HTML:

Los atributos JSX son muy útiles para personalizar y dar estilo a los elementos de la interfaz de usuario.


Anidar elementos


En React, podemos anidar elementos JSX dentro de otros elementos JSX, de la misma forma que en HTML. Por ejemplo, podemos anidar un elemento JSX <h1> dentro de un elemento JSX <a>:

JavaScript Copiar código 
 
                            
<a href="https://www.ejemplo.com">

    <h1>Haz clic aquí</h1>

</a>;

Si una expresión JSX ocupa más de una línea, debemos envolver la expresión JSX de varias líneas entre paréntesis. Esto puede parecer extraño al principio, pero se acostumbrara a ello:

JavaScript Copiar código 
 
                            
(

    <a href="https://www.ejemplo.com">

        <h1>

            Haz clic aquí

        </h1>

    </a>

)

Cuando escribimos JSX en varias líneas, es necesario colocar los paréntesis para que la expresión sea interpretada correctamente por React.

Sin los paréntesis, el compilador de JSX puede confundirse y generar errores en la sintaxis. Los paréntesis le indican a React que se trata de una expresión JSX y que la misma ocupa varias líneas.


Las expresiones JSX anidadas se pueden guardar en variables, pasarse a funciones, etc., al igual que las expresiones JSX no anidadas. Aquí vemos un ejemplo de una expresión JSX anidada que se guarda en una variable:

JavaScript Copiar código 
 
                            
const elEjemplo = (

    <a href="https://www.ejemplo.com">

        <h1>Haz clic aquí</h1>

    </a>

);

En React, cuando se utiliza JSX, existe una regla que se debe tener en cuenta: una expresión JSX debe tener exactamente un elemento JSX.

Si queremos incluir varios elementos dentro de una sola expresión JSX, debemos envolverlos en un elemento padre.


Por ejemplo, el siguiente código funcionará:

JavaScript Copiar código 
 
                            
const parrafos = (

    <div id="soy-el-elemento-más-exterior">

        <p>Soy un párrafo.</p>

        <p>Yo también soy un párrafo.</p>

    </div>

);

Pero este código no funcionará:

JavaScript Copiar código 
 
                            
const parrafos = (

    <p>Soy un párrafo.</p>

    <p>Yo también soy un párrafo.</p>

);

Esto se debe a que la etiqueta de apertura y la etiqueta de cierre final de una expresión JSX deben pertenecer al mismo elemento JSX.

Es fácil olvidarse de esta regla y terminar con errores difíciles de diagnosticar.

Atención

Si nos damos cuenta de que una expresión JSX tiene varios elementos, la solución suele ser simple: envolver la expresión JSX en un elemento padre, como <div></div>.

También es posible envolver expresiones JSX de varias líneas en un elemento distinto a un <div>. En lugar de usar un <div>, se puede usar cualquier elemento válido de HTML, como <section>, <article>, <main>, <header>, <footer>, entre otros.

Además, también se puede usar <></>. Los símbolos  <> y </> se utilizan en React para crear un componente sin necesidad de envolverlo en una etiqueta específica como <div>. Estos símbolos se conocen como fragmentos de React o React fragments.

Objetivo: Fragmento de React

Se usa cuando se necesita devolver varios elementos en un solo bloque, pero no se quiere agregar un elemento extra como <div> en el DOM.

Usando los fragmentos de React, podemos devolver varios elementos anidados sin envolverlos en un elemento padre.

JavaScript Copiar código 
 
                            
const parrafos = (

    <>

        <p>Soy un párrafo.</p>

        <p>Yo también soy un párrafo.</p>

    </>

);

El Virtual DOM


En el desarrollo web interactivo, la manipulación del DOM es esencial, pero también es muy lenta en comparación con la mayoría de las operaciones de JavaScript. Los frameworks de JavaScript, a menudo actualizan el DOM más de lo necesario, lo que resulta en una ineficiencia considerable.

Para resolver este problema, React introdujo el concepto de virtual DOM.

Definición: Virtual DOM

Es una representación liviana de un objeto DOM, y tiene las mismas propiedades que un objeto real DOM.

Manipular el virtual DOM es mucho más rápido que manipular el DOM real, ya que no se dibuja en la pantalla. React compara el virtual DOM actualizado con una instantánea del virtual DOM antes de la actualización y actualiza sólo los objetos que han cambiado en el DOM real.

React actualiza solo las partes necesarias del DOM, lo que lo hace mucho más eficiente. La reputación de React por su rendimiento se debe en gran parte a esta innovación.


Cuando usamos ReactDOM.render(), solo se actualizan los elementos que han cambiado desde la última vez que se renderizaron. Esto significa que si renderizamos dos veces exactamente lo mismo, la segunda renderización no hará nada. Esto es gracias a esta característica especial llamada Virtual DOM.

El Virtual DOM es una representación virtual de los elementos del DOM.

React actualiza primero esta representación virtual en lugar del DOM real. Luego, React compara la nueva representación virtual con la anterior y actualiza sólo los elementos que han cambiado en el DOM real.


Esto reduce el costo computacional de actualizar los elementos del DOM, lo que hace que la aplicación sea más rápida y eficiente. La idea de usar una representación virtual de los elementos del DOM no es nueva, pero React lo ha popularizado en el mundo de las aplicaciones web modernas.

Otro aspecto importante en JSX es que no se puede usar la palabra "class" para asignar una clase a un elemento, ya que esta palabra está reservada en JavaScript. En su lugar, se debe usar "className". Estos detalles pueden parecer triviales, pero son importantes para evitar errores y hacer que el código sea más legible y fácil de mantener.

La palabra this

En React, es común encontrarse con la palabra clave "this". Especialmente, se utiliza dentro del cuerpo de la declaración de una clase de componente. Por ejemplo:

¿Qué significa "this" en el código anterior? La respuesta sencilla es que "this" se refiere a una instancia de la clase Heladeria. La respuesta un poco más compleja es que "this" se refiere al objeto en el que se llama el método que encierra "this", en este caso ".render()". Es casi inevitable que este objeto sea una instancia de Heladeria, pero técnicamente podría ser otra cosa.

Ejemplo

Suponiendo que "this" se refiere a una instancia de la clase Heladeria, la expresión "this.helado" se evalúa a la llamada del método ".helado" de Heladeria. Este método, a su vez, se evalúa a la cadena de texto "chocolate".


¿Por qué no necesitamos paréntesis después de "this.helado"? ¿No debería ser "this.helado()"? No se necesitan los paréntesis porque ".helado" es un método getter. Se puede saber esto gracias al "get" en el cuerpo de la declaración de la clase anterior.

Los componentes React a menudo necesitarán información dinámica para renderizarse.

    }

}

Evitar bucles infinitos en el renderizado es fundamental para mantener la eficiencia y la estabilidad de la aplicación.


En lugar de cambiar el estado directamente en el método de renderizado, es recomendable utilizar métodos de ciclo de vida o eventos, como onClick o onChange, para modificar el estado del componente según sea necesario.


Cambios en React 18: ReactDOM.createRoot


A partir de React 18 se han realizado algunos cambios en la librería para optimizar el render e hidratación de aplicaciones o componentes de React en el DOM. El más importante, crucial y significativo fue la migración de ReactDOM.render a ReactDOM.createRoot(). A continuación eremos algunos ejemplos de este cambios:

Antes de React 18:

JavaScript Copiar código 
 
                            
const root = document.getElementById("root");

ReactDOM.render(e(LikeButton), root);

A partir de React 18:

JavaScript Copiar código 
 
                            
const rootElement = document.getElementById("root");

const root = ReactDom.createRoot(rootElement);

root.render(e(LikeButton));

Antes de React 18:

JavaScript Copiar código 
 
                            
ReactDOM.render(

    <React.StrictMode>

        <App />

    </React.StrictMode>,

    document.getElementById("root")

);

A partir de React 18:

JavaScript Copiar código 
 
                            
const rootElement = document.getElementById("root");

const root = ReactDom.createRoot(rootElement);

root.render(

    <React.StrictMode>

        <App />

    </React.StrictMode>

);
