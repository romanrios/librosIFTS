Presentación 


En esta sección vamos a conocer el proceso de despliegue de aplicaciones React, las diferentes etapas y herramientas involucradas. Veremos las diferencias entre los entornos de desarrollo y producción, y cómo preparar un proyecto React para el despliegue. Además, nos adentraremos en el mundo de Git, explorando cómo crear y gestionar repositorios en GitHub, así como la clonación y el trabajo con ramas. 

También discutiremos las opciones de alojamiento disponibles, incluyendo los servicios PaaS, servidores propios y los sistemas operativos y servidores web más comunes. Nos centraremos en diferentes plataformas y servicios en la nube, como Netlify, Vercel y Firebase Hosting, explicando cómo desplegar aplicaciones React en cada uno de ellos y analizando sus ventajas y características específicas. Además, cubriremos el despliegue de aplicaciones React en un servidor propio para conocer las diferencias con las plataformas PaaS.

Despliegue de aplicaciones


Siempre que desarrollamos una aplicación, ya sea estática o dinámica, únicamente va a vivir en nuestras computadoras (localhost). Para poder compartirla con el mundo exterior, debemos desplegarla en un servidor y compartir la url de este.

Definición: Despliegue de aplicaciones web

Es el proceso de poner en producción una aplicación web para que esté disponible para los usuarios. Esto implica tomar el código fuente de la aplicación y hacerlo disponible en un servidor para que los usuarios puedan acceder a él a través de una URL.

Un deployment es una instancia específica de una aplicación web que se ejecuta en un servidor en particular. Cuando creamos una nueva versión de la aplicación, se crea un nuevo deployment, que puede estar en el mismo servidor o en uno diferente.


Etapas del despliegue


El despliegue de aplicaciones web implica varias etapas, que incluyen:

Compilación: Este es el proceso de convertir el código fuente de la aplicación en un paquete que se puede ejecutar en un servidor.

Pruebas: Las pruebas son importantes para asegurarnos de que la aplicación funcione como se espera antes de ponerla en producción. Esto incluye pruebas de funcionalidad, seguridad y rendimiento.

Empaquetado: Una vez que la aplicación ha sido probada y aprobada, se empaqueta en un archivo que se puede cargar en un servidor.

Despliegue: El archivo de la aplicación empaquetada se carga en un servidor y se configura para que se ejecute como una aplicación web.

El proceso de despliegue consta de varias etapas, cada una con su propia serie de tareas.


A continuación, explicaremos cada etapa:

Preparación del entorno de producción: Antes de desplegar la aplicación, se necesita configurar el entorno de producción. Esto incluye la creación del servidor web, la instalación de las herramientas necesarias y la configuración de la base de datos.

Despliegue de la aplicación: Una vez que se ha preparado el entorno de producción, podemos proceder a subir el código de la aplicación al servidor. Existen varias maneras de hacer esto, como subir los archivos manualmente, usar FTP, o usar una herramienta de automatización.

Pruebas de la aplicación: Antes de que la aplicación esté disponible para los usuarios finales, es importante realizar pruebas para asegurarse de que todo funciona correctamente. Esto incluye pruebas de funcionalidad, pruebas de rendimiento y pruebas de seguridad.

Configuración del dominio y DNS: Una vez que la aplicación ha sido desplegada y probada, necesitamos configurar el dominio y el DNS para que el sitio web esté disponible públicamente. Esto incluye la creación de registros DNS y la asignación del dominio al servidor web.

Monitoreo: Es importante monitorear la aplicación para asegurarnos de que esté funcionando correctamente y de que los usuarios puedan acceder a ella.


DevOps


Esta metodología se ha vuelto cada vez más importante en el despliegue de aplicaciones web, ya que permite que los equipos trabajen juntos para garantizar que la aplicación se entregue de manera eficiente y confiable.

Definición: DevOps

Es una metodología que se centra en la colaboración y la comunicación entre los equipos de desarrollo y operaciones.

La nube proporciona escalabilidad y flexibilidad, lo que significa que las aplicaciones pueden escalar automáticamente según la demanda, lo que asegura que estén disponibles para los usuarios en todo momento.

Definición: La nube

Es una plataforma que permite a las empresas alojar sus aplicaciones web en servidores remotos, lo que puede ser beneficioso para el despliegue de aplicaciones.


Desarrollo vs Producción


Es importante distinguir entre el entorno de desarrollo y el de producción.

Entorno de desarrollo: Es donde los desarrolladores crean y prueban la aplicación.

Entorno de producción: Es donde la aplicación se ejecuta y está disponible para los usuarios finales.

En la etapa de producción, la aplicación se ejecutará en un servidor web en vivo y estará disponible para los usuarios finales.

Otra diferencia clave es la configuración del entorno.

Entorno de desarrollo: La configuración se puede cambiar y probar sin afectar a los usuarios finales.

Etapa de producción: Cualquier cambio en la configuración puede tener un impacto en la aplicación y en los usuarios finales.

Es importante asegurarnos de que la aplicación se haya probado adecuadamente en el entorno de desarrollo antes de ponerla en producción para garantizar que los usuarios tengan una experiencia satisfactoria.


Entorno de desarrollo


Un entorno de desarrollo se refiere al conjunto de herramientas, servicios y configuraciones que se utilizan para construir, probar y depurar aplicaciones de software antes de que se implementen en un entorno de producción.

Los entornos de desarrollo también se conocen como "entornos de prueba" o "entornos de preproducción".


En un entorno de desarrollo:

Los desarrolladores pueden tener acceso a herramientas de depuración y pruebas que no están disponibles en el entorno de producción.

Los desarrolladores también pueden usar herramientas como sistemas de control de versiones para rastrear cambios en el código fuente y coordinar el trabajo en equipo.

Los desarrolladores pueden crear, editar y probar su código antes de que se envíe al entorno de producción. Esto permite a los desarrolladores detectar y solucionar problemas antes de que la aplicación se lance en producción.

Los desarrolladores pueden simular situaciones de producción para probar el rendimiento y la escalabilidad de la aplicación.

Una forma común de crear un entorno de desarrollo es utilizando una herramienta de virtualización como VirtualBox, Vagrant o Docker.


Estas herramientas permiten crear un entorno de desarrollo aislado en el que se pueden instalar todas las dependencias necesarias para el proyecto, como librerías y bases de datos.

Importante

Entorno de desarrollo

Debe ser lo más similar posible al entorno de producción para evitar sorpresas una vez que la aplicación se despliega en producción. Para ello, es recomendable utilizar las mismas versiones de las herramientas y librerías en ambos entornos.


Variables de entorno


En el desarrollo de aplicaciones React, las variables de entorno se utilizan para definir valores que pueden variar dependiendo del entorno de ejecución, como por ejemplo la URL de la API, las credenciales de acceso a la base de datos o las claves de autenticación (API Key).

Definición: Variables de entorno

Son valores que se almacenan en el sistema operativo y que pueden ser accedidos por cualquier programa que se ejecute en él. Estas variables se utilizan para configurar y personalizar el comportamiento de la aplicación en tiempo de ejecución.

Existen varias formas de definir las variables de entorno en una aplicación React. La forma más común es a través del archivo .env. En este archivo, se definen las variables de entorno en formato NOMBRE_VARIABLE=VALOR_VARIABLE.

Para acceder a las variables de entorno en el código de la aplicación, se utiliza el objeto process.env.

Ejemplo

Si queremos acceder al valor de la variable de entorno API_URL, podemos hacerlo de la siguiente forma

JavaScript Copiar código 
 
                            
const apiUrl = process.env.API_URL;

Un ejemplo de variable de entorno podría ser la API KEY necesaria para utilizar la API de OMDB (https://www.omdbapi.com/) que utilizamos en secciones anteriores, la cual nos permite buscar información sobre películas. Para utilizar esta API, primero debemos registrarnos en su sitio web y obtener una clave de API.

Para proteger esta clave y evitar exponerla directamente en nuestro código fuente, podemos almacenarla como una variable de entorno en nuestro sistema operativo. De esta forma, podemos acceder a la clave de API en tiempo de ejecución desde nuestro código sin necesidad de escribirla directamente en nuestro código fuente.

Ejemplo

En una aplicación de React que utiliza la API de OMDB, podríamos tener el siguiente código:

JavaScript Copiar código 
 
                            
const API_KEY = process.env.REACT_APP_OMDB_API_KEY;

function searchMovies(query) {

  return fetch(`https://www.omdbapi.com/?apikey=${API_KEY}&s=${query}`)

    .then((response) => response.json())

    .then((data) => data.Search);

}

En este código, utilizamos la variable de entorno REACT_APP_OMDB_API_KEY para almacenar nuestra clave de API de OMDB.


La convención REACT_APP_.

Es una práctica recomendada para nombres de variables de entorno en aplicaciones React, ya que asegura que estas variables sean accesibles desde nuestro código sin necesidad de configuraciones adicionales.


De esta forma, podemos proteger nuestra clave de API de OMDB y asegurarnos de que no se exponga accidentalmente en nuestro código fuente. Además, si necesitamos cambiar la clave de API en el futuro, solo tendremos que actualizar la variable de entorno sin necesidad de editar nuestro código fuente.


Preparación para el despliegue


Antes de desplegar nuestra aplicación en un servidor o en la nube, debemos prepararla para el despliegue. Esto incluye optimizar el código, configurar el entorno, ajustar los ajustes de seguridad y crear una versión de producción de la aplicación que esté lista para su implementación.

Build

Para ello lo que normalmente hacemos es generar el build de la aplicación ejecutando el siguiente comando:

CLI Copiar código 
 
                            
npm run build

Cuando ejecutamos este comando, React compila los archivos JavaScript y los optimiza para que puedan ser entregados de manera más rápida al navegador.

Definición: Npm run build

Es una tarea de construcción de la aplicación en React que permite crear una versión optimizada y lista para producción de la aplicación.


También se generan los archivos HTML, CSS y otros archivos necesarios para que la aplicación se ejecute.

Importante

El navegador solo trabaja con JavaScript. Por lo tanto, todas las sintaxis JSX deben ser compiladas a código JavaScript compatible con la mayoría de navegadores.


Cuando se ejecuta npm run build, se realizan una serie de pasos.

1

Se eliminan todos los archivos de construcción anteriores para que la nueva versión no tenga conflictos con los archivos viejos.

2

Se compila y se optimizan los archivos de JavaScript, CSS y HTML.

3

Se crean los archivos necesarios para cargar los recursos de la aplicación, como imágenes y fuentes.

4

Se generan los archivos de construcción en una carpeta llamada "build".


Al usar el comando npm run build, la aplicación estará lista para ser desplegada en un servidor web. De hecho, el archivo generado en la carpeta "build" es el que debe ser utilizado en el servidor de producción para que la aplicación se pueda ejecutar.


GIT


Git es un sistema de control de versiones distribuido y de código abierto creado por Linus Torvalds en 2005.

Definición: Sistema de control de versiones

Es una herramienta que facilita la administración y seguimiento de cambios en el código fuente de un proyecto de software a lo largo del tiempo.

Git es ampliamente utilizado por desarrolladores y equipos de desarrollo de software en todo el mundo debido a su eficiencia, facilidad de uso y características robustas.

Git permite a los desarrolladores trabajar de manera colaborativa en proyectos de software, manteniendo un historial completo y detallado de todos los cambios realizados en el código fuente.


Esto significa que los desarrolladores pueden regresar a versiones anteriores del código, ver qué cambios se hicieron y quién los hizo, y resolver conflictos cuando varios desarrolladores están trabajando en el mismo archivo al mismo tiempo.


Algunas de las características clave de Git incluyen:

Control de versiones distribuido: Git permite a los desarrolladores tener una copia completa del repositorio en su máquina local, lo que facilita el trabajo sin conexión y una colaboración más rápida y eficiente. Esto contrasta con los sistemas de control de versiones centralizados, donde los desarrolladores deben conectarse a un servidor central para acceder al historial completo del proyecto.

Ramificación (branching) y fusión (merging) eficientes: Git facilita la creación y gestión de ramas de desarrollo separadas, lo que permite a los desarrolladores trabajar en diferentes características o correcciones de errores sin afectar la estabilidad del código en la rama principal. La fusión de ramas en Git es también eficiente y, a menudo, se puede realizar automáticamente sin intervención manual.

Rendimiento y escalabilidad: Git ha sido diseñado para ser rápido y eficiente, incluso en proyectos de software grandes y complejos. Esto se debe en parte a su diseño distribuido y al uso de técnicas avanzadas de compresión y almacenamiento de datos.

Seguridad y integridad: Git utiliza criptografía para garantizar la integridad y autenticidad de los datos almacenados en el repositorio. Cada commit en Git tiene un identificador único basado en un hash criptográfico que garantiza que el historial del repositorio no pueda ser modificado sin ser detectado.

Flexibilidad y adaptabilidad: Git es altamente configurable y se puede adaptar para satisfacer las necesidades de diferentes flujos de trabajo y equipos de desarrollo. También es compatible con una amplia variedad de plataformas y sistemas operativos, lo que facilita su adopción en diferentes entornos de desarrollo.

Antes de profundizar en cómo utilizar Git para desplegar aplicaciones React, es importante repasar algunos conceptos clave:

Repositorio: Un repositorio Git es una colección de archivos y directorios que están bajo el control de versiones de Git. Un repositorio contiene un historial completo de todos los cambios realizados en los archivos a lo largo del tiempo.

Commit: Un commit es un conjunto de cambios en el código que se guardan en el repositorio. Cada commit tiene un identificador único y una descripción que explica el propósito del commit.

Rama (Branch): Una rama es una línea separada de desarrollo dentro de un repositorio. Las ramas nos permiten trabajar en diferentes características o correcciones de errores sin afectar la rama principal (generalmente llamada "main" o "master").


GIT: Crear un repositorio con GitHub


Crear un repositorio en GitHub

Debemos crear una cuenta en GitHub (si aún no la tenemos) y acceder a nuestro perfil. A continuación, seguimos estos pasos para crear un nuevo repositorio:

Hacemos clic en el símbolo "+" en la esquina superior derecha de la página y seleccionamos "New repository" (Nuevo repositorio).

Asignamos un nombre al repositorio, escribimos una descripción (opcional) y elegimos si queremos que sea público o privado.

Podemos inicializar el repositorio con un archivo README, pero esto es opcional.

Seleccionamos una licencia y/o un archivo .gitignore (también es opcional).

Hacemos clic en "Create repository" (Crear repositorio) para crear el nuevo repositorio.

Configurar Git en nuestra máquina local

Si no lo hemos hecho antes, instalamos Git en nuestra computadora y lo configuramos con nuestro nombre de usuario y dirección de correo electrónico:

CLI Copiar código 
 
                            
git config --global user.name "Tu nombre"

git config --global user.email "tuemail@example.com"

Inicializar un repositorio Git local

Nos dirigimos al directorio de nuestro proyecto React en nuestra máquina local y ejecutamos el siguiente comando para inicializar un repositorio Git:

CLI Copiar código 
 
                            
git init

Esto crea una carpeta oculta llamada .git en el directorio del proyecto, que contiene toda la información necesaria para el control de versiones.

Agregar archivos al área de preparación

Antes de hacer nuestro primer commit, debemos agregar los archivos al área de preparación de Git. Podemos agregar archivos individuales utilizando:

CLI Copiar código 
 
                            
git add nombre_del_archivo

O podemos agregar todos los archivos del directorio del proyecto usando:

CLI Copiar código 
 
                            
git add .

Una vez que hayamos agregado los archivos al área de preparación, creamos un commit con un mensaje descriptivo usando el siguiente comando:

CLI Copiar código 
 
                            
git commit -m "Un mensaje descriptivo"

Conectar el repositorio local con el remoto (GitHub)

Para vincular nuestro repositorio local con el repositorio remoto que creamos en GitHub, utilizamos el siguiente comando:

CLI Copiar código 
 
                            
git remote add origin https://github.com/tu_usuario/nombre_del_repositorio.git

Ahora ya podemos subir (push) los cambios a GitHub:

CLI Copiar código 
 
                            
git push -u origin main

Este comando sube los cambios en la rama principal (main) al repositorio remoto.

A partir de ahora, podemos seguir trabajando en nuestro repositorio local y subir cambios futuros usando simplemente:

CLI Copiar código 
 
                            
git push

GIT: Más acciones


Clonar un repositorio de Git

Para comenzar a trabajar con un repositorio Git que contiene una aplicación React, primero necesitamos clonarlo en nuestra máquina local. Esto se hace utilizando el comando git clone:

CLI Copiar código 
 
                            
git clone https://github.com/usuario/repositorio.git

Reemplazamos "usuario" y "repositorio" con la información correspondiente al repositorio que queremos clonar. Una vez clonado, podemos acceder al directorio del repositorio y comenzar a trabajar con la aplicación React.

Crear y cambiar de rama

Al desplegar nuestra aplicación React, es una buena práctica trabajar en una rama separada para mantener la estabilidad del código en la rama principal. Para crear una nueva rama y cambiar a ella, usamos los siguientes comandos:

CLI Copiar código 
 
                            
git checkout -b nombre_de_la_rama

Reemplazamos "nombre_de_la_rama" con un nombre descriptivo para la rama en la que queremos trabajar.

Realizar cambios y crear commits

Una vez que hayamos realizado cambios en nuestra aplicación React, podemos agregarlos al área de preparación utilizando el comando git add:

CLI Copiar código 
 
                            
git add .

El punto indica que queremos agregar todos los archivos modificados al área de preparación. Luego, creamos un commit con los cambios utilizando el comando git commit:

CLI Copiar código 
 
                            
git commit -m "Mensaje descriptivo del commit"

Integrar cambios en la rama principal

Cuando estemos listos para integrar los cambios de nuestra rama de trabajo en la rama principal, primero debemos cambiar a la rama principal:

CLI Copiar código 
 
                            
git checkout main

Luego, fusionamos nuestra rama de trabajo en la rama principal utilizando el comando git merge:

CLI Copiar código 
 
                            
git merge nombre_de_la_rama

Proveedor de alojamiento


Cuando desarrollamos aplicaciones React, una de las decisiones importantes que enfrentamos es dónde desplegar nuestra aplicación.

Existen servicios como Netlify o Vercel, que facilitan el despliegue de aplicaciones web y estáticas, así como la opción de utilizar nuestros propios servidores.


A continuación, compararemos las diferencias entre ambas opciones, enumerando ventajas y desventajas. Esto nos ayudará a tomar una decisión informada sobre la mejor opción para nuestras necesidades.


PaaS


Estos servicios se han popularizado en los últimos años debido a su facilidad de uso y características específicas para el desarrollo y despliegue de aplicaciones web.

PaaS

Plataformas como servicio

Proporcionan un entorno de desarrollo y despliegue en la nube, lo que permite a los desarrolladores centrarse en la creación de aplicaciones sin preocuparse por la infraestructura subyacente.

Algunas de sus ventajas y características incluyen:

Ventajas:


Fácil de usar: Desplegar una aplicación React en Netlify o Vercel es un proceso rápido y sencillo, que se integra directamente con repositorios de Git como GitHub o GitLab.

CDN (Content Delivery Network) integrado: Estos servicios ofrecen una CDN integrada, lo que mejora la velocidad de carga y la experiencia del usuario.

Certificados SSL gratuitos: Ambos servicios proporcionan certificados SSL gratuitos para nuestros dominios personalizados, lo que garantiza la seguridad y la privacidad de nuestros usuarios.

Integración con servicios de terceros: Netlify y Vercel ofrecen integraciones con servicios de terceros como formularios, funciones sin servidor (serverless) y autenticación, entre otros.

Desventajas


Limitaciones en los planes gratuitos: Aunque estas plataformas ofrecen planes gratuitos, tienen ciertas limitaciones en términos de recursos, funcionalidades y uso. Por ejemplo, suelen incluir límites de ancho de banda, límites en la cantidad de proyectos desplegados o restricciones en la personalización de la configuración. Para acceder a características y recursos adicionales, es posible que sea necesario actualizar a un plan pago, lo que podría aumentar los costos en función de nuestras necesidades.

Menos control: Al utilizar servicios de terceros, perdemos cierto grado de control sobre nuestra infraestructura, lo que puede ser un factor a tener en cuenta para proyectos que requieren configuraciones muy específicas.


Servidor propio


El despliegue en un servidor propio nos permite tener un control total sobre la infraestructura y la configuración de nuestra aplicación. Algunas ventajas y características incluyen:

Ventajas


Control total: Tener control completo sobre nuestro servidor nos permite personalizar y optimizar nuestra infraestructura según las necesidades específicas de nuestro proyecto.

Escalabilidad: Podemos escalar nuestros recursos según las necesidades de nuestra aplicación, sin depender de los límites impuestos por servicios de terceros.

Desventajas


Mayor complejidad: Desplegar y mantener nuestra propia infraestructura puede ser más complicado y requerir más tiempo y esfuerzo en comparación con el uso de servicios como Netlify o Vercel.

Costos de mantenimiento: Tendremos que asumir los costos de mantenimiento y actualización de nuestro servidor, así como el costo de los recursos necesarios para alojar nuestra aplicación.

Gestión de seguridad: Seremos responsables de mantener la seguridad de nuestro servidor y aplicaciones, lo que incluye la actualización de software, la monitorización y la gestión de posibles amenazas.


Al utilizar un servidor propio para desplegar aplicaciones React, hay varias alternativas y opciones populares que podemos considerar. Estas opciones varían en términos de sistemas operativos, servidores web y servicios en la nube.

A continuación, se presentan algunas de las más conocidas:

Sistemas operativos


Ubuntu: Es una distribución popular de Linux que es ampliamente utilizada en servidores debido a su estabilidad y facilidad de uso. Ubuntu Server es una opción sólida y confiable para alojar aplicaciones React.

CentOS: Es otra distribución de Linux muy popular en entornos de servidores, especialmente para aquellos que buscan un entorno similar a Red Hat Enterprise Linux sin el costo asociado.

Servidores web


Nginx: Es un servidor web de alto rendimiento y proxy inverso que se utiliza comúnmente para servir aplicaciones React. Es conocido por su eficiencia, escalabilidad y fácil configuración.

Apache: Es un servidor web de código abierto ampliamente utilizado y muy versátil. Aunque es más conocido por servir sitios web dinámicos basados en PHP, también puede ser configurado para servir aplicaciones React.

Servicios en la nube


Amazon Web Services (AWS): Es una plataforma de servicios en la nube que ofrece una amplia variedad de servicios, incluidos servidores virtuales (EC2), almacenamiento (S3) y bases de datos (RDS). Podemos configurar una instancia EC2 con el sistema operativo y el servidor web de nuestra elección para desplegar nuestra aplicación React.

Google Cloud Platform (GCP): Es otra plataforma en la nube que proporciona una amplia gama de servicios para alojar y desplegar aplicaciones. Podemos utilizar Google Compute Engine para configurar una máquina virtual con el sistema operativo y el servidor web que prefiramos para desplegar nuestra aplicación React.

Microsoft Azure: Es una plataforma en la nube de Microsoft que ofrece una gran cantidad de servicios y soluciones para alojar aplicaciones. Podemos utilizar máquinas virtuales de Azure para configurar nuestro servidor con el sistema operativo y el servidor web que necesitemos para desplegar nuestra aplicación React.

Netlify


Es especialmente adecuado para proyectos pequeños a medianos y aplicaciones de una sola página (SPA).

Definición: Netlify

Es una plataforma de alojamiento que se centra en la simplicidad y la facilidad de uso. Ofrece integración continua, despliegue automático desde repositorios Git y una generosa capa gratuita.

Netlify ofrece toda una serie de características que no encontraremos en otros servicios para sitios estáticos como Firebase o Github Pages:

Despliegue continuo: Permite vincular nuestro sitio web con un proyecto Git, con el objetivo de poder actualizar automáticamente el primero tras cada actualización que llevemos a cabo en el repositorio.

Rollbacks: Si un cambio en el repositorio ha causado algún efecto indeseado en nuestro sitio web, siempre podremos hacer retroceder éste a una versión anterior de sí mismo con un único clic.

DNS: Permite administrar zonas dentro del dominio netlify.com, así como usar dominios personalizados que hayamos adquirido.

HTTPS: Disponible de forma automática para todas las zonas gestionadas por Netlify.

Netlify: Extras


Implementar un dominio propio

Para utilizar un dominio propio en Netlify, es decir, uno que hayamos comprado, debemos seguir los siguientes pasos:

1

Comprar un dominio: Necesitamos adquirir un dominio de un registrador de dominios de nuestra elección, como Namecheap, GoDaddy, Google Domains, entre otros. Debemos asegurarnos de tener acceso a la configuración de DNS del dominio.

2

Configurar el dominio en Netlify: Iniciamos sesión en nuestra cuenta de Netlify y vamos al panel de control del sitio. Hacemos clic en "Domain settings" y luego en "Add custom domain". Ingresamos el dominio personalizado en el campo proporcionado y hacemos clic en "Verify". Netlify verificará si el dominio está disponible y si ya lo hemos comprado. Si todo está en orden, hacemos clic en "Yes, add domain" para continuar.

3

Actualizar la configuración de DNS en el registrador de dominios: Netlify nos proporcionará una lista de registros DNS que debemos agregar o actualizar en nuestro registrador de dominios. Estos registros pueden incluir registros A, CNAME y/o registros TXT para la verificación de la propiedad del dominio.

Nos dirigimos al panel de control del registrador de dominios y accedemos a la sección de administración de DNS. Actualizamos o agregamos los registros según lo indicado por Netlify. Los cambios en los registros DNS pueden tardar algún tiempo en propagarse en todo el mundo; esto podría variar desde unos minutos hasta 48 horas o más.

4

Configurar el dominio principal y el redireccionamiento (opcional): Si deseamos configurar un dominio principal (por ejemplo, "example.com" en lugar de "www.example.com"), vamos a la pestaña "Domain settings" en Netlify y seleccionamos el dominio que queremos establecer como principal. Hacemos clic en el botón "Options" y elegimos "Set as primary domain". También podemos configurar redireccionamientos automáticos desde el dominio no principal al principal, lo que mejorará la experiencia del usuario y evitará problemas de contenido duplicado en términos de SEO.

5

Esperar la propagación de DNS y verificar la configuración: Después de haber actualizado los registros DNS en el registrador de dominios, debemos esperar a que se propaguen completamente. Una vez que esto suceda, la aplicación en Netlify estará disponible en nuestro dominio personalizado.

6

Configurar HTTPS: Netlify proporciona automáticamente certificados SSL para todos los sitios a través de Let's Encrypt. Sin embargo, puede tardar un poco en emitirse el certificado SSL después de agregar un dominio personalizado. Para verificar y administrar la configuración del certificado SSL, podemos ir a la pestaña "Domain settings" y luego a "HTTPS". Si el certificado SSL aún no se ha emitido, podemos hacer clic en "Retry" para solicitar nuevamente la emisión del certificado.

Configurar notificaciones

Si queremos recibir notificaciones por correo electrónico o mediante webhooks cada vez que se envíe un formulario.

Podemos ir a la configuración del sitio en Netlify “Site settings” y hacer clic en la pestaña "Forms".

Hacemos clic en el botón "Settings" junto al formulario que queremos configurar y seleccionamos "Form notifications". Acá podemos agregar notificaciones por correo electrónico o configurar webhooks para servicios como Slack o Zapier.

En el caso de Netlify Forms, los webhooks pueden utilizarse para notificar a aplicaciones o servicios externos cada vez que se recibe un envío de formulario.

Definición: Webhooks

Son una forma de enviar notificaciones automáticas a aplicaciones o servicios externos cuando ocurre un evento específico. En términos simples, un webhook es una solicitud HTTP enviada desde un sistema a otro para comunicar que algo ha sucedido.

Cuando un usuario envía un formulario en nuestro sitio web, Netlify Forms captura el envío y, si hemos configurado webhooks, envía una solicitud HTTP (generalmente una solicitud POST) al punto final del webhook que hayamos especificado. La solicitud contiene información sobre el envío del formulario en formato JSON. 

Luego, la aplicación o servicio que recibe la solicitud del webhook puede procesarla y realizar acciones correspondientes, como enviar una notificación a un canal de Slack o actualizar una hoja de cálculo de Google por ejemplo.

Netlify Forms también utiliza filtros de spam que analizan los envíos de formularios y detectan patrones comunes utilizados por los spammers. Si se detecta un envío de spam, se bloquea y no se envía a su destino.


Vercel


Es otra plataforma de alojamiento popular que se enfoca en la experiencia del desarrollador y el rendimiento de la aplicación.

Ofrece características similares a Netlify, como integración continua y despliegue automático desde repositorios Git.

Está diseñado específicamente para aplicaciones React, Next.js y otras aplicaciones de JavaScript, lo que lo convierte en una opción ideal para proyectos basados en estas tecnologías..

Definición: Vercel

Es una herramienta que nos permite crear y desplegar aplicaciones web de forma automatizada. Es una plataforma que se enfoca en brindar una experiencia de desarrollo web muy rápida y cómoda, con la que podemos desplegar nuestras aplicaciones en muy poco tiempo y sin preocuparnos por la configuración del servidor.


Funciona a través de un proceso automatizado que se inicia cada vez que subimos nuestro código a la plataforma.

Una vez que el código está en la plataforma, se compila y se optimiza para su uso en la web. De esta forma, el usuario puede acceder a la aplicación desde cualquier dispositivo sin problemas.


Para utilizar Vercel, es necesario contar con una cuenta en la plataforma. Luego de crear la cuenta, podemos subir nuestro proyecto React a Vercel. Para esto, necesitamos tener nuestro proyecto en un repositorio de control de versiones, como GitHub o GitLab.

Una vez que hemos subido nuestro proyecto, podemos configurar las opciones de despliegue en la plataforma. En Vercel, podemos configurar cosas como el dominio de nuestra aplicación, el tipo de certificado SSL que utilizaremos y la configuración de nuestras variables de entorno.

Una de las características más interesantes de Vercel es que nos permite realizar despliegues automáticos.


Esto significa que cada vez que hacemos un push a nuestro repositorio, Vercel compilará y desplegará automáticamente la aplicación en línea.

Otra característica interesante de Vercel es que nos permite integrarnos con diferentes herramientas de terceros, como Stripe, CMS, bases de datos y más. Esto hace que sea muy fácil integrar nuestras aplicaciones con otros servicios en línea.

Vercel es como AWS para desarrolladores fron-end. Además tiene soporte para:

Dominios personalizados: Una vez deplegada nuestra app en Vercel podemos asignar un dominio personalizado a nuestra app de Next.js.

Environment Variables: Podemos también definir variables de entorno en Vercel. Después, podemos utilizar estas variables de entorno en nuestra app de Next.js.

HTTPS automático: HTTPS está habilitado por defecto (incluyendo dominios personalizados) y no requiere ninguna configuración adicional. También auto-renueva los certificados SSL.

Vercel: Desplegar una aplicación React


Una vez que tengamos una cuenta en Vercel, es hora de subir nuestra aplicación React. Para ello, necesitaremos tener nuestro proyecto React en un repositorio de control de versiones, como GitHub o GitLab.

En el sitio de Vercel:

Se abrirá una ventana emergente en la que debemos seleccionar la plataforma de control de versiones en la que se encuentra nuestro proyecto React. Luego, debemos autorizar a Vercel para que acceda a nuestra cuenta de dicha plataforma.

Ahora que hemos configurado las opciones de despliegue, es hora de desplegar nuestra aplicación React en Vercel. Para hacer esto, hacemos clic en el botón "Deploy" ubicado en la esquina superior derecha de la pantalla. Se iniciará un proceso automatizado que compilará y optimizará nuestra aplicación para su uso en la web.

Una vez que el proceso de despliegue haya finalizado, podremos acceder a nuestra aplicación React en línea. Vercel nos proporcionará una URL única para nuestra aplicación, que podremos compartir con nuestros usuarios.


Vercel: Integración con GitHub


Después de desplegamos una aplicación React en Vercel a través de GitHub, tendremos acceso a algunas funcionalidades muy interesantes, como por ejemplo:


Una URL para cada PR: Cada pull request que hagamos en GitHub tendrá su propia URL de despliegue, que se actualiza automáticamente con los cambios que realicemos en el código. Esto hace que sea muy fácil compartir la URL con nuestro equipo y permitirles ver los cambios en tiempo real sin tener que pedir la URL constantemente.

Una URL para cada Commit: Cada commit que hagamos en nuestro repositorio de GitHub también tendrá su propia URL de despliegue en Vercel. Esto nos permite testear rápidamente los cambios que hayamos hecho en un commit específico de forma rápida y sencilla.

Asignación de alias automático: Cuando realizamos actualizaciones en la rama master de nuestro repositorio de GitHub, Vercel asignará automáticamente un alias a nuestro nombre de dominio de producción. De esta forma, si cometemos un error y necesitamos revertir los cambios, simplemente podemos hacer push de un commit revirtiendo los cambios y Vercel realizará una reversión instantánea. Esto significa que nuestro sitio volverá al estado anterior antes de que podamos siquiera refrescar el navegador para verificarlo.


Vercel: Características


Despliegue automático

Vercel nos permite configurar el despliegue automático de nuestra aplicación React cada vez que se realice un cambio en nuestro repositorio de control de versiones.

Esto significa que cada vez que hagamos un push en nuestro repositorio, Vercel detectará automáticamente los cambios y desplegará la nueva versión de nuestra aplicación en línea.

Escalabilidad

Vercel nos permite escalar nuestra aplicación React de manera horizontal y vertical.

Podemos aumentar la cantidad de recursos de computación disponibles para nuestra aplicación en función de la cantidad de usuarios que estén accediendo a ella.

Además, Vercel utiliza una red de distribución de contenido (CDN) global que nos permite ofrecer una experiencia de usuario más rápida y confiable a nivel mundial.


Vercel: Control de versiones y ramas


Vercel nos permite crear diferentes ramas y versiones de nuestra aplicación React para que podamos trabajar en diferentes características y funcionalidades sin afectar la versión principal.

Podemos realizar pruebas en estas ramas y versiones y, una vez que estemos listos, podemos fusionarlas con la versión principal.

Para empezar a utilizar el control de versiones en Vercel, debemos crear diferentes ramas en nuestro repositorio de control de versiones, como por ejemplo "desarrollo", "pruebas" o "producción".


Una vez que tengamos estas ramas, podemos configurar Vercel para desplegar automáticamente cada rama en una URL diferente. De esta manera, podemos trabajar en diferentes características y funcionalidades en paralelo sin afectar la versión principal de nuestra aplicación.

Ejemplo

Supongamos que queremos trabajar en una nueva característica para nuestra aplicación React, pero no queremos afectar la versión actual de la aplicación

Podríamos crear una nueva rama en nuestro repositorio de control de versiones y trabajar en la nueva característica en esa rama.

Podemos configurar Vercel para desplegar automáticamente la rama de la nueva característica en una URL diferente, lo que nos permite probar la nueva funcionalidad sin afectar la versión actual de la aplicación.

Una vez que estemos satisfechos con la nueva característica y hayamos realizado las pruebas necesarias, podemos fusionar la rama de la nueva característica con la rama principal de nuestro repositorio de control de versiones.

Luego, Vercel detectará automáticamente los cambios en la rama principal y desplegará la nueva versión de nuestra aplicación en línea.

Además del control de versiones en las ramas, Vercel también nos permite crear diferentes versiones de nuestra aplicación para diferentes entornos, como por ejemplo "desarrollo", "pruebas" y "producción".


De esta manera, podemos probar nuestra aplicación en diferentes entornos antes de lanzarla en producción.


Vercel: Análisis de rendimiento


Vercel nos ofrece herramientas de análisis de rendimiento que nos permiten medir y mejorar la velocidad y eficiencia de nuestra aplicación.

Podemos identificar cuellos de botella y mejorar el rendimiento de nuestra aplicación para ofrecer una mejor experiencia de usuario.

El análisis de rendimiento es una característica muy útil que ofrece Vercel, ya que nos permite medir y mejorar la velocidad y eficiencia de nuestra aplicación React.


Para ello, Vercel nos ofrece varias herramientas de análisis y monitoreo, que incluyen:

Lighthouse: Es una herramienta de análisis de rendimiento de código abierto, creada por Google, que nos permite medir la velocidad y eficiencia de nuestra aplicación. Lighthouse evalúa nuestra aplicación en diferentes categorías, como accesibilidad, rendimiento, mejores prácticas, SEO y más. Al final de la evaluación, Lighthouse nos proporciona una puntuación general y recomendaciones para mejorar el rendimiento de nuestra aplicación.

Tiempo de carga: Vercel nos ofrece información detallada sobre el tiempo de carga de nuestra aplicación, que nos permite identificar cuellos de botella y áreas de mejora. Podemos ver el tiempo de carga promedio, el tiempo de carga de la primera página y más.

Uso de recursos: Vercel nos permite ver el uso de recursos de nuestra aplicación, incluyendo el uso de CPU, memoria y almacenamiento. Esto nos permite identificar áreas donde nuestra aplicación puede estar utilizando demasiados recursos y optimizarla para mejorar el rendimiento.

Análisis de tráfico: Vercel nos permite ver información detallada sobre el tráfico de nuestra aplicación, incluyendo el número de visitas, la ubicación geográfica de los usuarios y más. Esto nos permite identificar patrones de uso y optimizar nuestra aplicación para satisfacer las necesidades de nuestros usuarios.


GitHub: Pages


Si bien GitHub Pages no ofrece características de backend y es menos adecuado para aplicaciones más grandes o dinámicas, es una excelente opción para alojar aplicaciones React de una manera sencilla y eficiente.

Definición: GitHub Pages

Es un servicio de alojamiento gratuito ofrecido por GitHub, que permite a los desarrolladores alojar sitios web estáticos directamente desde sus repositorios en GitHub. Es una excelente opción para proyectos de código abierto y sitios web de documentación.

A continuación veremos paso a paso cómo hacerlo:

Crear un repositorio en GitHub

Creamos un repositorio en GitHub para nuestro proyecto siguiendo los pasos mencionados en respuestas anteriores. Debemos asegurarnos de clonar este repositorio en nuestra máquina local.

Instalar la biblioteca gh-pages

Para desplegar nuestra aplicación React en GitHub Pages, primero necesitamos instalar la biblioteca gh-pages como una dependencia de desarrollo en nuestro proyecto. Ejecutamos el siguiente comando en la terminal:

CLI Copiar código 
 
                            
npm install gh-pages --save-dev

Modificar el archivo package.json.

A continuación, debemos agregar algunos scripts y configuraciones al archivo package.json de nuestro proyecto. Primero, agrega un campo homepage con la siguiente estructura:

JSON Copiar código 
 
                            
"homepage": "https://tu_usuario.github.io/nombre_del_repositorio"

Reemplazamos "tu_usuario" con el nombre de usuario de GitHub y "nombre_del_repositorio" con el nombre del repositorio.

Luego, añadimos dos scripts en la sección "scripts" del archivo package.json:

JSON Copiar código 
 
                            
"predeploy": "npm run build",

"deploy": "gh-pages -d build"

La sección "scripts" debería verse así:

JSON Copiar código 
 
                            
"scripts": {

// Otros scripts existentes...

    "predeploy": "npm run build",

    "deploy": "gh-pages -d build"

}

Estos scripts nos ayudarán a construir nuestra aplicación y desplegarla en GitHub Pages.

Desplegar la aplicación en GitHub Pages

Ahora que hemos configurado nuestro proyecto, podemos desplegar nuestra aplicación React en GitHub Pages ejecutando el siguiente comando en la terminal:

CLI Copiar código 
 
                            
npm run deploy

Este comando construirá la aplicación y la desplegará en la rama gh-pages de nuestro repositorio. Si la rama gh-pages no existe, el comando la creará automáticamente.

Configurar GitHub Pages en nuestro repositorio

Para que GitHub Pages sepa desde dónde servir nuestra aplicación, debemos configurar la fuente de GitHub Pages en nuestro repositorio. Realizamos los siguientes pasos:

Vamos a la página principal del repositorio en GitHub.

Hacemos clic en la pestaña "Settings" (Configuración).

Nos desplazamos hacia abajo hasta la sección "GitHub Pages".

En "Source" (Fuente), seleccionamos la rama gh-pages y guardamos los cambios.

Acceder a nuestra aplicación React desplegada

Una vez que hayamos realizado todos estos pasos, nuestra aplicación React estará desplegada en GitHub Pages. 

Podemos acceder a ella utilizando la URL especificada en el campo "homepage" del archivo package.json: https://tu_usuario.github.io/nombre_del_repositorio.

Ten en cuenta que puede tomar algunos minutos antes de que los cambios se reflejen en la URL.

Actualizar la aplicación

Cuando necesitemos realizar cambios en la aplicación, simplemente seguimos estos pasos para actualizar la versión desplegada en GitHub Pages:

Realizamos los cambios necesarios en el código fuente localmente.

Creamos un nuevo commit con los cambios.

CLI Copiar código 
 
                            
git add .

git commit -m "Descripción de los cambios realizados"

Subimos los cambios a la rama main de el repositorio remoto.

CLI Copiar código 
 
                            
git push origin main

Desplegamos la nueva versión de la aplicación en GitHub Pages.

CLI Copiar código 
 
                            
npm run deploy

Esto actualizará automáticamente la versión desplegada de la aplicación en GitHub Pages con los cambios realizados.

Siguiendo estas instrucciones, podemos desplegar y mantener actualizada nuestra aplicación React en GitHub Pages de manera rápida y eficiente.

Esta es una excelente manera de mostrar nuestros proyectos a otros desarrolladores, empleadores potenciales y usuarios interesados en nuestro trabajo. Además, al utilizar GitHub Pages, podemos aprovechar las herramientas de control de versiones y colaboración que ofrece Git y GitHub para gestionar y mejorar los proyectos a lo largo del tiempo.


GitHub: Actions


Se trata de una herramienta que nos permite automatizar tareas en nuestro repositorio de GitHub, como la compilación y el despliegue de nuestra aplicación.

Con GitHub Actions, podemos crear flujos de trabajo personalizados que se ejecuten automáticamente cuando se cumplan ciertas condiciones, como hacer push a una rama específica o crear un pull request.


Firebase Hosting


Es una opción sólida para aplicaciones de React que también utilizan otros servicios de Firebase, ya que permite una integración fácil y una escalabilidad eficiente.

Definición: Firebase Hosting

Es una solución de alojamiento proporcionada por Google que se integra con otros servicios de Firebase, como bases de datos en tiempo real y autenticación.

Despliegue de una aplicación React en Firebase Hosting:

Crear un proyecto en Firebase

Creamos una cuenta en Firebase y creamos un nuevo proyecto desde la consola de Firebase. Debemos asegurarnos de tener instalado Node.js en la computadora, ya que lo necesitarás para instalar la herramienta de línea de comandos de Firebase.

Instalar la herramienta CLI de Firebase

Para trabajar con Firebase Hosting, necesitamos instalar la herramienta de línea de comandos de Firebase en nuestra computadora. Ejecutamos el siguiente comando en la terminal:

CLI Copiar código 
 
                            
npm install -g firebase-tools

Iniciar sesión en Firebase desde CLI

Inicia sesión en la cuenta de Firebase ejecutando el siguiente comando.

CLI Copiar código 
 
                            
firebase login

Esto abrirá una ventana en el navegador para que podamos iniciar sesión con nuestra cuenta de Google asociada a Firebase.

Inicializar el proyecto de Firebase

Vamos al directorio de el proyecto React y ejecutamos el siguiente comando para inicializar Firebase en el proyecto:

CLI Copiar código 
 
                            
firebase init

Esto iniciará un proceso interactivo donde tendremos que responder algunas preguntas para configurar el proyecto:

Seleccionamos "Hosting" cuando nos pregunte qué servicios de Firebase deseamos configurar.

Elegimos el proyecto de Firebase que creaste anteriormente en la consola de Firebase.

Cuando nos pregunte por el directorio público, escribimos build ya que es el directorio donde se generan los archivos de producción de una aplicación React.

Debemos asegurarnos de aceptar la configuración de la aplicación como una SPA (Single Page Application) cuando nos pregunte.

Una vez que hayamos completado estos pasos, Firebase generará dos archivos en el proyecto: .firebaserc y firebase.json. Estos archivos contienen la configuración necesaria para desplegar la aplicación en Firebase Hosting.

Construir la aplicación React para producción:

Antes de desplegar la aplicación, necesitamos construir la versión de producción de nuestra aplicación React. Ejecutamos el siguiente comando en el directorio de nuestro proyecto:

CLI Copiar código 
 
                            
npm run build

Esto generará una carpeta build con los archivos optimizados para producción.

Desplegar la aplicación en Firebase Hosting

Ahora que hemos configurado nuestro proyecto de Firebase y construido la aplicación para producción, podemos desplegarla en Firebase Hosting. Ejecutamos el siguiente comando en el directorio de el proyecto:

CLI Copiar código 
 
                            
firebase deploy

Este comando subirá los archivos de la carpeta build a Firebase Hosting y nos proporcionará una URL para acceder a la aplicación desplegada.

Acceder a la aplicación desplegada

Visitamos la URL proporcionada después del despliegue exitoso para acceder a la aplicación React desplegada en Firebase Hosting. También podemos encontrar esta URL en la consola de Firebase en la sección "Hosting".


Despliegue: React.js y Nginx en Ubuntu 20.04


Requisitos:

Servidor cloud Ubuntu 20.04.

Nombre de dominio válido apuntando a la IP del servidor.

Usuario root o con privilegios sudo.

Repositorio Git (por ejemplo, GitHub) que contenga nuestra aplicación React.js desarrollada.


Instalación de Node.js y NPM


Antes de comenzar, debemos asegurarnos de tener Node.js y NPM instalados en nuestro servidor. Si no lo están, seguiremos estos pasos:

Instalar las dependencias requeridas con el siguiente comando:

CLI Copiar código 
 
                            
apt-get update && apt-get upgrade && apt-get install curl gnupg2 gcc g++ make libssl-dev -y

Añadir el repositorio Nodesource con el siguiente comando:

CLI Copiar código 
 
                            
curl -sL https://deb.nodesource.com/setup_current.x -o nodesource_setup.sh && bash nodesource_setup.sh

Instalar Node.js y NPM con el siguiente comando:

CLI Copiar código 
 
                            
apt-get install nodejs -y && npm install npm@latest -g

Verificar las versiones de Node.js y NPM con el siguiente comando:

CLI Copiar código 
 
                            
node --version && npm --version

Esto nos mostrará las versiones de Node.js y NPM.


Descargar la aplicación React


Clonar la app Reactjs desde Git

Ahora clonaremos nuestra aplicación ya desarrollada desde un repositorio de Git. Para ello, primero necesitamos instalar Git en nuestro servidor con el siguiente comando:

CLI Copiar código 
 
                            
apt-get install git -y

Luego, clonaremos el repositorio de nuestra aplicación React.js en el directorio deseado (por ejemplo, /root/reactapp). Reemplazamos <URL_DEL_REPOSITORIO> con la URL real del repositorio de Git:

CLI Copiar código 
 
                            
git clone <URL_DEL_REPOSITORIO> /root/reactapp

Instalar dependencias del proyecto

Ahora que hemos clonado nuestra aplicación, navegaremos al directorio del proyecto y ejecutaremos npm install para instalar las dependencias necesarias:

CLI Copiar código 
 
                            
cd /root/reactapp && npm install

Ejecutamos el comando build

Esto compilará JavaScript y sus activos en el directorio de la estructura.

CLI Copiar código 
 
                            
npm run build

Nginx: Despliegue de app React


Ahora desplegaremos el directorio de la estructura con Nginx siguiendo estos pasos:

1

Instalar el paquete del servidor Nginx con el siguiente comando:

CLI Copiar código 
 
                            
apt-get install nginx python3-certbot-nginx -y
2

Crear un directorio para la aplicación React con el siguiente comando:

CLI Copiar código 
 
                            
mkdir /var/www/html/react
3

Copiar todos los contenidos desde el directorio de la estructura al directorio React:

CLI Copiar código 
 
                            
cp -r /root/reactapp/build/* /var/www/html/react/
4

Establecer la propiedad adecuada del directorio React con el siguiente comando:

CLI Copiar código 
 
                            
chown -R www-data:www-data /var/www/html/react
5

Crear un archivo de configuración de host virtual Nginx para alojar nuestra aplicación React:

CLI Copiar código 
 
                            
nano /etc/nginx/sites-enabled/react.conf
6

Añadir las siguientes líneas (reemplazar "react.ejemplo.com" con nuestro dominio real):

CLI Copiar código 
 
                            
server {

    listen 80;

    listen [::]:80;

    root /var/www/html/react/;

    index index.html index.htm;

    server_name react.ejemplo.com;

    location / {

        try_files $uri $uri/ =404;

    }

}
7

Guardar y cerrar el archivo, y luego verificar Nginx para evitar errores de sintaxis:

CLI Copiar código 
 
                            
nginx -t
8

Reiniciar el servicio Nginx para aplicar los cambios:

CLI Copiar código 
 
                            
systemctl restart nginx
9

Utilizar Certbot para adquirir un certificado SSL para nuestro dominio:

CLI Copiar código 
 
                            
certbot --nginx -d react.ejemplo.com --redirect

Acceder a nuestra aplicación React.js


Finalmente, abriremos nuestro navegador web y accederemos a nuestra aplicación React.js desde la URL https://react.ejemplo.com (reemplazar "react.ejemplo.com" con nuestro dominio real). Deberíamos ver la pantalla de inicio de nuestra aplicación React.
