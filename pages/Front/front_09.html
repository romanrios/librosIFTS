Componentes contenedores y presentacionales


A continuación, aprenderemos a separar los componentes contenedores de los componentes presentacionales en React. Esta práctica es esencial para mantener una estructura de código ordenada y fácil de mantener.


Ventajas:

  • - Facilita la reutilización de componentes presentacionales.
  • - Mejora la legibilidad y mantenibilidad del código.
  • - Permite separar las responsabilidades entre la lógica y la presentación de la aplicación.
  • - Facilita la realización de pruebas unitarias.
  • - .

Componentes presentacionales

Los componentes presentacionales, también conocidos como componentes "tontos" o de vista, son responsables de la presentación y el estilo de la aplicación. Estos componentes no tienen estado y, en su mayoría, reciben datos a través de las propiedades (props) para renderizar la interfaz de usuario.

Ejemplo de componente presentacional:

JavaScript Copiar código 
 
                            
import React from "react";

const ListaUsuarios = ({ usuarios }) => (

    <ul>

        {usuarios.map((usuario) => (

            <li key={usuario.id}>{usuario.nombre}</li>

        ))}

    </ul>

);

export default ListaUsuarios;

Los componentes presentacionales, en general, no necesitan manejar estado.

Objetivos: Componentes presentacionales

No necesitan manejar estado. Su principal responsabilidad es la presentación y el estilo de la aplicación. Estos componentes reciben datos a través de las propiedades (props) y no manejan la lógica ni el estado de la aplicación.

Sin embargo, puede haber situaciones en las que un componente presentacional necesite manejar un estado local mínimo.

Ejemplo

Si un componente presentacional contiene un formulario con campos de entrada, puede necesitar manejar el estado local de los valores de entrada. En estos casos, es aceptable que un componente presentacional tenga su propio estado local, pero debe ser mínimo y estar relacionado exclusivamente con la presentación.

Componentes contenedores

Los componentes contenedor, también llamados componentes "inteligentes" o de lógica, son responsables de la gestión del estado y la lógica de la aplicación. Estos componentes tienen estado y se encargan de manejar la comunicación con servicios externos, como APIs, para obtener y manipular datos.

Ejemplo de componente contenedor:

JavaScript Copiar código 
 
                            
import React, { useState, useEffect } from "react";

import ListaUsuarios from "./ListaUsuarios";

const ContenedorUsuarios = () => {

    const [usuarios, setUsuarios] = useState([]);

    useEffect(() => {

        // Obtener datos de la API y actualizar el estado

        fetch("https://api.example.com/usuarios")

            .then((response) => response.json())

            .then((data) => setUsuarios(data));

    }, []);

    return <ListaUsuarios usuarios={usuarios} />;

};

export default ContenedorUsuarios;

CSS en React


En React, es posible aplicar estilos CSS a nuestros componentes de diversas maneras. A continuación exploraremos las principales técnicas para aplicar estilos en React.


Estilos en línea


En React, es posible aplicar estilos CSS a nuestros componentes de diversas maneras. A continuación exploraremos las principales técnicas para aplicar estilos en React.

Los estilos en línea se aplican directamente a los elementos HTML en el componente utilizando la propiedad style. Los estilos se definen como un objeto JavaScript con pares clave-valor, donde las claves son nombres de propiedades CSS en formato camelCase y los valores son cadenas o números.

Ventajas


La forma más rápida de escribir estilos.

Bueno para crear aplicaciones pequeñas (escribimos estilos en línea y luego los movemos a una hoja de estilos).

Tiene una gran preferencia (puede anular estilos de una hoja de estilo CSS).

Desventajas


Es tedioso convertir CSS plano a estilos en línea.

Demasiados estilos en línea hacen que JSX (React) sea ilegible.

No podemos usar características básicas de CSS como animaciones, selectores,etc.

No escala bien.

JavaScript Copiar código 
 
                            
const App = () => {

    return (

        <div style={{ color: "blue", fontSize: "20px" }}>

            Este texto tiene estilos en línea

        </div>

    );

};

¿Para qué sirven?

Llaves exteriores

Inyectan JavaScript en JSX. Indican que "todo lo que esté entre nosotros debe ser leído como JavaScript, no como JSX".

Las llaves interiores crean un objeto literal de JavaScript. Hacen que esto sea un objeto válido de JavaScript:

JavaScript Copiar código 
 
                            
{

    color: "blue";

}

Otra opción es crear un objeto styles de JavaScript con nuestros estilos y pasarselo directamente al elemento:

JavaScript Copiar código 
 
                            
class App extends Component {

    render() {

        const styles = {

            backgroundColor: "blue",

            color: "white",

            padding: 20

        };

        return <div style={styles}>¡Hola, mundo!</div>;

    }

}

Sintaxis


La sintaxis de nombre de estilo en React es diferente de la sintaxis de CSS estándar.

En lugar de utilizar guiones bajos (por ejemplo, font-size), utilizamos la notación camelCase (fontSize).


Esto se debe a que los nombres de las propiedades del objeto en JavaScript no pueden contener guiones.

JavaScript Copiar código 
 
                             
const estiloBoton = {

    backgroundColor: "green",

    borderColor: "black",
 
    borderRadius: "5px"

};

Sintaxis de valor de estilo

Los valores de las propiedades de estilo pueden ser números o cadenas.

Si el valor es un número, React lo interpretará como píxeles (px).


Para otros valores, como porcentajes o unidades de medida (por ejemplo, em, rem), necesitamos utilizar cadenas.

JavaScript Copiar código 
 
                            
const estiloDiv = {

    width: "50%", // Porcentaje

    padding: 10, // Se interpreta como 10px

    margin: "1rem" // Unidades de medida

};

Compartir estilos entre múltiples componentes

Para compartir estilos entre varios componentes, podemos crear un archivo separado que exporte los objetos de estilo y luego importarlos en los componentes que los necesiten.

1

Creamos un archivo estilos.js

JavaScript Copiar código 
 
                            
export const estiloTexto = {

    color: "blue",

    fontSize: "20px"

};



export const estiloBoton = {

    backgroundColor: "green",

    borderColor: "black",

    borderRadius: "5px"
 
};
2

Importamos y utilizamos los estilos en los componentes

JavaScript Copiar código 
 
                            
import React from "react";

import { estiloTexto, estiloBoton } from "./estilos";

const MiComponente = () => {

    return (

        <div>

            <p style={estiloTexto}>Texto con estilo compartido</p>

            <button style={estiloBoton}>Botón con estilo compartido</button>

        </div>

    );

};

export default MiComponente;
3

En el archivo `estilos.js`, definimos y exportamos dos objetos de estilo: `estiloTexto` y `estiloBoton`. Estos objetos contienen propiedades y valores de estilo que queremos compartir entre diferentes componentes.

4

En el componente `MiComponente`, importamos los objetos de estilo ` estiloTexto` y `estiloBoton` desde el archivo `estilos.js`. Asignamos estos objetos de estilo a las propiedades `style ` de los elementos <p> y <button> .

Al utilizar este enfoque, podemos compartir estilos entre múltiples componentes y mantener nuestro código ordenado y fácil de mantener.



Hoja de estilos externa


Ahora veremos cómo utilizar hojas de estilo externas en nuestros componentes de React. Aunque los estilos en línea y los objetos de estilo son comunes en React, también podemos aplicar estilos de la misma manera que en aplicaciones web tradicionales utilizando hojas de estilo CSS externas.

Ventajas



Nos da todas las herramientas modernas de CSS (variables, selectores avanzados, nuevas pseudoclases, etc).

Ayuda a mantener limpio nuestros componentes de estilos en línea.

Desventajas



Es necesario configurar los prefijos de navegador para asegurar que las funciones más recientes funcionen para todos los usuarios.

Requiere escribir más y a veces código repetitivo que otras librerías de CSS (SASS).

Cualquier hoja de estilo en cascada importada a un componente, afecta también a sus componentes hijos, no está limitado al componente.

Debemos usar una convención de nomenclatura confiable para garantizar que los estilos no entren en conflicto.


Podemos crear una hoja de estilos CSS externa e importarla en nuestro componente de React. Este enfoque es similar al de las aplicaciones web tradicionales.

1

Creamos un archivo styles.css: En la carpeta src del proyecto:

CSS Copiar código 
                            

/* styles.css */

.container {

    background-color: blue;

    color: white;

    padding: 20px;

}

En este archivo, definimos una clase CSS llamada .container con algunas propiedades y valores de estilo.

2

Importamos el archivo styles.css: En el componente de React y aplicamos la clase .container a un elemento:

JavaScript Copiar código 
                            

// App.js

import React from "react";

import "./styles.css";

const App = () => {

    return <div className="container">¡Hola, mundo!</div>;

};

export default App;

Este método nos permite aprovechar las características de CSS, como selectores y pseudo-clases, y mantener nuestros estilos en archivos separados para facilitar la organización y el mantenimiento del código.


Al utilizar este enfoque, podemos aplicar estilos en nuestros componentes funcionales de React de la misma manera que en aplicaciones web tradicionales.


CSS Modules


Los módulos CSS permiten definir estilos específicos para cada componente, evitando la posibilidad de conflictos de nombres de clases entre diferentes componentes. Para utilizar CSS Modules, creamos un archivo .module.css y lo importamos en un componente de React.

1

Creamos un archivo de estilos: Con la extensión .module.css en la carpeta src del proyecto:

CSS Copiar código 
                            

/* Button.module.css */

.button {

    background-color: blue;

    color: white;

    padding: 10px;

    border: none;

    border-radius: 5px;

    cursor: pointer;

}

En este archivo, definimos una clase CSS llamada .button con algunas propiedades y valores de estilo.

2

Importamos el archivo Button.module.css.: En el componente de React y aplicamos la clase .button:

JavaScript Copiar código 
                            

// Button.js

import React from "react";

import styles from "./Button.module.css";

const Button = (props) => {

    return (

        <button className={styles.button} onClick={props.onClick}>

            {props.children}

        </button>

    );

};

export default Button;

Al utilizar CSS Modules, los estilos se encapsulan automáticamente en el componente, y las clases CSS se convierten en nombres únicos, evitando conflictos con otros componentes y estilos.

Ventajas

Los estilos tienen un alcance a uno u otro componente específico (a diferencia de CSS / SASS).

Único, genera nombres de clases que aseguran no entrar en conflicto.

Pueden ser usados inmediatamente sin configuración en proyectos CRA (create-react-app).

Pueden ser usados con SASS / CSS.

Desventajas

Puede ser complicado hacer referencias a nombres de clases.

Puede ser complicado usar estilos CSS como propiedades de objetos.


Styled-components


La biblioteca styled-components permite crear componentes de estilo en React utilizando la sintaxis de las etiquetas de plantilla (template literals).

Resumen

Styled-components es una librería popular que permite escribir estilos en JavaScript utilizando etiquetas de plantilla, lo que facilita la creación de componentes estilizados y reutilizables.


Para utilizar styled-components en componentes de clase:

1

Debemos instalar la biblioteca con npm install styled-components o yarn add styled-components.

CLI Copiar código 
 
                            
npm install styled-components
2

Importamos la función styled desde la biblioteca styled-components en nuestro archivo de componente:

JavaScript Copiar código 
 
                            
import styled from "styled-components";
3

Utilizamos la función styled para crear un componente estilizado:

JavaScript Copiar código 
 
                            
const Button = styled.button`

    background-color: blue;

    color: white;

    padding: 10px;

    border: none;

    border-radius: 5px;

    cursor: pointer;

    &:hover {

        background-color: darkblue;

    }

`;

En este ejemplo, creamos un componente Button estilizado utilizando la función styled.button. A continuación, escribimos las reglas CSS entre las etiquetas de plantilla.

4

Usamos el componente estilizado en la aplicación como cualquier otro componente de React:

JavaScript Copiar código 
 
                            
import React from "react";

import Button from "./Button";

const App = () => {

    return (

        <div>

            <Button>Click me</Button>

        </div>

    );

};

export default App;

Ventajas

Encapsulación de estilos: Los estilos de un componente estilizado están encapsulados y no afectarán a otros componentes.

Componentes reutilizables: Podemos crear componentes estilizados reutilizables y compartirlos fácilmente en nuestra aplicación o entre proyectos.

Propiedades dinámicas: Podemos pasar props a los componentes estilizados y ajustar los estilos según sea necesario.

Estilos globales: Styled-components permite definir estilos globales que se aplicarán a toda nuestra aplicación.

Compatibilidad con temas: Styled-components es compatible con la creación de temas y su cambio dinámico.

Desventajas

Dependencia adicional: Utilizar Styled-components implica agregar una nueva dependencia a nuestro proyecto.

Curva de aprendizaje: Si bien Styled-components simplifica la gestión de estilos, puede ser necesario aprender nuevas técnicas y conceptos.

Rendimiento: La creación de componentes estilizados puede tener un impacto en el rendimiento de la aplicación.


SASS / SCSS


SASS nos permite escribir estilos en dos tipos diferentes de hoja de estilos, con las extensiones .scss y .sass.

SASS

Syntactically Awesome Style Sheets

Nos brinda poderosas herramientas, algunas de las cuales no existen en una hoja de estilos CSS normal. Esto incluye características como variables, estilos extendidos y anidamiento.

Los estilos SCSS están escritos en una sintaxis similar a CSS normal, sin embargo los estilos de SASS no requieren usar llaves de apertura ( { ) y cierre ( } ) .

Dado que esto no es CSS normal, debe compilarse a CSS plano. Para hacer esto en nuestras aplicaciones de React, podemos usar una librería como node-sass.

CLI Copiar código 
 
                            
npm install node-sass

Ventajas

Incluye muchas características dinámicas de CSS como el poder de extender las reglas, anidamiento, y mixins.

Los estilos con SCSS son mucho menos repetitivos que CSS normal.

Desventajas

Similar a CSS plano, los estilos son globales y no se limitan a un componente.

Las hojas de estilos CSS están comenzando a incluir un número de características que SASS tenía exclusivamente, como variables por ejemplo.

SASS / SCSS con frecuencia requieren una configuración, como instalar la librería de Node node-sass.


Bootstrap en React


Ahora aprenderemos a integrar Bootstrap en nuestros proyectos de React para diseñar y aplicar estilos de manera rápida y eficiente. Exploraremos cómo utilizar las clases de Bootstrap y componentes predefinidos para mejorar el aspecto visual de nuestras aplicaciones.

Definición: Bootstrap

Es un popular marco de diseño (framework) de código abierto que facilita el diseño y estilizado de sitios web y aplicaciones.

Bootstrap utiliza HTML, CSS y JavaScript para proporcionar una amplia gama de estilos y componentes predefinidos que se pueden aplicar fácilmente en nuestras aplicaciones React.


Integrar Bootstrap en React


Para integrar Bootstrap en un proyecto de React, es necesario instalarlo mediante npm o yarn. Ejecutamos el siguiente comando en la terminal dentro del directorio de nuestro proyecto:

CLI Copiar código 
 
                            
npm install bootstrap

Una vez que Bootstrap esté instalado, es necesario importarlo en nuestro proyecto para poder utilizar sus estilos y componentes. Para hacer esto, agrega la siguiente línea en nuestro archivo src/index.js:

JavaScript Copiar código 
 
                            
import "bootstrap/dist/css/bootstrap.min.css";

Utilizar clases Bootstrap en React


Con Bootstrap importado en el proyecto, podemos utilizar sus clases de CSS y componentes en nuestros componentes de React. Por ejemplo, para utilizar las clases de Bootstrap en un componente de "Card":

JavaScript Copiar código 
 
                            
const Card = ({ titulo, descripcion }) => {

    return (

        <div className="card" style={{ width: "18rem" }}>

            <div className="card-body">

                <h5 className="card-title">{titulo}</h5>

                <p className="card-text">{descripcion}</p>

                <a href="#" className="btn btn-primary">

                    Ir a algún lugar

                </a>

            </div>

        </div>

    );

};

Hemos aplicado las clases card, card-body, card-title, card-text y btn btn-primary de Bootstrap en nuestro componente Card.

Ahora podemos utilizar este componente en nuestra aplicación pasandole las props correspondientes:

JavaScript Copiar código 
 
                            
function App() {

    return (

        <div>

            <Card titulo="Tarjeta" descripcion="Esta es una tarjeta" />

        </div>

    );

}

export default App;

Utilizar componentes de React-Bootstrap


React-Bootstrap es una biblioteca que proporciona versiones de los componentes de Bootstrap adaptadas específicamente para su uso en React. Estos componentes hacen uso de las clases de Bootstrap y siguen la filosofía de React.

1

Instalamos la biblioteca React-Bootstrap:

CLI Copiar código 
 
                            
npm install react-bootstrap
2

Importamos los componentes de React-Bootstrap que queremos utilizar en los componentes de React:

JavaScript Copiar código 
 
                            
import { Card, Button } from "react-bootstrap";
3

Creamos un componente nuevo con ellos:

JavaScript Copiar código 
 
                            
const MyCard = ({ titulo, descripcion }) => {

  return (

    <Card style={{ width: "18rem" }}>

      <Card.Body>

        <Card.Title>{titulo}</Card.Title>

        <Card.Text>{descripcion}</Card.Text>

        <Button variant="primary">Ir a algún lugar</Button>

      </Card.Body>

    </Card>

  );

};



function App() {

  return (

    <div>

      <MyCard titulo="Tarjeta" descripcion="Esta es una tarjeta" />

    </div>

  );

}



export default App;

Importante

Componentes Bootstrap

El componente nuevo se llama MyCard ya que no podemos asignarle el nombre Card. Esto se debe a que Card ya existe en la aplicación y se refiere al componente que importamos de Bootstrap.

Si ejecutamos la aplicación, veremos que el componente es visualmente igual al que creamos utilizando clases, aunque ahora funciona con los componentes de Bootstrap:

En este ejemplo, hemos reemplazado las clases de Bootstrap con componentes de React-Bootstrap como Card, Card.Body, Card.Title, Card.Text y Button. Estos componentes ya incluyen los estilos de Bootstrap y hacen que el código sea más legible y fácil de mantener.

Usar componentes de Bootstrap en lugar de clases tiene algunas ventajas cuando se trabaja con React, aunque ambas opciones son válidas:

Consistencia con React: Utilizar componentes de Bootstrap en lugar de clases hace que el código sea más consistente con la filosofía de React y sus prácticas recomendadas. Los componentes de React se enfocan en la reutilización y composición, y utilizar componentes de Bootstrap en lugar de clases permite aprovechar estas características de React.

Mejor legibilidad: Los componentes de Bootstrap, especialmente cuando se usan con bibliotecas como React-Bootstrap, están diseñados para adaptarse al flujo de trabajo de React y, por lo tanto, hacen que el código sea más legible y fácil de mantener. Al utilizar componentes específicos, el propósito y la estructura del código se vuelven más claros.

Control de propiedades: Los componentes de Bootstrap proporcionan propiedades específicas que facilitan la personalización y el control de la apariencia y el comportamiento de los elementos. Estas propiedades están diseñadas para funcionar con React, lo que facilita la manipulación de estilos y la interacción con el componente.

Integración de eventos y estados: Los componentes de Bootstrap permiten una mejor integración con los eventos y estados de React. Al usar componentes en lugar de clases, es más fácil manejar eventos y estados en el contexto de React.


Personalización de estilos Bootstrap


Aunque Bootstrap proporciona estilos y componentes predefinidos, también es posible personalizarlos para adaptarlos a las necesidades específicas de nuestra aplicación. Podemos aplicar estilos personalizados a través de CSS o utilizando estilos en línea.

Ejemplo

Si queremos cambiar el color de fondo de un botón, podemos crear un archivo CSS y aplicar el estilo personalizado a la clase correspondiente:


1

CustomStyles.css:

CSS Copiar código 
 
                            
.btn-custom {

    background-color: #123456;

    color: #ffffff;

}
2

Importa el archivo CSS en el componente y aplica la clase btn-custom al botón:

JavaScript Copiar código 
 
                            
import React from "react";

import { Button } from "react-bootstrap";

import "./customStyles.css";

const CustomButton = () => {

    return <Button className="btn-custom">Botón personalizado</Button>;

};

export default CustomButton;

Otra opción es aplicar estilos en línea a los componentes utilizando la propiedad style:

JavaScript Copiar código 
 
                            
const CustomButton = () => {

    const customStyle = {

        backgroundColor: "#123456",
 
        color: "#ffffff" 

    };   
 
    return <Button style={customStyle}>Botón personalizado</Button>;

};

propTypes


A continuación, aprenderemos sobre PropTypes en React y cómo utilizarlos para validar las propiedades (props) que se pasan a los componentes.

Definición: PropTypes

Es un mecanismo de React que permite especificar y validar el tipo de datos de las propiedades que se pasan a un componente. PropTypes ayuda a garantizar que los componentes reciban las propiedades correctas y en el formato adecuado, lo que facilita la detección de errores y mejora la calidad del código.


Los propTypes son útiles por dos razones.

Validación de props: La validación puede asegurar que las props están haciendo lo que se supone que deben hacer. Si faltan props, o si están presentes pero no son lo que esperabas, entonces se imprimirá una advertencia en la consola.

Documentación: Documentar props hace que sea más fácil echar un vistazo a un archivo y entender rápidamente la clase componente que contiene. Cuando tenemos muchos archivos, y los tendremos, esto puede ser un gran beneficio.


Cómo funcionan


Para utilizar PropTypes en un componente:

Se debe importar la biblioteca prop-types.

Se define un objeto propTypes en el componente, que contiene las validaciones de tipos para cada propiedad que recibe el componente.

Estas validaciones pueden especificar el tipo de dato esperado (por ejemplo, string, number, bool, etc.), así como si una propiedad es opcional u obligatoria (usando .isRequired).

En el entorno de desarrollo

Cuando un componente recibe una propiedad que no coincide con las PropTypes definidas, React muestra una advertencia en la consola del navegador. Esto permite a los desarrolladores identificar y corregir problemas relacionados con las propiedades de los componentes.

Las advertencias de PropTypes se muestran en la consola del navegador en el entorno de desarrollo para que puedas identificar y corregir problemas relacionados con las propiedades de los componentes. En el entorno de producción, estas advertencias se eliminan automáticamente para evitar afectar el rendimiento de la aplicación.

Tipos de PropTypes


PropTypes ofrece varios tipos de validadores para diferentes tipos de datos:

PropTypes.string: Valida que la propiedad sea una cadena de texto.

PropTypes.number: Valida que la propiedad sea un número.

PropTypes.bool: Valida que la propiedad sea un valor booleano.

PropTypes.func: Valida que la propiedad sea una función.

PropTypes.array: Valida que la propiedad sea un array.

PropTypes.object: Valida que la propiedad sea un objeto.


También se pueden utilizar validadores más específicos, como:

PropTypes.oneOf([...]): Valida que la propiedad coincida con uno de los valores especificados.

PropTypes.oneOfType([...]): Valida que la propiedad coincida con uno de los tipos especificados.

PropTypes.arrayOf(...): Valida que la propiedad sea un array de elementos del tipo especificado.

PropTypes.shape(...): Valida que la propiedad sea un objeto con una estructura específica.


Propiedades opcionales y obligatorias


Podemos especificar si una propiedad es opcional u obligatoria utilizando .isRequired después del tipo de PropTypes:

Si agregamos .isRequired, la propiedad se considera obligatoria y se generará un error si no se proporciona.

Si no agregamos .isRequired, la propiedad se considera opcional y no se generará un error si no se proporciona.

Utilizar PropTypes en React es una práctica recomendada para validar las propiedades que se pasan a los componentes.


Esto mejora la calidad del código, facilita la detección de errores y garantiza que los componentes reciban las propiedades correctas y en el formato adecuado.

Rutas en React


Al organizar el contenido de una aplicación y mostrar solo lo que el usuario ha solicitado, el enrutamiento permite experiencias de usuario enriquecedoras, atractivas y claras.

Definición: Enrutamiento

Es el proceso mediante el cual una aplicación web utiliza la URL actual del navegador (Localizador Uniforme de Recursos) para determinar qué contenido mostrar al usuario.

Ejemplo

Un usuario que visita la página /wiki/Node.js de Wikipedia esperaría ver algo diferente a la página /wiki/React_(JavaScript_library).


Antes de adentrarnos en el funcionamiento de las rutas en React, revisemos brevemente la estructura básica de las URL. Considera esta URL:

Cada URL es esencialmente una solicitud de algún recurso y cada componente de la URL sirve para especificar qué recurso se desea.


Las URL constan de varios componentes, algunos de los cuales son obligatorios y otros opcionales:

El esquema (por ejemplo, HTTP, HTTPS, mailto, etc.), que especifica qué protocolo se debe utilizar para acceder al recurso.

El dominio (por ejemplo, google.com), que especifica el sitio web que aloja el recurso. El dominio sirve como punto de entrada para la aplicación.

La ruta (por ejemplo, /pages), que identifica el recurso específico o la página que se cargará y se mostrará al usuario. Aquí es donde comienza el enrutamiento.

La cadena de consulta opcional (por ejemplo, ?id=1), que aparece después de un '?' y asigna valores a los parámetros. Los usos comunes de las cadenas de consulta incluyen parámetros de búsqueda y filtros.


Dependiendo del tipo de aplicación que construyamos, hay diferentes formas de manejar las solicitudes que llegan al servidor. Una solución de back-end popular es utilizar el framework de enrutamiento Express.

En esta unidad aprenderemos React Router, una solución de enrutamiento de front-end popular diseñada específicamente para aplicaciones React.


React Router


Esta biblioteca facilita la creación de aplicaciones de una sola página (SPA) y mejora la experiencia del usuario al permitir una navegación fluida y sin recargas de página.

Definición: React Router

Es una biblioteca de enrutamiento para aplicaciones web desarrolladas con React. Nos permite manejar la navegación en nuestra aplicación de manera declarativa, utilizando componentes específicos de React Router.


Algunas de las ventajas de utilizar React Router en nuestras aplicaciones React son:

Facilita la navegación: React Router nos proporciona componentes como <BrowserRouter>, <Route> y <Link>, que nos permiten crear enrutamiento fácilmente y mantener nuestra aplicación organizada y estructurada.

Componentes dinámicos: React Router permite la carga dinámica de componentes basados en la URL actual, lo que hace que nuestras aplicaciones sean más rápidas y eficientes en términos de rendimiento.

Rutas anidadas y parametrizadas: React Router ofrece soporte para rutas anidadas y rutas con parámetros, lo que facilita la implementación de aplicaciones con estructuras de navegación más complejas.

Enrutamiento programático: Con React Router, podemos realizar enrutamiento programático, lo que nos permite cambiar dinámicamente las rutas y navegar entre páginas en respuesta a eventos o acciones del usuario.

Rutas anidadas y dinámicas


Las rutas anidadas son útiles cuando queremos representar una jerarquía de componentes en nuestra aplicación, mientras que las rutas dinámicas nos permiten acceder a recursos específicos.

Definición: Ruta anidada

Es una ruta dentro de otra ruta. Una Ruta que contiene una o más Rutas anidadas dentro de ella se conoce como ruta padre y una Ruta que está contenida dentro de otra Ruta se conoce como ruta hija. Cuando se anidan rutas, la ruta de la ruta hija es relativa a la ruta de la ruta padre, por lo que no debemos incluir la ruta padre en su path prop.

Hasta ahora, todas las rutas que hemos cubierto han sido estáticas, lo que significa que coinciden con una única ruta.

Resumen

Las rutas anidadas son aquellas que contienen otras rutas dentro de ellas. Esto es especialmente útil cuando queremos representar una estructura jerárquica en nuestra aplicación, donde un componente puede tener varios subcomponentes asociados a diferentes rutas.


Esto funciona para rutas predeterminadas que muestran una vista consistente. Pero, ¿qué pasa si necesitamos construir una ruta que sea más flexible?

Ejemplo

Imaginemos un sitio de noticias tecnológicas donde cada artículo es accesible desde la ruta '/artículos/' + algún-título donde algún-título es un ID (identificador único) para cada artículo.


Especificar una ruta distinta para cada artículo no sería prolijo y llevaría mucho tiempo, además se que requeriría un mantenimiento poco práctico en caso de que la estructura de rutas cambiara o tuviéramos que añadir nuevos artículos.

Sería mucho más conveniente definir una única ruta que pueda coincidir con cualquier ruta con el patrón '/articles/' + someTitle y saber exactamente qué componente renderizar. React Router nos permite hacer esto mediante el uso de parámetros de URL para crear rutas dinámicas.

Resumen

Las rutas dinámicas nos permiten incluir parámetros en nuestras rutas que representan recursos específicos.


Si nuestra aplicación muestra información sobre usuarios y cada usuario tiene un identificador único, podríamos utilizar rutas parametrizadas para acceder a la información de un usuario específico a través de su identificador.

React Router

Outlet

Es un componente especial proporcionado por React Router V6 que se utiliza en combinación con rutas anidadas. Su función principal es actuar como un marcador de posición para renderizar subrutas dentro de un componente de ruta padre.

Cuando tenemos una estructura de rutas anidadas, el componente padre de la ruta puede contener contenido adicional, como encabezados, pies de página, menús, etc., que deben mostrarse independientemente del componente de subruta activo.

Outlet permite definir el lugar exacto donde se debe renderizar el componente de la subruta activa dentro del componente padre.


Cuando el navegador navega a una ruta anidada, React Router busca el componente Outlet más cercano en el árbol de componentes del componente de ruta padre y renderiza el componente de la subruta activa en ese lugar.

          element={<InicioSesion setNombreUsuario={setNombreUsuario} />}

        />

        <Route

          path="/saludo"

          element={<Saludo nombreUsuario={nombreUsuario} />}

        />

      </Routes>

    </BrowserRouter>

  );

}

El componente App es el componente principal de nuestra aplicación.

Utilizamos el hook useState para manejar el estado del nombre de usuario.

Definimos las rutas utilizando los componentes Routes y Route.

Establecemos dos rutas: la ruta de inicio de sesión (/) y la ruta de saludo (/saludo).

function InicioSesion({ setNombreUsuario }) {

  const [nombre, setNombre] = useState("");

  const navigate = useNavigate();

  const handleSubmit = (e) => {

    e.preventDefault();

    setNombreUsuario(nombre);

    navigate("/saludo");

  };

        <input

          type="text"

          value={nombre}

      <button type="submit">Iniciar sesión</button>

    </form>

El componente InicioSesion muestra un formulario que permite al usuario ingresar su nombre. Cuando se envía el formulario, actualizamos el estado nombreUsuario en el componente App mediante la función setNombreUsuario. Además utilizamos el hook useNavigate para redirigir al usuario a la página /saludo.

Definición: UseNavigate

Es un hook proporcionado por React Router que facilita la navegación programática dentro de una aplicación React. En lugar de utilizar el componente <Navigate> para manejar redirecciones basadas en el estado o en acciones específicas del usuario, podemos utilizar useNavigate para navegar a diferentes rutas de forma imperativa.


El hook useNavigate se utiliza en componentes que necesitan redirigir al usuario a otra ruta como resultado de una acción, como enviar un formulario, hacer clic en un botón o completar una tarea específica.

JavaScript Copiar código 
 
                            
function Saludo({ nombreUsuario }) {

  if (!nombreUsuario) {

    return <Navigate to="/" />;

  }

  return <h1>Hola, {nombreUsuario}!</h1>;

}

El componente Saludo se encarga de mostrar un saludo al usuario si ha iniciado sesión. Si el nombreUsuario está vacío, significa que el usuario no ha iniciado sesión, por lo que utilizamos el componente <Navigate> para redirigir al usuario a la página de inicio de sesión. De lo contrario, mostramos un mensaje de saludo con el nombre del usuario.

Podemos observar que al ingresar a la aplicación veremos el componete Inicio con el formulario para iniciar sesión con nuestro nombre. También podemos comprobar que si intentamos navegar a http://localhost:3000/saludo, sin haber ingresado un nombre, automáticamente se nos redirige nuevamente a http://localhost:3000, es decir, al componente Inicio. Si ponemos un nombre e iniciamos sesión se nos dirigirá a la página /saludo que mostrará el saludo correspondiente.


Parámetros de consulta


Los parámetros de consulta aparecen en las URL comenzando con un signo de interrogación (?) y son seguidos por un nombre de parámetro asignado a un valor. Son opcionales y se utilizan principalmente para buscar, ordenar y/o filtrar recursos.

Por ejemplo, si visitamos la siguiente URL:

https://www.google.com/search?q=argentina

Seremos dirigidos a la página de búsqueda de Google mostrando resultados para el término de búsqueda "argentina". En este ejemplo, el nombre del parámetro de consulta es "q".

Tip

Parámetros de consulta

Pueden ser útiles para determinar qué contenido mostrar al usuario y React Router proporciona un mecanismo para obtener los valores de los parámetros de consulta con el hook useSearchParams(). Este hook devuelve un objeto URLSearchParams y una función que podemos utilizar para actualizarlo.

A continuación vamos a aprender cómo personalizar las rutas y manejar errores en una aplicación React utilizando React Router. Esto nos permitirá ofrecer una mejor experiencia al usuario al tratar de acceder a rutas inexistentes o al manejar posibles errores en nuestra aplicación.

React Router V6 nos permite personalizar nuestras rutas de diferentes maneras.

Manejo de errores y rutas no encontradas


Es importante manejar correctamente las rutas no encontradas o errores en nuestra aplicación. Con React Router V6, podemos hacer esto de manera sencilla. Veamos cómo crear un componente NotFound para manejar rutas no encontradas.

Ejemplo

NotFoundPage se utilizará para renderizar una página personalizada cuando se accede a una ruta no encontrada en nuestra aplicación. El componente renderiza un título, un mensaje y un enlace que permite al usuario volver a la página de inicio de nuestra aplicación.

JavaScript Copiar código 
 
                            
function NotFoundPage() {

  return (

    <div>

      <h1>Oops!</h1>

      <p>No hemos encontrado la página que estás buscando.</p>

      <Link to="/">Volver a la página de inicio</Link>

    </div>

  );

}

Luego utilizamos el componente NotFoundPage en el componente App de la aplicación:

JavaScript Copiar código 
 
                            
function App() {

  return (

    <Router>

      <Routes>

        <Route path="/" element={<Inicio />} />

        <Route path="/about" element={<AcercaDe />} />

        <Route path="*" element={<NotFoundPage />} />

      </Routes>

    </Router>

  );

}

Dentro de Routes, estamos utilizando Route para definir nuestras rutas. En este caso, estamos definiendo tres rutas diferentes:

La ruta de la página de inicio: Se activará cuando la URL sea la raíz de nuestro sitio ("/"). Esta ruta utiliza el componente Inicio.

La ruta de la página Acerca de: Se activará cuando la URL sea "/about". Esta ruta utiliza el componente AcercaDe.

La ruta "catch-all": Se utilizará para manejar todas las rutas no encontradas. Esta ruta utiliza el componente NotFoundPage.
