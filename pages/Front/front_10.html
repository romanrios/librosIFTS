Introducción


En esta sección, vamos a introducirnos en el mundo de los formularios en React. Los formularios son una parte fundamental de cualquier aplicación web, ya que nos permiten recopilar información del usuario para procesarla y realizar acciones específicas, como registrarse en una plataforma, enviar comentarios o realizar una compra en línea.

Los formularios en aplicaciones web son una herramienta clave para recopilar y enviar información ingresada por los usuarios.

Definición: Formulario

Consiste en diversos campos, como cuadros de texto, casillas de verificación, botones de opción y listas desplegables, que el usuario puede interactuar para ingresar datos. 
Una vez que el usuario ha proporcionado la información necesaria, puede enviar el formulario, y los datos se procesan en el servidor o en la aplicación del cliente.

 Formularios: React vs. HTML


En una aplicación HTML estática, los formularios se crean usando etiquetas HTML como <form>, <input>, <textarea>, <select> y <button>. Cuando se envía un formulario en HTML, generalmente los datos se envían al servidor para su procesamiento, utilizando el atributo action y method en la etiqueta <form>.

En cambio, en aplicaciones React, los formularios se manejan de manera diferente. Aunque todavía se utilizan las mismas etiquetas HTML, el flujo de datos y la lógica de procesamiento se implementan utilizando componentes y estados de React. Esto permite un mayor control sobre el comportamiento del formulario y permite realizar acciones como la validación en tiempo real y el manejo de errores de manera más eficiente.

Ejemplo de formulario en HTML:

HTML Copiar código 
 
                             
<form action="/submit" method="POST">

  <label for="name">Nombre:</label>

  <input type="text" id="name" name="name" required>

  <br>

  <label for="email">Correo electrónico:</label>

  <input type="email" id="email" name="email" required>

  <br>

  <button type="submit">Enviar</button>

</form>

Ejemplo de formulario en React:

JavaScript Copiar código 
 
                            
import React, { useState } from "react";



function Formulario() {

  const [nombre, setNombre] = useState("");

  const [correo, setCorreo] = useState("");

  const handleSubmit = (e) => {

    e.preventDefault();

    // Procesamiento de los datos, puede ser enviándolos a un servidor o manejándolos dentro de la aplicación

  };



  return (

    <form onSubmit={handleSubmit}>

      <label htmlFor="name">Nombre:</label>

      <input

        type="text"

        id="name"

        value={nombre}

        onChange={(e) => setNombre(e.target.value)}

        required

      />

      <br />

      <label htmlFor="email">Correo electrónico:</label>

      <input

        type="email"

        id="email"

        value={correo}

        onChange={(e) => setCorreo(e.target.value)}

        required

      />

      <br />

      <button type="submit">Enviar</button>

    </form>

  );

}



export default Formulario;

En este ejemplo, podemos observar cómo en React utilizamos el estado y las funciones de manejo de eventos para controlar y procesar los datos del formulario en lugar de confiar en la acción y el método de la etiqueta <form> en HTML.


Parámetro "e"


En React, el parámetro e en la función manejadora de eventos de entrada (input event handler) hace referencia al objeto evento (event object) que se pasa como argumento a la función cuando se dispara un evento de entrada, como puede ser un evento de cambio (change event) o un evento de entrada de teclado (keyboard input event).

Definición: Event object

El objeto evento contiene información sobre el evento que ha sido desencadenado, como por ejemplo la tecla que ha sido pulsada o el valor del elemento de entrada (input element) que ha sido modificado.

Al utilizar el parámetro e en la función manejadora de eventos, podemos acceder a esta información y realizar acciones en consecuencia, como actualizar el estado de nuestro componente React o realizar alguna otra operación lógica en nuestra aplicación.


Componentes controlados y no controlados


En esta parte, vamos a explorar dos enfoques distintos para manejar formularios en React: componentes controlados y no controlados. Ambos enfoques tienen sus ventajas y desventajas, y es importante comprender las diferencias entre ellos para elegir el enfoque más adecuado para nuestras aplicaciones.

Resumen

Un componente no controlado es un componente que mantiene su propio estado interno.


Pensemos en un elemento típico de <input type='text' />. Aparece en la pantalla como una caja de texto. Si necesitamos saber qué texto se encuentra actualmente en la caja, podemos preguntarle al <input /> , posiblemente con algo de código como este:

JavaScript Copiar código 
                            

let input = document.querySelector('input[type="text"]');

let typedText = input.value;

Input.value será igual a cualquier texto que se encuentre actualmente en la caja de texto.


Lo importante aquí es que el <input /> mantiene un seguimiento de su propio texto. Podemos preguntarle qué texto tiene en cualquier momento y él nos lo dirá. El hecho de que el <input /> mantenga un seguimiento de la información que contiene, lo convierte en un componente no controlado.

Por otro lado, un componente controlado no tiene memoria. Si le preguntamos por información sobre sí mismo, tendrá que obtener esa información a través de las props. La mayoría de los componentes en React son controlados.

Resumen

Un componente controlado es un componente que no mantiene ningún estado interno. Como un componente controlado no tiene estado, debe ser controlado por alguien más.


En React, cuando le damos al <input /> un atributo de valor (value attribute), ocurre algo interesante: el <input /> SE CONVIERTE en un componente controlado. Deja de usar su almacenamiento interno. Esta es una forma más "React" de hacer las cosas.


Componentes controlados


Al utilizar componentes controlados, React tiene el control total sobre los datos del formulario y su comportamiento.

Definición: Componentes controlados

Son aquellos en los que el estado del formulario y su lógica de manejo están completamente bajo el control de React. 

En este enfoque, el valor de cada campo del formulario se almacena en el estado del componente y se actualiza mediante eventos de cambio (onChange).

JavaScript Copiar código 
 
                            
import React, { useState } from "react";



function FormularioControlado() {

  const [nombre, setNombre] = useState("");

  const handleChange = (e) => {

    setNombre(e.target.value);

  };

  return (

    <div>

      <label htmlFor="nombre">Nombre:</label>

      <input type="text" id="nombre" value={nombre} onChange={handleChange} />

    </div>

  );

}



export default FormularioControlado;

Este es un formulario controlado básico. En este caso, el formulario solo tiene un campo de entrada de texto para que el usuario ingrese su nombre.

1

Declaramos una constante llamada [nombre, setNombre]: La inicializamos con el valor de cadena vacía utilizando useState. setNombre es la función que nos permite actualizar el valor del estado de nombre.

2

Luego, definimos una función llamada handleChange: Esta función se llama cada vez que el valor en el campo de entrada cambia. Dentro de handleChange, utilizamos la función setNombre para actualizar el estado del componente con el valor actual del campo de entrada.

3

El valor del campo de entrada se establece en el valor del estado de nombre utilizando la propiedad value: Se le asigna la función handleChange al evento onChange; onChange es un evento que se dispara cada vez que el valor del campo de entrada cambia.

Ventajas de los componentes controlados

Mayor control sobre los datos y el comportamiento del formulario.

Posibilidad de realizar validaciones y actualizaciones en tiempo real.

Facilita la implementación de lógica compleja en el formulario.

Desventajas de los componentes controlados

Mayor cantidad de código y complejidad en el manejo de eventos y estado.

Posible impacto en el rendimiento debido a la actualización constante del estado.


Componentes no controlados


En los componentes no controlados, React no tiene control directo sobre el estado y la lógica de manejo del formulario.

En lugar de almacenar el valor de cada campo en el estado de React, los componentes no controlados utilizan referencias (refs) para acceder a los valores de los campos del formulario cuando se necesita, como al enviar el formulario.

JavaScript Copiar código 
 
                            
import React, { useRef } from "react";



function FormularioNoControlado() {

  const nombreRef = useRef();

  const handleSubmit = (e) => {

    e.preventDefault();

    console.log(nombreRef.current.value);

  };

  return (

    <form onSubmit={handleSubmit}>

      <label htmlFor="nombre">Nombre:</label>

      <input type="text" id="nombre" ref={nombreRef} />

      <button type="submit">Enviar</button>

    </form>

  );

}



export default FormularioNoControlado;

Este ejemplo muestra un formulario no controlado en React. Esto significa que el estado del formulario no está siendo controlado por React, sino que los valores de los campos de entrada son manejados directamente por el DOM.

1

Importamos las dependencias necesarias de React: En este caso, el hook useRef. Luego, definimos una función llamada FormularioNoControlado.

2

Dentro de la función: Creamos una referencia utilizando el hook useRef y la asignamos a la variable nombreRef. Esta referencia se utiliza para obtener el valor del campo de entrada en el momento del envío del formulario.

3

Creamos una función llamada handleSubmit: Se activa cuando se envía el formulario. El evento preventDefault() se llama para evitar que el formulario se envíe automáticamente y recargue la página. En su lugar, obtenemos el valor actual del campo de entrada utilizando la referencia nombreRef y lo registramos en la consola.

4

Creamos un formulario y un campo de entrada: Asignándole el atributo ref con el valor de nombreRef. También añadimos un botón de envío para enviar el formulario.


useRef()


El primer ejemplo que vamos a ver es cómo usar useRef para enfocar un elemento.

Definición: UseRef

Es un Hook que permite tener acceso a los nodos del DOM o a cualquier otro valor que persista en el tiempo. Esto es útil para realizar operaciones como enfocar un elemento, cambiar su valor, etc.

Ejemplo

Supongamos que tenemos un formulario y queremos que el usuario empiece a escribir en el primer campo automáticamente. Para lograr esto, podemos usar el método focus del nodo del DOM:

JavaScript Copiar código 
 
                            
import React, { useRef } from "react";


function Formulario() {

  const inputRef = useRef(null);

  const handleButtonClick = () => {

    inputRef.current.focus();

  };

  return (

    <div>

      <label htmlFor="campo">Campo:</label>

      <input type="text" id="campo" ref={inputRef} />

      <button onClick={handleButtonClick}>Enfocar</button>

    </div>

  );

}



export default Formulario;

Aquí hemos creado una referencia al input utilizando useRef. Luego, cuando se hace clic en el botón, llamamos al método focus del input.

Ejemplo

Obtener el valor de un input sin utilizar el estado. Imaginemos que tenemos un formulario que se envía al hacer clic en un botón. Podríamos hacer lo siguiente:

JavaScript Copiar código 
 
                            
import React, { useRef } from "react";


function Formulario() {

  const inputRef = useRef(null);

  const handleButtonClick = () => {

    console.log(inputRef.current.value);

  };

  return (

    <div>

      <label htmlFor="campo">Campo:</label>

      <input type="text" id="campo" ref={inputRef} />

      <button onClick={handleButtonClick}>Enviar</button>

    </div>

  );

}



export default Formulario;

Aquí hemos creado una referencia al input y cuando se hace clic en el botón, obtenemos el valor del input utilizando la propiedad value.

UseRef también puede ser utilizado para mantener valores que persistan en el tiempo.

Ejemplo

Supongamos que queremos mantener una lista de elementos que se agregan al hacer clic en un botón. Podríamos hacer lo siguiente:

JavaScript Copiar código 
 
                            
import React, { useState, useRef } from "react";



function Lista() {

  const [lista, setLista] = useState([]);

  const inputRef = useRef(null);

  const handleButtonClick = () => {

    const nuevoElemento = inputRef.current.value;

    setLista([...lista, nuevoElemento]);

    inputRef.current.value = "";

  };

  return (

    <div>

      <ul>

        {lista.map((elemento, indice) => (

          <li key={indice}>{elemento}</li>

        ))}

      </ul>

      <input type="text" ref={inputRef} />

      <button onClick={handleButtonClick}>Agregar</button>

    </div>

  );

}



export default Lista;

Aquí hemos creado una referencia al input y cuando se hace clic en el botón, obtenemos el valor del input y lo agregamos a la lista utilizando la función setLista. También hemos utilizado la referencia al input para limpiar su valor después de agregar un elemento a la lista.


Ventajas y desventajas


Al evaluar los enfoques de componentes controlados y no controlados, es esencial tener en cuenta las necesidades específicas de nuestra aplicación y los requisitos de nuestros formularios.

Ventajas de los componentes no controlados

Menor cantidad de código y menos complejidad en el manejo de eventos y estado.

Mejor rendimiento al no depender de actualizaciones constantes del estado.

Similitud con el enfoque de manejo de formularios en HTML puro.

Desventajas de los componentes no controlados

Menor control sobre los datos y el comportamiento del formulario.

Dificultad para realizar validaciones y actualizaciones en tiempo real.

Menos adecuado para la implementación de lógica compleja en el formulario.

Resumen

Los componentes controlados son más adecuados cuando se necesita un mayor control sobre el comportamiento del formulario y la validación en tiempo real, mientras que los componentes no controlados pueden ser una buena opción si se busca simplicidad y un mejor rendimiento en casos en los que la validación y la lógica del formulario no son críticas.


Ambos enfoques tienen sus ventajas y desventajas, y no hay una solución única que funcione para todos los casos. Es importante comprender las diferencias entre ellos y elegir el enfoque que mejor se adapte a nuestras necesidades y preferencias.

A continuación profundizaremos en el manejo de eventos y estados en formularios, la creación de componentes de formulario personalizados y cómo implementar la validación en tiempo real. Estos conceptos nos ayudarán a construir formularios más sólidos y efectivos en nuestras aplicaciones React, independientemente del enfoque que elijamos.



Manejo de estados y eventos


A continuación, nos enfocaremos en el manejo de estados y eventos en formularios en aplicaciones React.

Los eventos y estados son fundamentales para gestionar y controlar el comportamiento y la interacción del usuario con los formularios en nuestras aplicaciones.


El estado en React nos permite almacenar y gestionar la información en nuestros componentes. En el caso de los formularios, utilizamos el estado para mantener y actualizar los valores de los campos del formulario a medida que el usuario interactúa con ellos. El estado nos permite mantener un registro actualizado de los datos ingresados por el usuario y realizar acciones basadas en estos datos, como validaciones y envío de formularios.

Ejemplo

Uso de estados en un componente de formulario.

JavaScript Copiar código 
 
                            
import React, { useState } from "react";



function Formulario() {

  const [nombre, setNombre] = useState("");

  const [email, setEmail] = useState("");

  const handleChangeNombre = (e) => {

    setNombre(e.target.value);

  };

  const handleChangeEmail = (e) => {

    setEmail(e.target.value);

  };

  return (

    <div>

      <label htmlFor="nombre">Nombre:</label>

      <input

        type="text"

        id="nombre"

        value={nombre}

        onChange={handleChangeNombre}

      />

      <label htmlFor="email">Email:</label>

      <input

        type="email"

        id="email"

        value={email}

        onChange={handleChangeEmail}

      />

    </div>

  );

}



export default Formulario;

Eventos de formularios


Los eventos en formularios nos permiten detectar y responder a las acciones del usuario, como ingresar datos, seleccionar elementos o enviar el formulario. Algunos de los eventos de formulario más comunes en React son:

OnChange: Se activa cuando el valor de un campo del formulario cambia. Este evento se utiliza comúnmente para actualizar el estado y realizar validaciones en tiempo real.

OnSubmit: Se activa cuando se envía el formulario. Se utiliza para procesar y validar los datos del formulario antes de enviarlos al servidor o realizar otras acciones en la aplicación.

OnFocus: Se activa cuando un campo del formulario recibe el foco. Puede utilizarse para mostrar información adicional o realizar acciones específicas.

OnBlur: Se activa cuando un campo del formulario pierde el foco. A menudo se utiliza para realizar validaciones o actualizar el estado cuando el usuario termina de interactuar con un campo.

Ejemplo

JavaScript Copiar código 
 
                            
import React, { useState } from "react";



function Formulario() {

  const [nombre, setNombre] = useState("");

  const [email, setEmail] = useState("");

  const handleChange = (e) => {

    const { id, value } = e.target;

    if (id === "nombre") {

      setNombre(value);

    } else if (id === "email") {

      setEmail(value);

    }

  };

  const handleSubmit = (e) => {

    e.preventDefault();

    // Procesar y validar los datos del formulario

    console.log({ nombre, email });

  };

  const handleFocus = (e) => {

    console.log(`${e.target.id} ha recibido el foco`);

  };

  const handleBlur = (e) => {

    console.log(`${e.target.id} ha perdido el foco`);

  };

  return (

    <form onSubmit={handleSubmit}>

      <label htmlFor="nombre">Nombre:</label>

      <input

        type="text"

        id="nombre"

        value={nombre}

        onChange={handleChange}

        onFocus={handleFocus}

        onBlur={handleBlur}

      />{" "}

      <label htmlFor="email">Email:</label>

      <input

        type="email"

        id="email"

        value={email}

        onChange={handleChange}

        onFocus={handleFocus}

        onBlur={handleBlur}

      />

      <button type="submit">Enviar</button>

    </form>

  );

}



export default Formulario;

En este ejemplo, hemos utilizado los eventos `onChange`, `onSubmit`, `onFocus` y `onBlur` para manejar la interacción del usuario con el formulario y realizar acciones específicas en cada caso.

El manejo adecuado de eventos y estados nos permite crear formularios interactivos y dinámicos que ofrecen una experiencia de usuario óptima y facilitan la recopilación y validación de datos en nuestras aplicaciones React.


Propagación de eventos


Al trabajar con eventos en formularios de React, es importante comprender cómo se propagan los eventos y cómo se actualizan los estados en respuesta a dichos eventos.

Definición: Propagación de eventos

Se refiere al proceso mediante el cual un evento se propaga a través de la jerarquía de componentes, desde el componente que generó el evento hasta los componentes ancestros.

En React, los eventos se propagan de manera similar a cómo lo hacen en el DOM (Document Object Model).

Cuando un evento ocurre en un elemento, como un clic o un cambio en un campo de formulario, el evento se propaga desde el elemento que generó el evento (el objetivo) hasta el componente raíz de la aplicación.


Por defecto, los eventos en React se propagan en dos fases: captura y burbujeo.

Fase de captura: El evento se propaga desde el componente raíz hacia el objetivo.

Fase de burbujeo: El evento se propaga desde el objetivo de vuelta al componente raíz.

Podemos detener la propagación de un evento en cualquier momento utilizando el método stopPropagation() del objeto del evento.


En algunos casos, es posible que deseemos detener la propagación de eventos para evitar que los eventos afecten a componentes ancestros o para mejorar el rendimiento de la aplicación al evitar el procesamiento innecesario de eventos.


Actualización de estados


La actualización de estados en React es un proceso esencial para mantener la información relevante y actualizada en nuestros componentes. En el caso de los formularios, esto significa actualizar el estado cuando el usuario interactúa con los campos del formulario, como ingresar datos o seleccionar elementos.

Definición: Actualización de estados

Se refiere al proceso de modificar el estado de un componente en función de los eventos y las interacciones del usuario.


Cuando manejamos eventos en formularios de React, a menudo utilizamos los eventos onChange, onBlur, onFocus y onSubmit para actualizar el estado y realizar acciones basadas en los datos del formulario.

Resumen

Al comprender la propagación de eventos y la actualización de estados en React, podemos crear formularios más eficientes y dinámicos que respondan a las interacciones del usuario y se adapten a las necesidades de nuestras aplicaciones.


Al actualizar el estado en respuesta a estos eventos, podemos controlar el comportamiento y la apariencia de nuestros formularios de manera dinámica y ofrecer una experiencia de usuario interactiva y atractiva.

Ejemplo

Propagación de eventos y actualización de estados en un componente de formulario:

JavaScript Copiar código 
 
                            
import React, { useState } from 'react';



function Formulario() {

  const [nombre, setNombre] = useState('');

  const [email, setEmail] = useState('');

  const handleChange = (e) => {

    e.stopPropagation(); // Detener la propagación del evento

    const { id, value } = e.target;

    if (id === 'nombre') {

      setNombre(value); // Actualizar el estado

    } else if (id === 'email') {

      setEmail(value); // Actualizar el estado

    }

  };

  return (

    <div>

      <label htmlFor="nombre">Nombre:</label>

      <input type="text" id="nombre" value={nombre} onChange={handleChange} />

      <label htmlFor="email">Email:</label>

      <input type="email" id="email" value={email} onChange={handleChange} />

    </div>

  );

}



export default Formulario;

En este ejemplo:

Hemos utilizado el evento onChange: Para actualizar el estado en función de las interacciones del usuario con los campos del formulario.

Hemos utilizado el método stopPropagation(): Para detener la propagación del evento onChange, evitando así que el evento afecte a los componentes ancestros.

Esto puede ser útil en casos en los que otros componentes de la jerarquía también estén escuchando y respondiendo a eventos similares, y no queremos que se activen en respuesta a eventos de nuestros campos de formulario.


Formulario: Componentes personalizados


En esta parte, exploraremos cómo crear componentes de formulario personalizados en React.

Definición: Componentes personalizados

Nos permiten diseñar campos de formulario reutilizables y fácilmente personalizables, mejorando la eficiencia y la consistencia en nuestras aplicaciones.

Al diseñar componentes de formulario personalizados, es importante tener en cuenta la reutilización y la flexibilidad. Queremos crear componentes que sean fáciles de reutilizar en diferentes contextos y que se puedan personalizar para adaptarse a las necesidades específicas de cada caso de uso.

Tip

Props y estados

Podemos utilizarlos para controlar el comportamiento, la apariencia y los datos de nuestros componentes de formulario personalizados.


Formularios personalizados: Props

Validación de formularios en tiempo real


En esta sección, exploraremos cómo implementar la validación de formularios en tiempo real en React.

Definición: Validación en tiempo real

Nos permite verificar los datos ingresados por el usuario mientras interactúa con el formulario, proporcionando una experiencia de usuario más rápida y receptiva.

La validación en tiempo real es importante por varias razones:

Mejora la experiencia del usuario: Al proporcionar retroalimentación inmediata sobre la validez de los datos ingresados, los usuarios pueden corregir errores antes de enviar el formulario, lo que resulta en una experiencia más fluida y menos frustrante.

Reduce la carga en el servidor: Al validar los datos en el lado del cliente antes de enviarlos al servidor, podemos reducir la cantidad de solicitudes inválidas que se envían al servidor y, por lo tanto, disminuir la carga en el servidor.

Aumenta la seguridad: Si bien la validación en tiempo real en el lado del cliente no puede reemplazar completamente la validación en el lado del servidor, puede ayudar a identificar y prevenir intentos de entrada de datos maliciosos o inválidos.

Para implementar la validación en tiempo real en React, utilizamos eventos y estados en nuestros componentes de formulario.

En particular, nos centraremos en el evento onChange, que se activa cada vez que el usuario interactúa con un campo de formulario.


Ejemplo

Cómo implementar la validación en tiempo real en un formulario de React.

JavaScript Copiar código 
 
                            
import React, { useState } from "react";



function Formulario() {

  const [nombre, setNombre] = useState("");

  const [nombreError, setNombreError] = useState("");

  const handleChangeNombre = (e) => {

    const valor = e.target.value;

    setNombre(valor);

    if (valor.length < 3) {

      setNombreError("El nombre debe tener al menos 3 caracteres");

    } else {

      setNombreError("");

    }

  };

  return (

    <div>

      <label htmlFor="nombre">Nombre:</label>

      <input

        type="text"

        id="nombre"

        value={nombre}

        onChange={handleChangeNombre}

      />

      {nombreError && <div className="error">{nombreError}</div>}

    </div>

  );

}



export default Formulario;

En este ejemplo:

Hemos creado un estado nombreError para almacenar el mensaje de error de validación.

Lo hemos actualizado en función de la longitud del valor ingresado en el campo de nombre.

Si el nombre tiene menos de 3 caracteres, mostramos un mensaje de error. De lo contrario, no mostramos ningún mensaje de error.

Al implementar la validación en tiempo real en nuestros formularios de React, podemos proporcionar una experiencia de usuario mejorada y aumentar la eficiencia y la seguridad de nuestras aplicaciones.

Seguiremos explorando técnicas y conceptos relacionados con el manejo de formularios en React, lo que nos ayudará a construir formularios sólidos y efectivos que se adapten a las necesidades de nuestras aplicaciones y usuarios.


Librerías de validación en React


Aunque la validación en tiempo real utilizando eventos y estados en React es una forma efectiva de validar los datos del formulario, existen librerías de validación que nos pueden facilitar el trabajo y mejorar la eficiencia y consistencia de nuestras aplicaciones.

Librerías populares de validación



Yup: Es una librería de validación de objetos JavaScript ligera y fácil de usar que nos permite definir esquemas de validación de forma declarativa. Es una opción popular para la validación en aplicaciones React y se integra fácilmente con otras librerías como Formik.

Joi: Es otra librería de validación de objetos JavaScript que nos permite definir esquemas de validación de forma declarativa. Joi es más antigua que Yup y ofrece una amplia variedad de opciones de validación y personalización.

Utilizaremos Yup, aunque ambas librerías son excelentes opciones para la validación en aplicaciones React, y la elección entre ellas dependerá de las preferencias personales y los requisitos específicos de la aplicación.


Yup: Usos e instalación


Es una biblioteca de JavaScript para la validación de esquemas de datos.

Objetivos: Yup

Garantizar que los datos ingresados por los usuarios en un formulario cumplan con ciertas reglas y restricciones antes de ser procesados.

Facilita la definición y verificación de esquemas de validación de una manera simple y declarativa.

Ventajas de utilizar Yup en la validación de formularios en React:

JavaScript Copiar código 
 
                            
import React, { useState } from "react";

import * as Yup from "yup";



const SignupForm = () => {

  // Crear el esquema de validación

  const validationSchema = Yup.object().shape({

    username: Yup.string()

      .required("El nombre de usuario es obligatorio")

      .min(3, "El nombre de usuario debe tener al menos 3 caracteres"),

    email: Yup.string()

      .required("El correo electrónico es obligatorio")

      .email("Ingrese un correo electrónico válido"),

    password: Yup.string()

      .required("La contraseña es obligatoria")

      .min(8, "La contraseña debe tener al menos 8 caracteres")

  });



  // Estados para los campos del formulario y los errores de validación

  const [values, setValues] = useState({

    username: "",

    email: "",

    password: ""

  });

  const [errors, setErrors] = useState({});



  // Función para manejar los cambios en los campos del formulario

  const handleChange = (event) => {

    const { name, value } = event.target;

    setValues({ ...values, [name]: value });

  };



  // Función para manejar el envío del formulario

  const handleSubmit = async (event) => {

    event.preventDefault();

    try {

      // Validar los datos del formulario utilizando el esquema de validación

      await validationSchema.validate(values, { abortEarly: false });

      // Procesar los datos del formulario (por ejemplo, enviarlos a un servidor)

      console.log("Formulario enviado con éxito", values);

    } catch (error) {

      if (error instanceof Yup.ValidationError) {

        // Crear un objeto de errores con las propiedades name de los campos y sus mensajes de error

        const newErrors = error.inner.reduce((acc, curr) => {

          acc[curr.path] = curr.message;

          return acc;

        }, {});

        // Actualizar el estado de errores

        setErrors(newErrors);

      } else {

        console.error("Error inesperado al validar el formulario:", error);

      }

    }

  };



  return (

    <form onSubmit={handleSubmit}>

      <div>

        <label htmlFor="username">Nombre de usuario</label>

        <input

          id="username"

          name="username"

          type="text"

          value={values.username}

          onChange={handleChange}

        />

        {errors.username && <p>{errors.username}</p>}

      </div>

      <div>

        <label htmlFor="email">Correo electrónico</label>

        <input

          id="email"

          name="email"

          type="email"

          value={values.email}

          onChange={handleChange}

        />

        {errors.email && <p>{errors.email}</p>}

      </div>

      <div>

        <label htmlFor="password">Contraseña</label>

        <input

          id="password"

          name="password"

          type="password"

          value={values.password}

          onChange={handleChange}

        />

        {errors.password && <p>{errors.password}</p>}

      </div>

      <button type="submit">Registrarse</button>

    </form>

  );

};



export default SignupForm;

En este ejemplo:

Hemos utilizado el esquema de validación en conjunto con un formulario controlado en React.

Los datos del formulario se validan utilizando el esquema de validación cuando el usuario envía el formulario.

Si la validación falla, se muestra un mensaje de error debajo de cada campo no válido.

Si la validación es exitosa, los datos del formulario se procesan según lo deseado (en este caso, simplemente se imprimen en la consola).

También podemos utilizar el método `when()` para crear validaciones condicionales basadas en el valor de otro campo.

Ejemplo

Imaginemos que tenemos un formulario donde el usuario puede seleccionar un país y, en función de su selección, se debe validar el código postal según diferentes reglas. Podemos utilizar `when()` para lograr esto

      } else if (country === "CA") {

        return schema.matches(

          /^\d{6}$/,

          "El código postal de Canadá debe tener 6 dígitos"

        );

      } else {

        return schema;

      }

   })

});

En este ejemplo:

Utilizamos el método when() para modificar el esquema de validación del campo postalCode en función del valor del campo country.

Si el país es 'US', validamos que el código postal tenga 5 dígitos.

Si el país es 'CA', validamos que el código postal tenga 6 dígitos.

Para otros países, no aplicamos ninguna validación adicional.

Control: errores de validación asíncronos


Yup también admite la validación asíncrona, lo cual es útil cuando necesitamos realizar validaciones que dependan de una solicitud de red, como verificar si un correo electrónico ya está registrado en nuestra base de datos.

Para manejar errores de validación asíncronos, podemos utilizar el método validate() de Yup, que devuelve una promesa.

Ejemplo

Cómo implementar la validación asíncrona.

JavaScript Copiar código 
 
                            
import * as Yup from "yup";



const emailSchema = Yup.string()

  .required("El correo electrónico es obligatorio")

  .email("Debe ser un correo electrónico válido")

  .test(

    "is-unique-email",

    "Este correo electrónico ya está registrado",

    async (value) => {

      // Simulamos una solicitud de red para verificar si el correo electrónico ya está registrado

      const isEmailRegistered = await checkEmailInDatabase(value);

      return !isEmailRegistered;

    }

  );

// Esta función simula una solicitud de red para verificar si un correo electrónico ya está registrado

async function checkEmailInDatabase(email) {

  // Aquí haríamos una solicitud a tu servidor/API para verificar si el correo electrónico ya está registrado

  // Simulamos un retraso de 500 ms en la respuesta

  await new Promise((resolve) => setTimeout(resolve, 500));

  // Para este ejemplo, supondremos que el correo "test@example.com" ya está registrado

  return email === "test@example.com";

}

En este ejemplo:

Agregamos una validación asíncrona llamada is-unique-email al esquema de validación del correo electrónico.

La función de validación realiza una solicitud de red simulada para verificar si el correo electrónico ya está registrado y devuelve false si lo está, mostrando el mensaje de error proporcionado.

Importante

Errores de validación asíncronos

Para manejar errores de validación asíncronos en nuestro componente de React, debemos asegurarnos de utilizar await al llamar al método validate() de Yup y manejar cualquier error utilizando try y catch.
