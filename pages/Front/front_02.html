Introducción


HTML es un lenguaje que describe un documento hipertextual, describe la estructura y el contenido que puede ser texto, imágenes, vídeos y otros componentes de las páginas web.


Con HTML no se define la presentación visual de las páginas web. Esto es muy importante de comprender, el lenguaje HTML se emplea únicamente para definir la estructura y el contenido o información de una página web. La presentación visual no se define con HTML. 


HTML no es exactamente un lenguaje de programación en el sentido tradicional. En lugar de eso, es un lenguaje de marcado que se utiliza para crear y estructurar contenido para la web. HTML proporciona una serie de elementos y etiquetas que se utilizan para definir y dar formato al contenido, como títulos, párrafos, imágenes y enlaces. Aunque HTML no se utiliza para crear programas o aplicaciones, es una parte esencial de la web y se utiliza en conjunto con otros lenguajes de programación para crear sitios web y aplicaciones web interactivas.

Una página web está compuesta por un conjunto de elementos escritos en un orden concreto.


Primera página web


Vamos a crear una primera página web sencilla para ver cómo funciona. En primer lugar, abrimos el editor de código Visual Studio Code (VScode), podemos crear y abrir alguna carpeta específica para esta semana de trabajo.

Dentro de esta carpeta, inicialmente vacía, vamos a crear un archivo llamado “index.html”. Para esto podemos hacer click en el icono para crear un nuevo archivo: 


Vamos a copiar y pegar este código en nuestro archivo index.html.

HTML Copiar código 
          
<!DOCTYPE html>
<html lang="es">
   <head>
       <title>Mi página web</title>
   </head>
   <body>
       <h1>Mi página web</h1>
       <p>Estoy aprendiendo desarrollo web.</p>
   </body>
</html>  
 

A continuación, probemos cómo se muestra la página en nuestro navegador. Para ello, vamos a utilizar la extensión de Visual Studio Code llamada “Live Server” que instalamos la semana anterior (si aún no la tenés instalada, podés buscarla en el marketplace de extensiones o seguir las instrucciones en Live Server).

La extensión Live Server de Visual Studio Code permite la ejecución de un servidor local para visualizar los cambios realizados en tiempo real en el navegador, sin tener que actualizar la página manualmente.


Al crear una página HTML y luego instalar la extensión, debería aparecer un ícono llamado "Go Live" debajo de la barra azul en la interfaz de Visual Studio Code. Si no figura, podemos reiniciar VS Code para que aparezca. 

Al hacer clic en el botón "Go Live", se iniciará el servidor local y se abrirá en nuestro navegador predeterminado.


Otra forma de utilizarlo es, al hacer clic derecho sobre un archivo HTML en el Explorador de Archivos, seleccionar "Open with Live Server".



Sintaxis y semántica


En el contexto de HTML, la semántica y la sintaxis son importantes para el desarrollo de páginas web efectivas y accesibles. El HTML semántico se refiere a la utilización de etiquetas HTML que tienen un significado específico en lugar de simplemente ser utilizadas como un contenedor visual. 

La semántica en HTML permite que los motores de búsqueda, lectores de pantalla y otros dispositivos de asistencia comprendan mejor el contenido de una página web.


Por otro lado, la sintaxis en HTML se refiere a la estructura y reglas que se deben seguir al escribir el código HTML. 

La sintaxis correcta es esencial para que un navegador pueda interpretar correctamente el código HTML y mostrar la página web como se pretende.


Estructura básica de HTML en una página web


La anatomía básica de una página web está compuesta por etiquetas que determinan la estructura de una página o aplicación web.

Normalmente la anatomía básica está compuesta por: un contenedor, un header, el espacio del contenido principal, un sidebar y un footer. Esto no quiere decir que todas las páginas web tienen la misma estructura de composición, pero sí es la más básica.


Container: contenedor principal

Header: cabecera de la página. Aquí usualmente encontrás el logo y el menú de navegación del sitio.

Navigation: En este lugar se encuentran los hipervínculos para navegar en las diferentes páginas del sitio web.

Main content: estructura principal. Generalmente en este espacio se halla la información principal de la página web. Por ejemplo, el feed o lista de publicaciones de una red social.

Sidebar: contenido secundario de una página, que por lo general está a los lados del contenido principal (o main). Es una columna que nos permite visualizar información relevante, pero no más relevante que el contenido principal.

Footer: pie de página. Se encuentra al final del sitio web. En este espacio por lo general se coloca información de pie de página, puede ser una navegación secundaria u otra información menos relevante de la página web, producto o servicio.



Elementos


Un archivo HTML está conformado por elementos HTML. Estos elementos son componentes individuales en HTML que nos permiten crear la estructura de la página.

Ciertos elementos pueden contener a otros elementos y crear estructuras más complejas. Al anidar los elementos uno dentro del otro, se debería indentar los elementos internos para reflejar esta estructura.


DOCTYPE HTML5


Al diseñar una página Web, se indica con un código denominado DOCTYPE html (Document Type Definition DTD), que se sitúa al inicio del documento HTML.


DOCTYPE HTML5 es una declaración que se coloca al comienzo de un documento HTML5 y que indica al navegador web qué tipo de documento se va a cargar. Básicamente, es una instrucción para el navegador para que sepa qué versión de HTML está siendo utilizada en el documento.

La sintaxis de DOCTYPE HTML5 es muy simple. Se escribe como sigue:

HTML Copiar código 
<!DOCTYPE html>


Esta línea se coloca en la parte superior del archivo HTML, antes de la etiqueta html>. A diferencia de versiones anteriores de HTML, en HTML5 el DOCTYPE no requiere la especificación del tipo de documento o la URL de la especificación. Simplemente se escribe.

Es importante tener en cuenta que la inclusión de DOCTYPE HTML5 en un documento HTML es crucial para que la página web se cargue correctamente. 

Si no se incluye la declaración, el navegador puede entrar en modo de compatibilidad, lo que significa que el contenido de la página se puede mostrar de manera incorrecta o inconsistente. 


Por lo tanto, es recomendable incluir siempre esta línea en todos los documentos HTML5.



Etiquetas HTML


El lenguaje HTML está formado por etiquetas que se escriben encerradas por los corchetes angulares menor que (<) y mayor que (>). Desde que el lenguaje HTML se creó en el año 1990, el número de etiquetas que posee ha ido aumentando y en la actualidad, en la versión HTML5 está formado por más de 100 etiquetas.

El elemento es el componente básico de una página web. Una página web está compuesta por un conjunto de elementos escritos en un orden concreto. Un elemento se compone de una etiqueta inicial (o de apertura) y una etiqueta final (o de cierre). La etiqueta final se define con una barra inclinada y el nombre de la etiqueta inicial.

Sin embargo, algunos elementos no necesitan una etiqueta de cierre porque no actúan como contenedores. Entre ellos se incluyen las imágenes:

HTML Copiar código 
 <img src="image.jpg">


En este ejemplo, la etiqueta se llama “p” y define un párrafo de texto de la página. La etiqueta inicial <p> puede llevar atributos, pero la final </p> nunca lleva. En HTML los atributos pueden llevar un valor. Por último, las etiquetas pueden tener contenido, que es todo lo que se escriba entre la etiqueta inicial y la etiqueta final. El contenido puede estar formado por otras etiquetas de HTML o puede ser simplemente texto como en este ejemplo.


El orden dentro de los elementos de HTML es muy importante:


Etiqueta inicial

Atributo

Valor (pueden llevarlo o no)

Contenido

Etiqueta final (nunca lleva atributos y se caracteriza por llevar una /).


Los elementos se pueden anidar, es decir, se pueden escribir unos dentro de otros, pero respetando algunas reglas que analizaremos más adelante.


Atributos


Cada etiqueta HTML tiene diferentes atributos que se pueden utilizar para proporcionar información adicional sobre el elemento. Estos incluyen class, id, style, lang, src, y href. Por ejemplo, la etiqueta "a" se utiliza para crear enlaces y tiene un atributo href que se utiliza para especificar la dirección URL del enlace. Otros ejemplos de atributos comunes incluyen class y id, que se utilizan para especificar estilos y seleccionar elementos específicos para el estilo o la manipulación del DOM mediante Javascript.

Aquí tenemos un ejemplo de un elemento HTML con el atributo class:

HTML Copiar código 
  <h1 class="titulo-principal">Mi Página Web</h1>

Los atributos deben ser escritos en la etiqueta de apertura, antes del paréntesis angular de cierre.

Los atributos están separados por un signo igual = de su valor. A la izquierda escribimos el nombre del atributo y a la derecha especificamos su valor. En este ejemplo, el valor es titulo-principal.

Recuerda que...

  • El valor del atributo debe estar rodeado por comillas.
  • Cada tipo de elemento tiene un conjunto específico de atributos que podemos asignar y cada atributo tiene un conjunto de valores posibles.

Etiquetas básicas


A continuación conoceremos las etiquetas básicas que debe tener todo documento HTML:

Etiqueta HTML

La etiqueta HTML es la etiqueta principal, sólo puede existir una y toda la página web debe escribirse entre la etiqueta inicial y final de HTML. La etiqueta html es la primera etiqueta que aparece en un documento, precisamente marca el inicio y fin de todo el documento.

HTML Copiar código 
<!DOCTYPE html>
<html>
   <head>
       ...
   </head>
   <body>
       ...
   </body>
</html>


 <head>


La etiqueta head define la primera parte de una página web, la cabecera. Normalmente, lo que se escribe en la cabecera no aparece representado en la página web cuando se visualiza en un navegador web, sino que se emplea para definir los metadatos, información sobre la página o instrucciones sobre cómo procesar la página web. En la etiqueta head describimos las cosas que son comunes al documento como el título, hojas de estilos y scripts. Puede incluir información opcional sobre la página Web, como puede ser el título (el navegador lo muestra en la barra de título), palabras clave de búsqueda opcionales y una hoja de estilo opcional.

HTML Copiar código 
<head>
   <title>Document title</title>
</head>


<body>


Dentro de esta etiqueta escribimos el contenido del documento como tal.

HTML Copiar código 
<html>
   <head>
       <title>Here goes the title of the document</title>
   </head>
   <body>
       Here goes the he content of the document......
   </body>
</html>


<head> y <body>


Sólo hay una manera correcta de combinar estos tres elementos. He aquí su colocación exacta, con el doctype al comienzo de la página:

HTML Copiar código 
<!DOCTYPE html>
<html>

   <head>
       ...
   </head>


   <body>
       ...
   </body>
</html>


Toda página web utiliza esta estructura básica. Los puntos suspensivos (...) muestran dónde insertaría la información adicional.


Etiquetas del encabezado


Estas etiquetas deben ir dentro de la etiqueta head. Describen metadatos y recursos comunes a todo el documento.

<title>


Dentro de la cabecera (head), se debe escribir la etiqueta title, que es obligatoria, sólo puede haber una y se emplea para definir el título de la página. Dentro de la etiqueta title debemos escribir el título del documento, este será visible en la pestaña del navegador:

HTML Copiar código 
<title>This is the page title</title>

Además, cuando la página aparece en una búsqueda web, el motor de búsqueda (por ejemplo Google) suele enseñar este título como primera línea en los resultados, seguido de un fragmento del contenido de la página.

<meta>


La etiqueta meta se usa para describir información del documento, como el autor, una breve descripción y algunas palabras clave. Esta información es importante para que los buscadores y las redes sociales puedan saber de qué se trata la página.

HTML Copiar código 
<meta charset="utf-8">


<link>


Con la etiqueta link podemos hacer referencia a un recurso externo, su uso más común es asociar una o más hojas de estilo a un documento HTML.

HTML Copiar código 
<link href="style.css" rel="stylesheet">


<script>


La etiqueta script se usa para hacer referencia a archivos externos de código o puede también incluir directamente código dentro de ella. De esta forma asociamos código JavaScript a los documentos HTML.


Etiquetas estructurales


Estas etiquetas no tienen una visualización en particular, pero nos son útiles para estructurar el documento.

Conozcamos cada una de ellas. 

<div>


El objetivo de la etiqueta div es agrupar elementos diversos dentro de un único bloque, esto es útil, por ejemplo, al momento de aplicar estilos CSS, pues podemos aplicar un estilo a todos los elementos que estén dentro del mismo bloque div.

<span>


Con la etiqueta span también es posible agrupar elementos, pero no dentro de un bloque, sino en forma lineal.

<header>


La etiqueta header nos permite definir el encabezado del documento, generalmente allí encontramos el logo y menú de un sitio web.

HTML Copiar código 
<header>Logo</header>


<footer>


Con la etiqueta footer definimos el pie del documento, por lo general tenemos aquí los enlaces a información corporativa, preguntas frecuentes y datos de contacto.

HTML Copiar código 
<footer>Some copyright info goes here</footer>


Etiquetas de interfaz de usuario


Estas etiquetas presentan controles visuales en el documento, las usamos para recibir información del usuario, por ejemplo en formularios.

select La etiqueta select presenta una lista de opciones desplegable. Podemos hacer clic sobre las flechas en la parte de visualización para que pruebes su funcionamiento.
button Como su nombre lo indica, esta etiqueta presenta un botón.
checkbox Esta etiqueta presenta un botón de tipo check. Este tipo de botones se suele usar en grupos para mostrar múltiples opciones.
radio button Esta etiqueta presenta un botón de tipo radio. Este tipo de botones se suele usar en grupos para elegir una única opción entre varias.
input Esta etiqueta se usa para presentar al usuario un campo de entrada de datos que pueden ser de tipo texto, numérico, fecha u otros. Podemos escribir dentro del campo para comprobar su funcionamiento.
textarea Esta etiqueta presenta un área que permite la entrada de una mayor cantidad de texto. Podemos escribir un párrafo dentro del campo para comprobar su funcionamiento.


Etiqueta imagen


Estas etiquetas nos permiten incluir elementos diferentes a texto. Algunas aparecen a partir de la versión 5 del estándar HTML y hacen posible la reproducción de audio, video y elementos 2D y 3D sin utilizar plugins adicionales.

<img>


Para poner una imagen simple en una página web, utilizamos el elemento <img>. Se trata de un elemento vacío (lo que significa que no contiene texto o etiqueta de cierre) que requiere al menos un atributo para ser utilizado: src (a veces denominado por su nombre completo, source). El atributo src contiene una ruta que apunta a la imagen que queremos poner en la página, que puede ser una URL relativa o absoluta, de la misma forma que el elemento <a> contiene los valores del atributo href.


Uno de los atributos más importantes de la etiqueta <img> es el atributo "src" que se utiliza para especificar la ruta de la imagen. Esta ruta puede ser una dirección URL que apunte a la imagen en la web, o una ruta local en la computadora donde se encuentra el archivo de imagen.

Otros atributos de la etiqueta <img> incluyen "alt" para proporcionar un texto alternativo para la imagen en caso de que no se pueda cargar, y "width" y "height" para definir el tamaño de la imagen en píxeles.


Por ejemplo, si tenemos una imagen que se llama dinosaur.jpg, y está en el mismo directorio que nuestra página HTML, debemos incrustar la imagen de la siguiente manera:

HTML Copiar código 
<img src="dinosaur.jpg">



En la etiqueta de apertura especificamos la ubicación de la imagen en internet (o en nuestro dispositivo) con el atributo src.

Si la imagen estaba en el subdirectorio “images”, que estaba en el mismo directorio que la página HTML (lo que Google recomienda con fines de indexación y posicionamiento en buscadores SEO), entonces debemos incrustar la imagen así:

HTML Copiar código 
 <img src="images/dinosaur.jpg">

Sin embargo, esto hace que el navegador trabaje más buscando la dirección IP desde el servidor DNS cada vez. Casi siempre conviene mantener las imágenes para nuestro sitio web en el mismo servidor del HTML.

Etiquetas multimedia (2)


Texto alternativo

También debemos incluir el atributo alt en todas las imágenes de nuestras páginas web para que se dé un texto alternativo en caso de que la imagen no se descargue correctamente o en caso de que el usuario use un lector de pantalla.

HTML Copiar código 
<img src="dinosaur.jpg" alt="Un dinosaurio con traje">


El atributo "alt" es importante en las imágenes en HTML por varias razones. En primer lugar, "alt" es la alternativa en texto de una imagen para aquellos que no pueden cargar o ver imágenes, lo que hace que el contenido de la imagen sea accesible para todos los usuarios, incluyendo a aquellos con discapacidades visuales o con conexiones lentas a internet.

Además, el uso de etiquetas "alt" claras y descriptivas puede mejorar la experiencia del usuario al permitir que sepan qué imagen se está mostrando, incluso si no se carga correctamente o si no pueden verla.

Por último, el uso de etiquetas "alt" también puede ayudar en la optimización de motores de búsqueda, ya que les brinda información sobre el contenido de la imagen y su relevancia para la página.

De esta forma, si la imagen no se puede encontrar o mostrar al usuario (por ejemplo, si cambiamos el atributo src e intentamos llamar a una imagen .png en vez de .jpg), se mostrará el texto alternativo en su lugar, para que el usuario pueda comprender de qué se trata.


Anchura y altura

Podemos usar los atributos ancho (width) y alto (height) para especificar la anchura y altura de la imagen.

HTML Copiar código 
<img src="dinosaur.png" alt="Un dinosaurio con traje" width="400" height="341">


Esto no proporciona una gran diferencia en la pantalla en circunstancias normales. Pero si la imagen no se muestra, por ejemplo, porque el usuario acaba de acceder a la página y esta aún no se ha cargado, observamos que el navegador reserva un espacio para la imagen.

Sin embargo, no deberíamos alterar el tamaño de las imágenes utilizando atributos HTML. Las imágenes podrían verse granulosas y borrosas si establecemos un tamaño demasiado grande; o bien demasiado pequeñas, y se desperdiciaría ancho de banda descargando una imagen que no se ajusta a las necesidades del usuario. La imagen también podría quedar distorsionada, si no mantenemos la proporción de aspecto correcta. Lo ideal es utilizar un editor de imágenes para dar a la imagen el tamaño adecuado antes de colocarla en una página web.

Título de imágenes

Al igual que con los enlaces, también podemos añadir atributos title a las imágenes para proporcionar más información de ayuda si es necesario. En nuestro ejemplo, podríamos hacer esto:

HTML Copiar código 
<img src="dinosaur.jpg" alt="Un dinosaurio con traje" width="400" height="341" title="Dibujo artístico de un dinosaurio con traje">


Esto nos da una etiqueta de ayuda (tooltip):



Formatos de imagen para web


GIF (Graphics Interchange Format)

PNG 8 (Portable Network Graphics): usa colores de 256, se utiliza para logotipos e iconos para la página.

PNG 24 (Portable Network Graphics): utilización de colores ilimitados, alta calidad, si intentamos comprimir no ayudará demasiado por la gran cantidad de colores.

JPG / JPEG (Photographic Experts Group): perdemos calidad a la hora de comprimirlas, pero llegan a ser óptimas para la carga en la página web.

SVG - Vector (Scalable Vector Graphics): Formato muy ligero sin pérdida, con svg no perdemos calidad, ya que está compuesta por vectores.

WebP: Es un formato gráfico en forma de contenedor que sustenta tanto compresión con pérdida como sin ella. ​​Fue desarrollado por Google.


audio

Con la etiqueta audio podemos incluir archivos de audio en el documento y reproducirlos directamente a través del browser o navegador.

video

Con la etiqueta video podemos incluir videos en el documento, con la posibilidad de mostrar controles de reproducción, como pausa, avance y ver en pantalla completa.

canvas

La etiqueta canvas nos permite incluir gráficos personalizados en 2D o 3D, es un área del documento en donde podemos dibujar a través de código.


Etiquetas de Formato


Estas etiquetas representan elementos de texto con diferentes características.


Títulos

Las etiquetas h1, h2, h3, h4, h5 y h6 se emplean para definir los encabezados o títulos en la página. Existen seis niveles de importancia, siendo h1 el más importante y h6 el nivel menos importante.

HTML Copiar código 
       <h1>Heading level 1</h1>
       <h2>Heading level 2</h2>
       <h3>Heading level 3</h3>
       <h4>Heading level 4</h4>
       <h5>Heading level 5</h5>
       <h6>Heading level 6</h6>


El elemento h1 se utiliza para el título principal de la página, mientras que los encabezados de nivel inferior (h2, h3, h4, etc) se utilizan para los títulos de secciones subordinadas. Los encabezados deben ir ordenados jerárquicamente, es decir, el h2 debe ir después del h1, el h3 después del h2, y así sucesivamente.


Párrafo <p>

Los párrafos (p) se utilizan para dividir el contenido de la página en secciones separadas. Cada párrafo puede contener texto, imágenes u otros elementos HTML.

HTML Copiar código 
 

<p>Soy un párrafo</p>


Los párrafos se enmarcan entre la etiqueta p, agrupando un texto que tiene sentido completo.

La estructura jerárquica es una parte fundamental del marcado de texto en HTML. Esto permite que el navegador web muestre el contenido de una página de manera ordenada y jerarquizada. Para lograr esto, es necesario utilizar una serie de elementos en HTML que permiten organizar el contenido de la página.

Es importante tener en cuenta que el orden en que se colocan los elementos es fundamental para una correcta estructuración del contenido en la página. En general, se recomienda utilizar una estructura jerárquica en la que el título principal (h1) esté seguido por los encabezados de secciones (h2, h3, h4, etc.) y los párrafos (p) se utilicen para dividir el contenido dentro de cada sección.

Por ejemplo, en esta historia, <h1> representa el título de la historia, <h2> representará el título de cada capítulo y <h3> las diferentes secciones del capítulo, y así sucesivamente.


¿Por qué necesitamos estructura?

Los usuarios que miran una página web tienden a escanearla rápidamente para encontrar el contenido relevante que hay en ella y a menudo empiezan por leer los encabezados para comenzar (solemos pasar muy poco tiempo en una página web). Si el usuario no puede ver nada útil en unos segundos, es probable que se sientan frustrados y se vayan a otro sitio.


Los motores de búsqueda que indexan la página consideran el contenido de los títulos como palabras clave importantes e influyen en la puntuación de búsqueda de la página. Sin encabezados, la página tendrá un rendimiento bajo en términos de optimización de motores de búsqueda SEO.


Las personas con discapacidad visual severa no suelen leer páginas web: en lugar de ello, las escuchan. Lo hacen con un software llamado Lector de pantalla. Este software proporciona acceso rápido a un contenido textual dado. Entre las diversas técnicas que emplean, leen los encabezados para proporcionar un esquema del documento que permite a los usuarios encontrar rápidamente la información que quieren. Si no hay encabezados disponibles, se ven obligados a escuchar el documento entero.

Para aplicar estilos al contenido con CSS, o para hacer cosas interesantes con JavaScript, como estudiaremos en las próximas secciones, necesitamos tener elementos que delimiten el contenido relevante para que CSS/JavaScript se puedan focalizar en este efectivamente.

Algunas buenas prácticas:

Preferentemente usar sólo un <h1> por página; este es el nivel de título superior, y todos los demás se sitúan por debajo de él en la jerarquía.

Usar los títulos en el orden correcto en la jerarquía. No usar los <h3> para representar subtítulos, seguidos de los <h2> para representar los subtítulos de los subtítulos; eso no tiene sentido y puede provocar resultados extraños.

De los seis niveles de títulos disponibles, debemos procurar no usar más de tres por página, a menos que creamos que es realmente necesario. Los documentos con muchos niveles (es decir, una jerarquía de títulos muy profunda) son de difícil manejo y navegación. En esos casos se recomienda, si es posible, separar el contenido en varias páginas.




Énfasis e importancia


En el lenguaje humano, a menudo enfatizamos ciertas palabras para alterar el significado de una frase, y a menudo queremos destacar ciertas palabras como importantes o diferentes en algún sentido. HTML nos dota de diversos elementos semánticos que nos permiten destacar contenido textual con tales efectos, y en esta sección analizaremos los más comunes.

Énfasis <em>

Cuando queremos dar énfasis al lenguaje hablado, acentuamos ciertas palabras y así alteramos sutilmente el significado de lo que decimos. De manera similar, en el lenguaje escrito ponemos palabras en cursiva para destacarlas.

En HTML usamos el elemento <em> («emphasis») para marcar estos casos. El documento logra entonces transmitir una lectura más interesante y además así lo reconocen los lectores de pantalla, que lo expresan con un diferente tono de voz. El navegador, de manera predeterminada, aplica el estilo de letra itálica, pero no debemos utilizar esta etiqueta solamente para establecer el estilo de letra itálica. Para usar ese estilo, debemos utilizar CSS como estudiaremos en otras semanas.

Me alegro de que no llegues tarde.

HTML Copiar código 
 
<p>Me <em>alegro</em> de que no llegues <em>tarde</em>.</p>


Importancia fuerte <strong>

Para enfatizar palabras importantes al hablar solemos acentuarlas, y al escribir lo hacemos en estilo negrita. Por ejemplo:

Cuento contigo. ¡No llegues tarde!

En HTML usamos el elemento <strong> (importancia fuerte) para marcar tales expresiones. El documento resulta entonces más útil, y de nuevo los lectores de pantalla reconocen estos elementos y el tono de voz cambia a uno más fuerte. El estilo negrita es el que aplican los navegadores por omisión, pero no debemos usar esta etiqueta solamente para aplicar este estilo. Para hacer eso usamos CSS o un elemento <b>.

Cuento contigo. ¡No llegues tarde!

HTML Copiar código 
 
<p>Cuento contigo. <strong>¡No llegues tarde!</strong></p>

La etiqueta strong se emplea para destacar un fragmento de texto importante mientras que la etiqueta em se emplea para definir un texto enfatizado. 


Podemos anidar elementos de énfasis dentro de elementos de importancia y viceversa:
Este líquido es altamente tóxico - si lo bebes, podrías morir.

HTML Copiar código 
       <p>
           Este líquido es <strong>altamente tóxico</strong> — si lo bebes,
           <strong>podrías <em>morir</em></strong>.
       </p>

Cursiva, negrita, subrayado


Los elementos que hemos comentado hasta ahora tienen asociada una semántica clara. La situación con <b> (negrita o «bold»), <i> (cursiva o «italic») y <u> (subrayado o «underline») es algo más complicada. Surgieron para que las personas pudieran escribir textos en negrita, cursiva o subrayado en un tiempo en el que pocos navegadores o ninguno admitían el CSS. Elementos como estos, que solo afectan a la presentación y no a la semántica, se conocen como elementos de presentación y no se deberían usar porque, como hemos visto, la semántica es muy importante para la accesibilidad y el SEO, entre otros aspectos.

HTML5 redefinió los elementos <b>, <i> y <u> con roles semánticos nuevos un tanto confusos. Esta es la regla de oro: el uso de <b>, <i> o <u> resulta adecuado cuando transmiten el significado que suele transmitir el uso tradicional de las negritas, itálicas o el subrayado, si no hay ningún otro elemento que resulte más adecuado. Sin embargo, siempre resulta crítico mantener una actitud orientada a la accesibilidad. El concepto de itálica no es demasiado útil para las personas que usan lectores de pantalla o para las personas que utilizan un sistema de escritura distinto del alfabeto latino.

<i> se usa para transmitir el significado que tradicionalmente transmite la itálica: extranjerismos, clasificaciones taxonómicas, conceptos técnicos, un pensamiento...
<b> se usa para transmitir el significado que tradicionalmente transmite la negrita: palabras clave, nombres de productos, frases principales...
<u> se usa para transmitir el significado que tradicionalmente conlleva el subrayado: nombres propios, errores ortográficos...



Listas


El lenguaje HTML posee tres tipos de listas, las listas no ordenadas, las listas ordenadas y las listas de descripción . 


 Cada una de estas listas se crea con una etiqueta específica de HTML, <ul>, para la lista no ordenada, <ol>, para la lista ordenada y <dl>, para la lista de descripción.



Listas numeradas y listas no numeradas

La etiqueta con cierre para listas no numeradas es <ul>, cada elemento de la lista utilizará un símbolo llamado bullet. Para listas numeradas se usa la etiqueta <ol>, aquí, a cada elemento de la lista le precederá un número. Al iniciar la lista se utiliza la etiqueta <ul> o bien la etiqueta <ol>, se coloca cada elemento de la lista delimitado por la etiqueta <li> y su cierre </li>, y de inmediato se cierran estas listas con </ul> o </ol> según sea el caso.

TAG <ul>

La etiqueta <ul>, permite definir una lista desordenada (“Unordered List”). Esta etiqueta es a su vez contenedora de otras etiquetas, siendo estas últimas las que componen cada uno de los elementos de los que está compuesta la lista.

Para especificar un elemento en la lista se ha de utilizar la etiqueta <li> , en la que se ha de especificar el texto que compone el elemento de la lista.

Una lista puede contener como elemento de lista, otra lista, es decir se pueden crear listas anidadas.

La sintaxis de la etiqueta es la siguiente:

HTML Copiar código 
        <ul>
           <li>Elemento de lista</li>
           <li>Elemento de lista</li>
           <li> … </li>
       </ul>


TAG <ol>

La etiqueta <ol>, permite definir listas o viñetas ordenadas (“Ordered List”), bien con numeración o alfabéticamente. Esta etiqueta es a su vez contenedora de otras etiquetas, siendo estas últimas las que componen cada uno de los elementos de los que está compuesta la lista.

Para especificar un elemento en la lista se ha de utilizar la etiqueta <li> , en la que se ha de especificar el texto que compone el elemento de la lista.

Una lista puede contener como elemento de lista, otra lista, es decir se pueden crear listas anidadas.

La sintaxis de la etiqueta es la siguiente:

HTML Copiar código 
 
<!DOCTYPE HTML>
<html>
   <head>
       <title>Ejercicios prácticos HTML5</title>
       <meta charset="utf-8">
   </head>

   <body>
       <p>Lista con elementos desordenados</p>

       <ul>
           <li>Uno</li>
           <li>Dos</li>
           <li>Tres</li>
       </ul>

       <p>Lista con sublistas anidadas</p>

       <ul>
           <li>Primero</li>
           <li>Segundo
               <ul>
                   <li>Segundo Uno</li>
                   <li>Segundo Dos</li>
               </ul>
           </li>
           <li>Tercero</li>
       </ul>
   </body>
</html>


Los atributos que pueda utilizar la etiqueta, aparte de los globales, son los siguientes:

<type>


El atributo type permite especificar el tipo de orden que se debe aplicar a la lista, estableciendo de esta forma que la lista se represente mediante diferentes tipos de numeración.

Los diferentes valores que puede recibir este atributo son los siguientes:


1: Es el valor por defecto en caso de no especificarse el atributo. Realiza la numeración de la lista utilizando valores numéricos. (1, 2, 3…)

a: Establece que la numeración de la lista se realice utilizando caracteres alfabéticamente en letra minúscula. (a, b, c …)

A: Establece que la numeración de la lista se realice utilizando caracteres alfabéticamente en letra mayúscula. (A, B, C …)

i: Establece que la numeración de la lista se realice utilizando numeración en formato de números romanos en letra minúscula. (i, ii, iii, iv …)

I: Establece que la numeración de la lista se realice utilizando numeración en formato de números romanos en letra mayúscula. (I, II, III, IV …)



<start>


El atributo start permite indicar el valor de inicio, por el cual ha de comenzar la lista ordenada, siendo este un valor numérico. En caso de ser la lista ordenada de forma alfabética, el valor de inicio que se exprese se corresponderá con el orden del abecedario ASCII correspondiente.


La sintaxis de la etiqueta con el atributo es la siguiente:

HTML Copiar código 
       <ol start="valor">
           <li>Elemento de lista</li>
           <li>Elemento de lista</li>
           <li> … </li>
       </ol>
Ejemplo:
       <ol start="3">
           <li>Elemento de lista</li>
           <li>Elemento de lista</li>
           <li> … </li>
       </ol>


<reversed>

El atributo reversed permite indicar que la numeración u orden que se haya establecido se represente de forma inversa. Si tuviéramos una lista con valor de inicio (start) 1, 2, 3, sucesivamente, su representación al encontrarse este atributo sería 3, 2, 1.


Es un atributo con valor booleano; en caso de aparecer el atributo en la etiqueta, se aplica el valor. Ejemplo:

HTML Copiar código 

       <ol type="I" start="10" reversed>
           <li>Elemento</li>
           <li>Elemento</li>
           <li>Elemento</li>
       </ol>


TAG <dl>

La etiqueta <dl> permite definir lo que se denominan listas de definición, las cuales a su vez están compuestas por un término y una definición.

Para realizar la lista se utilizan dos etiquetas más. La primera es <dt>, que permite especificar el término en la lista. Y la otra etiqueta es <dd>, que permite especificar la definición.


La sintaxis de la lista con las etiquetas correspondientes es la siguiente:

HTML Copiar código 
       <dl>
           <dt>Termino a definir</dt>
           <dd>Definición del termino</dd>
           <dt>Termino a definir</dt>
           <dd>Definición del termino</dd>
           …
       </dl>

Etiquetas de hipertexto


Estas etiquetas son las que permiten navegar de un documento a otro en el modelo de lectura hipertextual.

Enlace <a>

Los enlaces son el elemento principal que confiere a la web su carácter de hipertexto. El hipertexto se puede definir como un documento digital que se puede leer de forma no secuencial y que está compuesto de nodos o secciones de enlaces o hipervínculos y de anclajes. La web está formada por millones de nodos, documentos o páginas web, interconectados por millones de enlaces.

La etiqueta <a> en HTML es utilizada para crear hipervínculos o enlaces que permiten a los usuarios navegar a través de diferentes páginas web. Al hacer clic en un enlace, el navegador carga la página web que está vinculada a la dirección URL especificada en el atributo "href" de la etiqueta <a>.
Por ejemplo, este elemento nos llevaría a la página web de Google en español:

HTML Copiar código 
  
<a href="https://www.google.com/?hl=es">google</a>

  • El atributo href nos permite especificar el destino del enlace. Su valor es el destino del enlace.
  • El texto que escribimos entre las etiquetas <a> es el texto que verá el usuario. En este caso, el texto es "google".
También podemos llevar a los usuarios a otra página de nuestro sitio web:

HTML Copiar código 
<a href="about.html">Sobre nosotros</a>

Y si asignamos el atributo id a un elemento HTML, podemos crear un enlace para llevar al usuario hasta ese elemento:

HTML Copiar código 
           <a href="#imagen">Ir a la imagen</a>
           ...
           <img src="image.jpg" id="imagen">

En este ejemplo, el enlace llevaría al usuario hasta el elemento con id “imagen”.

Enlaces internos y externos

Los enlaces pueden llevar rápidamente a una página a otra dentro del mismo sitio Web o transportarnos a una ubicación diferente en un servidor lejano. En cada caso se utilizan distintos tipos:


Los enlaces internos apuntan a otras páginas del sitio Web. También pueden apuntar a otros tipos de recursos multimedia de su sitio.

Los enlaces externos apuntan a páginas (o recursos) de otros sitios Web.


Existen tres tipos de rutas o paths que se pueden utilizar en los enlaces de HTML:


Absolute Path: El enlace contiene la URL completa de la página web que se va a vincular, incluyendo el protocolo (HTTP o HTTPS), el nombre de dominio y la ruta de la página web. Por ejemplo, https://www.ejemplo.com/pagina.html.

Relative Path: El enlace contiene solo la ruta de la página web que se va a vincular, relativa a la ubicación actual de la página web. Por ejemplo, si la página actual es "https://www.ejemplo.com/index.html" y se desea vincular a una página llamada "pagina.html" en el mismo directorio, se puede utilizar el enlace "../pagina.html". El uso de rutas relativas es útil para vincular páginas dentro del mismo sitio web.

Root-Relative Path: El enlace contiene la ruta de la página web que se va a vincular, relativa a la raíz del sitio web. Por ejemplo, si la página actual es "https://www.ejemplo.com/pagina1/index.html" y se desea vincular a una página llamada "pagina2.html" ubicada en la raíz del sitio web, se puede utilizar el enlace "/pagina2.html". El uso de rutas root-relative es útil para vincular páginas en diferentes directorios del mismo sitio web.


La etiqueta <a> representa los enlaces a otros documentos html en la web, se componen del texto visible y la URL de destino.

El tag <a> es un elemento de línea (inline): se introduce dentro de cualquier otro elemento bloque. Eso quiere decir que es completamente aceptable crear un enlace de varias palabras en un párrafo normal:


Cuando se pasa el mouse por encima de un enlace, el cursor se convierte en una mano. Nota: el elemento <a> es distinto al elemento <link>. El elemento <link> se usa para especificar la relación entre el archivo HTML y una fuente externa como un archivo CSS.

Abrir hipervínculo con una nueva ventana

¿Cómo puedo crear un enlace que abra una página en una nueva ventana del navegador? Cuando los visitantes hacen clic en enlaces externos, tal vez no deseemos que abandonen el sitio tan fácilmente. Los/las desarrolladores/as Web utilizan un truco común que abre páginas externas en ventanas de navegador separadas (o en una nueva pestaña, dependiendo de la configuración). De esta forma, nuestro sitio permanecerá abierto en la ventana original, asegurando que el visitante no se olvidará de él. Para que esto funcione debe establecer otro atributo en el elemento <a>: target (destino).

El atributo target nombra el marco en el que un navegador debería mostrar la página de destino. El valor de _blank indica que el enlace debe abrirse en una ventana nueva del navegador.

Colores en los links

Es posible definir y personalizar los colores en los que aparecen tanto los enlaces, como los enlaces ya visitados, lo veremos en próximas clases cuando aprendamos CSS. Es recomendable mantener siempre el color estándar para los enlaces (azul para enlace y morado para los enlaces ya visitados).


Tablas HTML


Una tarea muy común en HTML es la estructuración de datos, y para esto hay múltiples elementos y atributos.

¿Qué es una tabla? Una tabla es un conjunto estructurado de datos distribuidos en filas y columnas (datos tabulados). Una tabla permite buscar con rapidez y facilidad valores entre diferentes tipos de datos que indiquen algún tipo de conexión. Por ejemplo, una persona y su edad, o un día de la semana o el horario de una piscina municipal.


El aspecto básico de una tabla es que es un elemento rígido. Es fácil interpretar la información haciendo asociaciones visuales entre los encabezados de las filas y las columnas. Cuando se hace correctamente, incluso las personas ciegas pueden interpretar los datos de una tabla HTML. Una tabla HTML bien hecha debe mejorar la experiencia de los usuarios videntes e invidentes por igual.


¿Cuándo no debemos usar tablas HTML? 

Las tablas HTML están pensadas para utilizarse con datos tabulados. Por desgracia, mucha gente utiliza las tablas HTML para hacer compaginaciones de páginas web. Por ejemplo, una fila para contener la cabecera, una fila para contener las columnas de contenido, una fila para contener el pie de página, etc. Se solía hacer este uso de las tablas porque la compatibilidad CSS entre navegadores solía ser terrible. Los diseños de tablas son mucho menos comunes hoy en día, pero aún se pueden ver en algunos rincones de la web.


En documentos HTML, una tabla puede ser considerada, resumidamente, como un grupo de filas donde cada una contiene a un grupo de celdas. Esto es conceptualmente distinto a un grupo de columnas que contiene a un grupo de filas, y esta diferencia tendrá un impacto en la composición y comportamiento de la tabla.


Como muchas otras estructuras de HTML, las tablas son construidas utilizando elementos. En particular, una tabla básica puede ser declarada usando tres elementos, a saber, table (el contenedor principal), tr (representando a las filas contenedoras de las celdas) y td (representando a las celdas).

Unificación de celdas

La unificación de celdas es una técnica que permite combinar dos o más celdas adyacentes en una sola celda, lo que puede ser útil para presentar información de manera más clara y concisa. Para unificar celdas en una tabla HTML, se pueden utilizar los atributos colspan y rowspan.

El atributo colspan se utiliza para unificar celdas horizontalmente, es decir, para combinar dos o más celdas en una sola fila. Por ejemplo, si se tiene una tabla con tres filas y dos columnas, y se quiere unificar las dos celdas de la primera fila en una sola celda, se puede usar el siguiente código HTML:

Por último, el elemento tbody se utiliza para agrupar el contenido principal de la tabla.

Al utilizar estos elementos para agrupar filas, se puede mejorar la organización y legibilidad de la tabla, lo que puede resultar muy útil para presentar datos de una manera clara y concisa.

Reglas básicas 


Las reglas básicas que se deben aplicar en la creación de una página web son:


Etiquetas siempre cerradas.

Documentos bien formados.

Etiquetas en minúsculas.

Valores de los atributos siempre entre comillas.


También es aconsejable utilizar los nombres de los ficheros del alfabeto inglés, los números, el guión y el guión bajo. Se pueden utilizar otros caracteres pero pueden dar problemas. Por último, es importante recordar que las extensiones de las páginas web son .htm o .html.

Idioma

Podemos especificar el idioma principal de la página web o de cualquier elemento en la estructura con el atributo lang y el código correspondiente del idioma:

HTML Copiar código 
 <html lang="es">

Comentarios en HTML


HTML también nos permite escribir comentarios. Los comentarios son elementos que se utilizan para añadir notas o, justamente, comentarios, que no se muestran en la página web. Esto puede ser útil para proporcionar información sobre el código a otros/as desarrolladores/as o para hacer notas uno mismo, las cuales se pueden leer al ver el archivo. Son muy útiles para explicar y entender la estructura de la página o saber qué tareas están pendientes.

Los comentarios en HTML se escriben entre los caracteres "<!--" y "-->" y pueden contener cualquier tipo de texto o código HTML dentro de ellos. 

Todo el texto que se escribe dentro de un comentario es ignorado por el navegador y no se muestra en la página web, lo que los hace útiles para explicar el propósito de una sección de código o para dejar notas para los desarrolladores futuros.


Para insertar un comentario en HTML, se utiliza la sintaxis <!-- Comentario --> . Los comentarios pueden ir en cualquier lugar del código HTML, excepto dentro de etiquetas que requieren contenido, como la etiqueta "title". También se pueden usar para comentar líneas individuales de código, pero en ese caso, se debe colocar el comentario en una línea separada del código.


Formulario HTML


Según la W3C, un formulario HTML es una sección de un documento que contiene contenido normal, código, elementos especiales llamados controles y rótulos para esos controles. Los usuarios, normalmente, rellenan un formulario modificando sus controles antes de enviar el formulario a un agente para que lo procese, a un servidor web, a un servidor de correo, etcétera.

¿Para qué se utilizan?

Se utilizan para permitir que el usuario envíe información al servidor, ya sea en formularios de registro o acceso a sitios web, formularios de contacto, o incluso al escribir un comentario en Facebook.


Un formulario (form) es básicamente un contenedor para controles. Cada control en un formulario está pensado para recolectar información ingresada por los usuarios, en formas que pueden ir desde líneas de texto hasta la subida de archivos, pasando por opciones, fechas, contraseñas y mucho más. Una vez que los usuarios han rellenado el formulario con los datos, pueden enviarlo de regreso al servidor para que el agente procesador administre la información recolectada.


A continuación, veremos las etiquetas básicas para formularios, y las analizaremos una a una:

<form>


La etiqueta <form> es fundamental para que el resto de elementos puedan ser enviados. Todos los campos que se quieren enviar deben estar entre las etiquetas form y </form>. Esta etiqueta no tiene representación gráfica.

Pero este modelo no dice nada acerca de cómo y dónde será procesado el formulario. Tal información puede ser especificada con atributos tales como:

    action: Indica la ubicación del agente procesador; espera una cadena de texto que especifique la URL o ruta a la acción de destino que procese los datos de la petición. Se puede utilizar una ruta absoluta o relativa. Action es el único atributo requerido para la etiqueta form.

    method: Determina el método utilizado para empaquetar el formulario antes de ser enviado al agente procesador. Este atributo acepta dos valores, get y post.

    get: Utilizado normalmente al navegar, los datos de envío son visibles en la URL. Se utiliza en casos donde el resultado del envío es siempre el mismo y se quiere que el resultado sea guardado, por ejemplo, en un formulario de búsqueda.

    post: Más seguro que get, especialmente para información sensible como contraseñas, ya que los datos no son visibles en la URL. No tiene límite en el envío y permite la subida de archivos al servidor. Se utiliza, por ejemplo, en formularios de registro.

    Si no se utiliza method, el formulario se enviará por defecto mediante el método GET.

    enctype: Especifica el tipo de codificación por defecto.

    application/x-www-form-urlencoded: Convierte todos los espacios en signos de suma y caracteres especiales en valores ASCII hexadecimal.

    multipart/form-data: Se utiliza cuando se van a subir archivos mediante el formulario.

    text/plain: Convierte solo los espacios en signos de suma, los caracteres especiales no son convertidos.

El siguiente ejemplo muestra cómo se pueden utilizar estos atributos en la etiqueta form:

HTML Copiar código 
<form action="/procesar-formulario" method="post" enctype="multipart/form-data"> </form>

Controles de Texto


Los controles de texto proveen los medios necesarios para recolectar información textual, como nombres, direcciones, frases, mensajes, contraseñas, etc. Todos los controles de texto básicos comparten algunos comportamientos comunes:


Se pueden marcar como readonly (el usuario no puede modificar el valor de entrada, pero este se envía con el resto de los datos del formulario) o disabled (el valor de entrada no se puede modificar y nunca se envía con el resto de los datos del formulario).

Se pueden marcar como readonly (el usuario no puede modificar el valor de entrada, pero este se envía con el resto de los datos del formulario) o disabled (el valor de entrada no se puede modificar y nunca se envía con el resto de los datos del formulario).

Pueden tener un placeholder; se trata de un texto que aparece dentro de la caja de entrada de texto y que se usa para describir brevemente el propósito de la caja de texto.

Pueden presentar una limitación de tamaño (el tamaño físico de la caja de texto) y de la extensión máxima (el número máximo de caracteres que se pueden poner en la caja de texto).
Admiten corrección ortográfica (utilizando el atributo spellcheck), si el navegador la admite.

Input


La etiqueta input es de las más utilizadas en formularios. Con ella podemos definir gran parte de los controles típicos de entradas de tipo texto, casillas de verificación y de opción conocidas como radio, archivos, contraseñas, campos ocultos y botones varios. La etiqueta input no tiene etiqueta de cierre.

 <input type="text" id="comment" name="comment" value="I'm a text field">


Input type

El elemento <input> tiene nuevos valores para el atributo type:


Valores Descripción Ejemplo
text Este tipo de input toma un valor de "texto", por lo que crea una sola línea de entrada de texto.  <input type="text" placeholder="Ingresar nombre" />
password El valor de la contraseña no añade restricciones especiales al texto que se introduce, pero oculta el valor que se introduce en el campo (por ejemplo, con puntos o asteriscos) para impedir que otros puedan leerlo. <input type="password" id="pwd" name="pwd">
checkbox

Las casillas de verificación se crean estableciendo el atributo type del elemento <input> en el valor checkbox. 

Al incluir el atributo checked, la casilla de verificación se marca automáticamente cuando se carga la página. Al hacer clic en la casilla de verificación o en su etiqueta asociada, la casilla de verificación se activa o desactiva.

 <input type="checkbox" id="carrots" name="carrots" value="carrots" checked>
radio

Un botón de opción se crea estableciendo el atributo type del elemento <input> en el valor radio. 

Es posible asociar diversos botones de opción. Si comparten el mismo valor de atributo name, se considera que están en el mismo grupo de botones. Sólo un botón dentro de un grupo puede estar activado en cada momento, todos los demás se deseleccionan automáticamente. Al enviar el formulario, sólo se envía el valor del botón de opción seleccionado. Si ninguno de ellos está seleccionado no se envía ningún valor con el formulario.

<input type="radio" id="soup" name="meal" checked>

Input type (continuación)

Continuamos con los tipos de entrada para el elemento input:

Valores Descripción Ejemplo
file Crea un botón en un formulario HTML que permite al usuario seleccionar uno o varios archivos de su dispositivo local para ser enviados al servidor web junto con el formulario. 

<input type="file">
search Representa una caja de búsqueda. Los saltos de línea son quitados del valor ingresado pero no se modifica ninguna otra sintaxis.  <input type="search" placeholder="Buscar">
tel Representa un control para editar un número de teléfono. Los números telefónicos varían ampliamente en el mundo. Se pueden utilizar atributos como pattern y maxlength para restringir los valores ingresados en la caja.

<input type="tel" placeholder="Número de teléfono" pattern="[0-9]{10}" maxlength="10">
url Representa un control para editar una URL. Se quitan los saltos de línea y espacios en blanco antes y después del valor ingresado. 

<input type="url" placeholder="URL">
email Representa una dirección de correo electrónico. Cualquier entrada con el tipo de correo electrónico define un campo para ingresar una dirección de correo electrónico. <input type="email" placeholder="Ingresar correo electrónico">
number Permite al usuario insertar solo números. <input type="number" placeholder="Ingresar un número">
reset Crea un botón en un formulario HTML que permite al usuario borrar los datos ingresados en todos los campos del formulario y restaurarlos a sus valores predeterminados. <input type="reset" value="Este es un botón de reinicio">
submit

Crea un botón en un formulario HTML que permite al usuario enviar los datos del formulario a un servidor web para su procesamiento. 

El uso de <input type="submit"> es común en formularios de registro, encuestas, compras en línea y cualquier otra situación en la que se requiera la entrada de información del usuario. Es importante destacar que, por defecto, el botón "submit" enviará el formulario al servidor y recargará la página, a menos que se especifique lo contrario utilizando JavaScript.

 <input type="submit" value="Enviar formulario">
button

Crea un botón en un formulario HTML que puede ser utilizado para activar una acción o una función JavaScript específica cuando se hace clic en él. A diferencia de otros elementos de entrada, como <input type="submit">, los botones creados con <input type="button"> no están diseñados para enviar el formulario. En su lugar, son útiles para agregar funcionalidad adicional a una página web, como cambiar el texto de una página, mostrar u ocultar elementos, cargar datos adicionales, o cualquier otra acción que se pueda implementar con JavaScript.

Es importante destacar que los botones creados con <input type="button"> no realizan ninguna acción por sí solos, sino que deben ser programados para realizar una tarea específica mediante el uso de JavaScript o alguna otra tecnología de programación en el lado del cliente.

 <input type="button">
color Este es un tipo de entrada elegante introducido por HTML5. Con él, el usuario puede enviar su color favorito, por ejemplo. El negro (#000000) es el valor predeterminado, pero se puede anular configurando el valor en el color deseado. <input type="color" value="#000000">
hidden Se usa para crear un control de formulario que es invisible para el usuario, pero que aún así se envía al servidor junto con el resto de los datos del formulario una vez se transmiten. Por ejemplo, es posible que desees enviar una marca de tiempo al servidor que indique cuándo se realizó un pedido. Al estar oculto, el usuario no puede ver ni editar su valor intencionadamente, el control nunca recibirá el foco y un lector de pantalla tampoco lo detectará. <input type="hidden" name="timestamp" value="2023-11-17T12:00:00">




Label


El elemento HTML <label> es una etiqueta utilizada en el lenguaje de marcado HTML para proporcionar una descripción de un elemento de control en un formulario web. Su función es asociar un texto descriptivo con un elemento de entrada en un formulario. Puede estar asociado con un control de formulario mediante el uso del atributo "for", que apunta al identificador del control correspondiente, o ubicando el control dentro del propio elemento <label>.


Al hacer clic en el texto del elemento <label>, el cursor se posiciona en el elemento de control correspondiente, lo que mejora la accesibilidad del formulario.

Ver código HTML de una página web


Ahora que ya aprendimos un poco sobre el HTML básico, podemos explorar el código HTML que usan las páginas web, directamente desde nuestro navegador y sin necesidad de usar herramientas especiales. Lo único que necesitamos son las funciones "Ver código fuente" e "Inspeccionar elemento".

La forma más sencilla para ver el código HTML de una página web es usando la opción "Ver código fuente" o "Ver fuente". La mayoría de navegadores conocidos, como Chrome, Firefox, o Edge, la incluyen como una opción del navegador y no es necesario configurar nada.

El proceso es similar de navegador a navegador, pero en Chrome simplemente debemos hacer clic derecho en cualquier punto de la página web y seleccionar "Ver el código fuente de la página".


Esto nos permitirá ver el documento HTML exacto que el navegador está leyendo. Dependiendo del sitio web, puede ser útil para observar cómo está estructurado el código HTML. Sin embargo, en otros casos, es posible encontrar algo muy desordenado.


Inspeccionar elemento


Una herramienta más simplificada, pero llena de funcionalidades para ver el código HTML de un sitio web es "Inspeccionar elemento". También está incluida en la mayoría de navegadores populares, aunque algunos requieren de una configuración especial para habilitarla.

En Chrome, solo debemos hacer clic derecho en cualquier parte del sitio web y seleccionar "Inspeccionar". Se abrirá la ventana del inspector, la cual se encuentra en la parte de abajo o a la derecha del navegador. Se ve un poco diferente dependiendo del navegador que usemos, pero las funciones básicas son siempre las mismas. Hay muchas pestañas e información interesante en el inspector, como veremos más adelante. Sin embargo, solo la parte HTML del código nos interesa en este momento.

En la ventana del inspector, veremos todo el código HTML de la página desplegado con anotaciones legibles. También podemos usar las flechas al lado de cada elemento para expandirlos y cerrarlos, lo que permite mostrar u ocultar los elementos que contienen:


Una de las funciones más útiles de esta ventana es que la podemos usar para ver la conexión entre un elemento HTML y cómo se ve en la página en tiempo real.

 Cuando pasamos el cursor sobre un elemento en la ventana del inspector, el elemento correspondiente será resaltado en la página web.


Pestañas disponibles y sus usos


En la ventana del inspector, existen diversas pestañas con funcionalidades específicas:


Elements: Muestra el HTML renderizado de la página, que es distinto del código fuente de la página. Si se crea o edita algún elemento HTML mediante JavaScript mientras se carga la página, los cambios se reflejarán en el HTML renderizado, mientras que el código fuente de la página se mostrará sin ninguna alteración.

Resources: Permite inspeccionar una tabla de los distintos recursos que se han cargado junto con la página inspeccionada. Incluye imágenes, documentos HTML y archivos JavaScript, entre otros elementos.

Network: Es un analizador de protocolos de proxy que permite supervisar el tráfico de HTTP de la página, tanto mientras se carga como posteriormente.

Scripts (Secuencias de comandos): Se puede utilizar para la depuración de los códigos de JavaScript.

Timeline (Cronología): Muestra el tráfico HTTP y el uso de la memoria a lo largo del tiempo.

Console (Consola): Detecta automáticamente los errores dentro del código de la página.


¿Qué es la minificación?


Algunos sitios web tienen un HTML simple, como el que hemos estudiado en esta semana. Sin embargo, hay otras páginas que manejan códigos más complejos. Por ejemplo, el HTML de sitios grandes como Amazon o Google es tan denso y confuso que parece algo totalmente distinto a lo que hemos aprendido. Existen varias razones por lo que esto pasa, una de las más comunes es la minificación.

La minificación es un proceso en el que el código de programación —ya sea HTML, CSS o JavaScript— es comprimido automáticamente para reducir el tamaño del archivo.


Cualquier formato que se haya incluido para hacer que el código sea más fácil de leer para un humano es removido, porque nuestro navegador, que es una máquina, no lo necesita. Remover estas cosas también hace que el archivo sea más pequeño, y mientras más pequeño sea un archivo, más rápido lo puede cargar el navegador.
