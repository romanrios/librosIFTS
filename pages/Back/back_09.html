Introducción


API


El término API es una abreviatura de Application Programming Interfaces, que en español significa interfaz de programación de aplicaciones. Se trata de un conjunto de definiciones y protocolos que se utiliza para desarrollar e integrar el software de las aplicaciones, permitiendo la comunicación entre dos aplicaciones de software a través de un conjunto de reglas.

De manera general, podemos concluir que una API es un software que tiene la capacidad de conectarse e interactuar con otros para cumplir una o varias tareas.


¿Por qué REST?

Se refiere a cualquier interfaz entre sistemas que emplean el protocolo de comunicación HTTP para solicitar datos o realizar operaciones sobre estos usando algún formato como JSON.



API REST

Cuando hablamos de REST API nos referimos a un estándar para el desarrollo de WEB API que incluye una arquitectura y protocolos HTTP. REST es un acrónimo de Representational State Transfer.

Con REST podemos realizar las 4 operaciones básicas conocidas como CRUD:


Create: Creación de datos

Read: Lectura de datos

Update: Actualización de datos

Delete: Eliminación de datos


Los métodos básicos de HTTP que se utilizan son:


Post: Para generar o crear un registro nuevo.

Get: Para obtener uno o todos los elementos de un recurso.

Put: Realiza actualizaciones en los registros.

Patch: Realiza actualizaciones parciales de un registro.

Delete: Para borrar un registro.


Todos los objetos se manipulan mediante URI. Por ejemplo, si tenemos un recurso usuario y queremos acceder a un usuario en concreto, nuestra URI sería /user/identificadordelobjeto, con lo cual ya tendríamos un servicio USER preparado para obtener la información de un usuario dado un ID.

A continuación, veremos cómo desarrollar una API REST básica con Node.js, Express y MongoDB.


Implementación de una API REST completa


Introducción

Anteriormente, creamos un servidor web con Node y Express para obtener, crear, modificar y eliminar notas.

Sin embargo, en ese momento no sabíamos utilizar bases de datos, por lo que las notas no persistían ni se modificaban realmente.


¿Que sigue?

Ahora que conocemos el funcionamiento de Mongodb y Mongoose, podemos crear una API REST completa que persista datos como corresponde. Para el ejemplo, vamos a crear una API que gestione series de TV, por lo que debemos crear los modelos y la estructura necesaria.


Creación del proyecto y estructura


Los pasos a tener en cuenta son:
                

Paso 1: Para comenzar, debemos crear una carpeta con el nombre del proyecto en cualquier directorio de nuestra computadora, en este caso lo llamaremos “api-rest-series”.

Paso 2: Abrimos una terminal, nos posicionamos dentro de la carpeta y ejecutamos el comando:

Paso 3: Después, con el siguiente comando procederemos a instalar express, mongoose, body-parser y method-override:

JavaScript Copiar código

Estas son las librerias instaladas:

  • express: La librería para Node.js que ya hemos visto.

  • mongoose: La librería para MongoDB para interactuar con la base de datos, como ya hemos visto.

  • body-parser: Un middleware para express que analiza el cuerpo de las solicitudes HTTP y lo convierte en un objeto JSON accesible. Permite recibir y procesar datos enviados en el cuerpo de una solicitud HTTP.

  • method-override: Un middleware para express que permite sobrescribir el método HTTP utilizado en una solicitud.

Creando los modelos de nuestra API REST

En esta parte vamos a crear un modelo usando Mongoose para poder guardar la información en la base de datos siguiendo el esquema.

Como dijimos, para este ejemplo vamos a crear una base de datos de series de TV, por lo tanto vamos a crear un modelo (en el archivo: models/tvshow.js ) que incluya la información de una serie de TV, como pueden ser su título, el año de inicio, país de producción, una imagen promocional, número de temporadas, género y resumen del argumento:

JavaScript Copiar código 
var mongoose = require('mongoose');
var Schema = mongoose.Schema;

var tvshowSchema = new Schema({
  title: { type: String },
  year: { type: Number },
  country: { type: String },
  poster: { type: String },
  seasons: { type: Number },
  genre: {
    type: String,
    enum: ['Drama', 'Fantasy', 'Sci-Fi', 'Thriller', 'Comedy']
  },
  summary: { type: String }
});

module.exports = mongoose.model('TVShow', tvshowSchema);

Explicación


En este código definimos un esquema (schema) para el modelo de Mongoose llamado "TVShow", que representará a una serie de televisión. El esquema especifica los campos (o propiedades) que un documento de la colección "TVShow" debe tener, y los tipos de datos de cada uno de estos campos.

La primera línea importa el paquete mongoose y guarda una referencia en una variable llamada mongoose. La siguiente línea crea una nueva instancia de Schema utilizando la referencia de mongoose.

JavaScript Copiar código

Luego se crea una nueva instancia de Schema llamada tvshowSchema , que define los campos para el modelo TVShow. Los campos incluyen "title", "year", "country", "poster", "seasons", "genre" y "summary".

Para cada campo se especifica un tipo de datos:

"country": es de tipo String

Finalmente, se utiliza el método model() de mongoose para generar un modelo a partir del esquema y se exporta este modelo para su uso en otras partes de la aplicación.

Implementando los controladores de nuestras rutas o endpoints


Los controladores de las rutas de nuestro API los vamos a crear en un archivo separado que llamaremos controllers/tvshows.js. Gracias a exports conseguimos modularizarlo y que pueda ser llamado desde el archivo principal de la aplicación. 

Obtener todas las series de tv (READ - GET)


El código de a continuación es el comienzo del archivo con la primera función que será la que devuelva todos los registros almacenados:

JavaScript Copiar código 
//File: controllers/tvshows.js

var mongoose = require('mongoose');
var TVShow = mongoose.model('TVShow');

//GET - Return all tvshows in the DB
exports.findAllTVShows = function (req, res) {
  TVShow.find(function (err, tvshows) {
    if (err) res.send(500, err.message);

    console.log('GET /tvshows');
    res.status(200).jsonp(tvshows);
  });
};

Explicación


Este código es un controlador de express para la colección de documentos TVShow.

En la primera línea importamos el paquete mongoose y guardamos una referencia en una variable llamada mongoose .

JavaScript Copiar código

La siguiente línea utiliza el método model() de mongoose para obtener una referencia al modelo TVShow que se había definido previamente.

El controlador exporta una función llamada findAllTVShows, que utilizaremos cuando se reciba una petición GET en la ruta "/tvshows". Obtener una serie de tv (READ - GET)


Si queremos que sólo nos devuelva un registro con un identificador único, tenemos que crear una función tal que la siguiente:

JavaScript Copiar código 
//GET - Return a TVShow with specified ID
exports.findById = function (req, res) {
  TVShow.findById(req.params.id, function (err, tvshow) {
    if (err) return res.send(500, err.message);
    console.log('GET /tvshow/' + req.params.id);
    res.status(200).jsonp(tvshow);
  });
};

Explicación


Este fragmento de código permite recuperar un documento específico de la colección TVShow. La función findById es exportada y la utilizaremos cuando se reciba una petición GET en una ruta con un parámetro id: '/tvshow/:id'.

¿Como funciona?

La función utiliza el método findById() del modelo TVShow para buscar y recuperar un documento específico (una serie) de la colección TVShow de la base de datos. El método toma dos parámetros: el primer parámetro es el identificador del documento a buscar, que obtenemos de req.params.id , el segundo parámetro es una función de callback que se ejecuta cuando la operación de búsqueda ha sido completada.


Al igual que en la función anterior, dentro de la función callback, si hay un error al buscar el documento, se envía una respuesta HTTP con código 500 y el mensaje de error. Si la búsqueda se realiza correctamente, se imprime un mensaje en la consola y se envía una respuesta HTTP con código 200 y el documento recuperado en formato JSON.

Eliminar una serie de tv (DELETE)


Y por último, para completar la funcionalidad CRUD de nuestra API, necesitamos la función que nos permita eliminar registros de la base de datos, y eso lo podemos hacer con el código a continuación:

JavaScript Copiar código 
//DELETE - Delete a TVShow with specified ID
exports.deleteTVShow = function (req, res) {
    TVShow.findById(req.params.id, function (err, tvshow) {
        tvshow.remove(function (err) {
            if (err) return res.send(500, err.message);
            res.status(200).end();
        });
    });
};

Como podemos ver, usamos de nuevo el método .findById() para buscar en la base de datos y para borrarlo usamos .remove() de la misma forma que usamos el .save() para guardar.

Explicación


Esta función se ejecutará cuando se reciba una petición PUT en una ruta con un parámetro id: '/tvshow/:id' .

Buscamos en la base de datos el registro dado el ID, y actualizamos sus campos con los valores que devuelve el cuerpo de la petición.

¿Como es el proceso?

La función utiliza el método findById() del modelo "TVShow" para buscar y recuperar un documento específico de la colección "TVShow" de la base de datos. El método toma dos parámetros: el primer parámetro es el identificador del documento (ID ) a buscar, obtenido de req.params.id, el segundo parámetro es una función de callback que se ejecuta cuando la operación de búsqueda ha sido completada.

Dentro de la función de callback, se actualizan los valores de los campos del objeto "tvshow" con los valores enviados en el cuerpo de la petición (req.body), luego se utiliza el método save() para guardar los cambios en la base de datos.

Si hay un error al guardar los cambios, se envía una respuesta HTTP con código 500 y el mensaje de error. Si el guardado se realiza correctamente, se envía una respuesta HTTP con código 200 y el documento actualizado en formato JSON.

Error 500


Como hemos visto al modificar el código, en lugar de obtener un código de error 500 con el mensaje de error, obtendremos un 404 siempre. Para mejorar esto, podemos modificar el código incluyendo un bloque catch simple para manejar los casos en los que la promesa devuelta por el método findById es rechazada:

JavaScript Copiar código 
//GET - Return a TVShow with specified ID
exports.findById = function (req, res) {
console.log('GET /tvshow/' + req.params.id);
TVShow.findById(req.params.id)
.then((tvshow) => {
if (tvshow) {
res.json(tvshow);
 } else {
res.status(404).end();
}
})
.catch((error) => {
console.log(error);
res.send(500, err.message);
});
};

Estado del objeto

Si no se encuentra ningún objeto coincidente en la base de datos, el valor de tvshow será null y se ejecutará el bloque else. Esto da como resultado una respuesta con el código de estado 404 not found . Si se rechaza la promesa retornada por el método findById, la respuesta tendrá el código de estado 500 internal server error . La consola muestra información más detallada sobre el error.


Además de la serie que no existe, hay una situación de error más que debe manejarse.

Buscar series


Una de las características útiles que podemos ofrecer en una API REST consiste en la búsqueda, en este caso de series, a partir de ciertos parámetros.

Por género

Creamos una función searchByGenre en el archivo /controllers/tvshows.js . Utilizamos el método “find” y buscamos por el atributo genre , el cual se recibe por parámetro en la solicitud GET a /tvshow/genre/:genre

//GET - Search TVShows with specified Genre
exports.searchByGenre = function (req, res) {
    console.log('GET /tvshow/genre/' + req.params.genre);
    TVShow.find({ genre: req.params.genre })
    .then((tvshows) => {
        if (tvshows) {
            res.status(200).jsonp(tvshows);
        } else {
            res.status(404).end();
        }
    })
    .catch((error) => {
        console.log(error);
        res.status(500).end();
    });
};

Agregamos la ruta de búsqueda en el archivo app.js

Nota: 

Si en la URL no mandamos el género con la primera letra mayúscula, la respuesta será un array vacío. No existen series con género “drama”, sino “Drama”. Para arreglar esto, convertimos la primera letra del parámetro genre a mayúscula, de forma tal que siempre podamos pasarle el parámetro genre en minúsculas como convención.


Utilizamos el método charAt(0) para obtener la primera letra del genre .

Utilizamos el método toUpperCase() para convertir ese primer carácter en mayúsculas.

Utilizamos el método slice(1) para obtener una nueva cadena que comienza en el segundo carácter del string original.

Concatenamos los dos valores resultantes (el primer carácter en mayúsculas y la subcadena desde el segundo carácter) y asignamos el resultado a la variable genre .


Middlewares y Hooks en Mongoose


Además del uso que hemos visto, Mongoose nos ofrece varias herramientas poderosas, entre ellas los middlewares y hooks.

Los middlewares en Mongoose están basados en eventos que ocurren en el ciclo de vida de un documento dentro de una colección de mongo. Estos eventos son:

init: es el momento en el que hacemos un find, este es perfecto para cuando necesitamos datos calculados a partir de valores dentro del documento o para asignar valores por defecto dinámicamente.

validate: este evento se da cuando se aplican las reglas de validación que especificamos en el modelo o esquema, también es importante saber que este evento se da antes de save, y si no pasa la validación se aborta el evento save.

save: este es el evento en el cual se guarda el documento en la colección.

remove: este evento ocurre cuando se elimina un documento de la colección, este es perfecto para hacer cascadas de eliminación, cuando borramos un documento, borrar los demás documentos en distintas colecciones que hacen referencia a este.


Para Saber: 

Una de las principales diferencias entre los middlewares, además del momento en el que se ejecutan dependiendo del hook, es los parámetros que recibe el callback. Cuando el hook es pre los parámetros que recibe el callback son next y done. La diferencia entre estos es que done se pasa cuando se desea hacer una ejecución en paralelo después de llamar a next.


JavaScript Copiar código 
const schema = new Schema(..);
schema.pre("save", (next) => {
// do stuff
next();
});
